# Задание 11.1: Config Loader

## Цель

Реализовать типобезопасный загрузчик конфигурации со схемой валидации, автоматическим преобразованием типов и поддержкой переопределений для разных окружений.

## Требования

1. Определите интерфейс `ConfigSchema` с полями: `type` ('string' | 'number' | 'boolean'), `required`, `default`, `validate`
2. Создайте тип `InferConfig<S>`, который выводит типизированный результат из схемы (string/number/boolean для каждого поля)
3. Реализуйте `createConfigLoader(schema)` с методом `load(env, overrides?)`, возвращающим `ConfigResult<T>`
4. `load()` должен: парсить строки в соответствующие типы, применять defaults, проверять required-поля, запускать validate
5. При ошибках возвращайте `{ ok: false, errors: ConfigError[] }` со всеми ошибками (не только первой)
6. Реализуйте метод `merge(base, override)` для объединения конфигураций
7. Продемонстрируйте: успешную загрузку, применение defaults, ошибки валидации, overrides

## Чеклист

- [ ] `ConfigSchema` описывает структуру с type/required/default/validate
- [ ] `InferConfig` выводит number для полей с type: 'number' и т.д.
- [ ] Отсутствие required-поля без default -- ошибка
- [ ] Строка 'not-a-number' для числового поля -- ошибка с описанием
- [ ] Все ошибки собираются за один проход
- [ ] Overrides имеют приоритет над env
- [ ] Merge корректно объединяет два объекта конфигурации

## Как проверить себя

1. Загрузите конфигурацию с полным набором переменных -- все типы должны быть правильными
2. Уберите обязательные переменные -- получите список ошибок
3. Передайте невалидные значения (строку вместо числа) -- получите понятные ошибки
4. Используйте overrides -- они должны перекрыть значения из env
