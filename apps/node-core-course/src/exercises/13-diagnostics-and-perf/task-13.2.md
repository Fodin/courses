# Задание 13.2: Memory Profiling

## Цель

Научиться профилировать память в Node.js: отслеживать `heapUsed`, обнаруживать утечки памяти, использовать heap snapshots и понимать `v8.getHeapStatistics()`.

## Требования

1. Покажите `process.memoryUsage()` с объяснением каждого поля (rss, heapTotal, heapUsed, external)
2. Продемонстрируйте периодический мониторинг памяти для обнаружения утечек
3. Перечислите типичные причины утечек памяти с примерами
4. Покажите ручной GC через `global.gc()` и флаг `--expose-gc`
5. Продемонстрируйте создание heap snapshots через `v8.writeHeapSnapshot()`
6. Покажите `v8.getHeapStatistics()` и на что обращать внимание

## Чеклист

- [ ] process.memoryUsage() с описанием всех полей
- [ ] Периодический мониторинг с formatMB helper
- [ ] 4+ типичных утечки с примерами кода
- [ ] Ручной GC для диагностики
- [ ] Heap snapshots и открытие в Chrome DevTools
- [ ] v8.getHeapStatistics() с ключевыми метриками

## Как проверить себя

1. Нажмите "Запустить" — должна отобразиться полная информация о профилировании
2. Убедитесь, что описаны признаки утечки памяти
3. Проверьте, что показаны типичные причины утечек с fix
4. Убедитесь, что heap snapshots объяснены с инструкцией
