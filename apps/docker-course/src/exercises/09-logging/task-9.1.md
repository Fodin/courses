# Задание 9.1: docker logs

## Цель

Разобраться в команде `docker logs` и её флагах, понять разницу между STDOUT и STDERR, изучить logging drivers и настройку ротации логов.

## Требования

1. Создайте компонент с тремя переключаемыми секциями: "Флаги docker logs", "Logging drivers", "Ротация логов"
2. Секция "Флаги docker logs" показывает таблицу флагов (`-f`, `--tail`, `--since`, `--until`, `-t`, `--details`) с описанием и примером команды
3. Секция "Logging drivers" показывает таблицу драйверов (json-file, local, journald, syslog, fluentd, awslogs, none) с указанием поддержки `docker logs`
4. Секция "Ротация логов" показывает примеры настройки ротации: через CLI-флаги, в docker-compose.yml и в daemon.json
5. Добавьте кнопку-переключатель "Показать STDOUT vs STDERR" -- по нажатию отображается блок с примерами вывода в оба потока (bash, Node.js, Python)
6. Внизу компонента: предупреждение о том, что docker logs показывает только STDOUT/STDERR, а логи в файлы -- невидимы

## Подсказки

- Массив объектов для секций: `{ key, label, content }` -- content может быть JSX или данные для рендера
- `useState<string>` для активной секции, `useState<boolean>` для переключателя STDOUT/STDERR
- Таблица флагов: `{ flag, description, example }[]`
- Таблица драйверов: `{ name, description, supportsLogs }[]`

## Чеклист

- [ ] Три секции переключаются корректно
- [ ] Таблица флагов docker logs (6 флагов)
- [ ] Таблица logging drivers (7 драйверов) с колонкой "docker logs"
- [ ] Примеры ротации: CLI, Compose, daemon.json
- [ ] Кнопка STDOUT vs STDERR показывает/скрывает блок
- [ ] Предупреждение о файловых логах внизу

## Как проверить себя

1. Переключайте секции -- содержимое меняется, каждая секция уникальна
2. В "Флаги docker logs" убедитесь, что все 6 флагов в таблице с примерами
3. В "Logging drivers" для json-file, local, journald стоит "да", для остальных "нет"
4. В "Ротация логов" есть три варианта настройки (CLI, Compose, daemon.json)
5. Кнопка STDOUT/STDERR корректно показывает/скрывает блок с примерами для трёх языков
6. Предупреждение внизу видно всегда
