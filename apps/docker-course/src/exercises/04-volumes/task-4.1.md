# Задание 4.1: Именованные тома

## Цель

Разобраться в именованных томах Docker: их создание, жизненный цикл, управление командами `docker volume` и обмен данными между контейнерами.

## Требования

1. Создайте интерактивный компонент с тремя секциями: "Жизненный цикл", "Команды", "Обмен данными"
2. Секция "Жизненный цикл" -- визуальная диаграмма (ASCII или div-блоки) показывающая: создание тома, подключение к контейнеру, удаление контейнера (данные остаются), подключение к новому контейнеру
3. Секция "Команды" -- интерактивный терминал-симулятор: кнопки с командами `docker volume create/ls/inspect/rm/prune`, при нажатии показывается пример вывода команды
4. Секция "Обмен данными" -- пошаговая демонстрация: два контейнера подключены к одному тому, кнопки "Записать данные" и "Прочитать данные" с визуализацией потока данных
5. Используйте табы или аккордеон для переключения между секциями
6. Добавьте сравнительную таблицу "Именованные vs Анонимные тома" с кнопкой показа/скрытия

## Подсказки

- Используйте `useState` для активной секции, вывода команд, состояния данных
- Для диаграммы жизненного цикла используйте div-блоки со стрелками (символы)
- Для терминала: массив объектов `{ command: string, output: string }`
- Для обмена данными: состояние `sharedData` обновляется кнопками

## Чеклист

- [ ] Три секции переключаются по табам/кнопкам
- [ ] Диаграмма жизненного цикла наглядно показывает персистентность данных
- [ ] Минимум 5 команд `docker volume` с примерами вывода
- [ ] Демонстрация записи/чтения данных через общий том
- [ ] Таблица сравнения именованных и анонимных томов

## Как проверить себя

1. Переключайте секции -- каждая должна отображать свой контент
2. Нажимайте кнопки команд -- вывод должен обновляться
3. Нажмите "Записать данные", затем "Прочитать данные" -- данные должны быть видны
4. Проверьте, что таблица сравнения корректно отображает различия
