# Задание 0.3: Раздача статических файлов

## 🎯 Цель

Реализовать сервер статических файлов с определением Content-Type, потоковой передачей и защитой от path traversal.

## Требования

1. Создайте маппинг расширений файлов на MIME-типы (`.html`, `.css`, `.js`, `.json`, `.png`, `.jpg` и др.)
2. Реализуйте обработку запросов к статическим файлам с определением Content-Type по расширению
3. Верните 404 для несуществующих файлов
4. Реализуйте защиту от path traversal (`../../etc/passwd`) -- проверяйте, что путь остаётся внутри `publicDir`
5. Покажите разницу между `fs.readFileSync` (плохо) и `fs.createReadStream().pipe(res)` (хорошо)

## Чеклист

- [ ] Маппинг расширений на Content-Type покрывает основные типы файлов
- [ ] Для неизвестных расширений используется `application/octet-stream`
- [ ] Файлы отдаются с корректными заголовками Content-Type и Content-Length
- [ ] Возвращается 404 Not Found для отсутствующих файлов
- [ ] Возвращается 403 Forbidden при попытке path traversal
- [ ] Файлы передаются через `stream.pipe(res)`, а не загрузкой в память целиком

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: файлы отдаются с корректными Content-Type, попытки path traversal блокируются с 403, несуществующие файлы возвращают 404, и показана потоковая передача через pipe.
