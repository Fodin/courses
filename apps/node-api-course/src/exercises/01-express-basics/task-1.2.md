# Задание 1.2: Цепочка Middleware

## 🎯 Цель

Понять механизм middleware в Express: функцию `next()`, порядок выполнения, области действия и практические паттерны.

## Требования

1. Объясните сигнатуру middleware: `(req, res, next) => { ... next() ... }`
2. Продемонстрируйте цепочку middleware с замером времени выполнения каждого шага
3. Покажите правильный порядок подключения: парсинг -> CORS -> логирование -> аутентификация -> маршруты -> ошибки
4. Покажите разные области действия: глобальный, на группу маршрутов, на конкретный маршрут

## Чеклист

- [ ] Middleware имеет сигнатуру `(req, res, next)` и вызывает `next()` для передачи управления
- [ ] Цепочка middleware выполняется последовательно с замером времени
- [ ] Показан правильный порядок: парсеры первыми, error handler последним
- [ ] Продемонстрированы три уровня скоупа: `app.use()`, `app.use('/api', ...)`, `app.get('/path', mw, handler)`

## Как проверить себя

Нажмите "Запустить" и проследите порядок выполнения middleware для запроса GET /api/users. Каждый шаг должен показывать имя, действие и время выполнения.
