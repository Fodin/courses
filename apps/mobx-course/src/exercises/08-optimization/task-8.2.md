# Задание 8.2: intercept & observe

## 🎯 Цель

Научиться использовать `intercept` для валидации изменений до их применения и `observe` для логирования изменений после применения.

## 📋 Требования

Создайте `AgeStore` с перехватом и наблюдением:

1. Observable-свойство `age` (начальное значение: 25)
2. Observable-массив `log` для записи событий
3. `intercept` на свойстве `age`:
   - Блокировать значения меньше 0 или больше 150
   - При блокировке добавлять запись в `log`: `[intercept] Blocked: <value> (out of range 0-150)`
   - Вернуть `null` для отмены изменения
4. `observe` на свойстве `age`:
   - При успешном изменении добавлять запись: `[observe] age: <oldValue> -> <newValue>`
5. Метод `setAge(value)` для установки значения
6. Метод `clearLog()` для очистки лога
7. UI: поле ввода, кнопки для тестовых значений (-5, 200), панель лога

```typescript
class AgeStore {
  age: number
  log: string[]
  setAge(value: number): void
  clearLog(): void
}
```

## Чеклист

- [ ] `AgeStore` создан с `makeAutoObservable`
- [ ] `intercept` блокирует значения вне диапазона 0-150
- [ ] Заблокированные изменения записываются в лог с пометкой `[intercept]`
- [ ] `observe` логирует успешные изменения с пометкой `[observe]`
- [ ] Значение `age` не меняется при попытке установить невалидное значение
- [ ] Кнопки «Set -5» и «Set 200» демонстрируют блокировку

## 🔍 Как проверить себя

1. Введите допустимое значение (например, 30) — в логе должна появиться запись `[observe]`
2. Нажмите кнопку «Set -5» — в логе должна появиться запись `[intercept] Blocked`, значение `age` не изменится
3. Нажмите кнопку «Set 200» — аналогично, значение заблокировано
4. Нажмите «Clear log» — лог должен очиститься
