# Задание 1.1: string и number

## Цель

Создать две отдельные схемы — для строки и для числа — и наглядно продемонстрировать их валидацию и приведение типов.

## Требования

1. Создайте `stringSchema` — `yup.string().required().min(2)` с кастомными сообщениями об ошибках

2. Создайте `numberSchema` — `yup.number().required().min(0).max(150)` с кастомными сообщениями

3. Для каждой схемы реализуйте отдельную кнопку "Validate"

4. При успешной валидации покажите результат и его тип (`typeof`)

5. При ошибке покажите сообщение об ошибке

## Чеклист

- [ ] `stringSchema` создана с `required()` и `min(2)`
- [ ] `numberSchema` создана с `required()`, `min(0)`, `max(150)`
- [ ] Каждая схема валидируется отдельной кнопкой
- [ ] Результат показывает значение и его тип
- [ ] Ошибки отображаются корректно

## Как проверить себя

1. Введите "A" в строковое поле — ошибка "Minimum 2 characters"
2. Введите "John" — успех, тип `string`
3. Введите "abc" в числовое поле — ошибка (NaN)
4. Введите "42" в числовое поле — успех, значение `42`, тип `number` (обратите внимание на cast!)
5. Введите "200" — ошибка "Must be <= 150"
