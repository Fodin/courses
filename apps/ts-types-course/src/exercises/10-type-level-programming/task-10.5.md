# Задание 10.5: Type-Level SQL Builder

## Цель

Создать типобезопасный SQL query builder, который проверяет корректность запросов на этапе компиляции: валидные имена таблиц, корректные имена колонок, правильные типы значений в WHERE.

## Требования

1. Определите интерфейс `DBSchema` с таблицами `users`, `posts`, `comments` и их колонками
2. Реализуйте утилитарные типы `TableName`, `ColumnOf<T>` для извлечения метаданных схемы
3. Реализуйте класс `SQLBuilder<T, Selected>` с методами:
   - `select(...columns)` -- выбирает конкретные колонки (с type narrowing результата)
   - `where(column, op, value)` -- добавляет условие фильтрации
   - `orderByCol(column, direction)` -- сортировка
   - `limit(n)` -- ограничение количества
   - `toSQL()` -- генерация SQL-строки
   - `execute()` -- возвращает массив с типом `Pick<Schema[T], Selected>[]`
4. Реализуйте функцию `from<T>(table)` для удобного создания builder
5. Покажите несколько примеров запросов с разными комбинациями методов

## Чеклист

- [ ] `from("users").select("id", "name")` -- компилируется
- [ ] `from("users").select("nonexistent")` -- ошибка компиляции
- [ ] `from("users").where("name", "=", "Alice")` -- компилируется
- [ ] `from("users").where("fake", "=", 1)` -- ошибка компиляции
- [ ] `from("nonexistent")` -- ошибка компиляции
- [ ] `toSQL()` генерирует корректный SQL
- [ ] `execute()` возвращает тип с только выбранными колонками

## Как проверить себя

1. Убедитесь, что `select` сужает тип результата `execute()`
2. Попробуйте передать значение неправильного типа в `where` -- должна быть ошибка
3. Проверьте, что без вызова `select` возвращаются все колонки
