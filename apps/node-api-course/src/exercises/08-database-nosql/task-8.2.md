# Задание 8.2: Mongoose

## 🎯 Цель

Освоить Mongoose ODM: определение схем с валидацией, виртуальные поля, middleware (хуки) и population для загрузки связей.

## Требования

1. Определите Schema с типами, валидацией (required, min, max, enum, match), значениями по умолчанию и timestamps
2. Создайте виртуальные поля (computed) и виртуальные связи (ref)
3. Реализуйте middleware: pre('save') для хеширования пароля, post('save') для побочных эффектов, pre(/^find/) для soft-delete
4. Покажите population: простой, выборочный (select), глубокий (nested), с фильтрацией (match + options)
5. Добавьте instance methods и static methods на схему

## Чеклист

- [ ] Schema определена с валидацией и типами
- [ ] Виртуальные поля вычисляются из существующих данных
- [ ] Middleware pre/post зарегистрированы для save и find
- [ ] Population загружает связанные документы с фильтрацией
- [ ] Instance и static методы добавлены на схему

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: схема содержит валидацию, виртуалы вычисляются, middleware выполняются в правильном порядке, population загружает связи с фильтрацией.
