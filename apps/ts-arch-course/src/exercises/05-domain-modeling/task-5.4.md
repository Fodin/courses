# Задание 5.4: Specifications

## Цель

Реализовать паттерн Specification для инкапсуляции и композиции бизнес-правил с поддержкой логических операторов `and`, `or`, `not`.

## Требования

1. Создайте интерфейс `Specification<T>` с методами `isSatisfiedBy`, `and`, `or`, `not`
2. Реализуйте базовый класс `BaseSpec<T>`, принимающий предикат в конструкторе
3. Создайте фабричную функцию `spec<T>(predicate)` для удобного создания спецификаций
4. Реализуйте набор спецификаций для типа `Product`: `isInStock`, `isAffordable(maxPrice)`, `hasMinRating(min)`, `inCategory(cat)`
5. Продемонстрируйте простые спецификации, композицию через `and`, `or`, `not` и сложные комбинации

## Чеклист

- [ ] `Specification<T>` generic по типу проверяемого объекта
- [ ] `and` возвращает спецификацию, удовлетворённую только если обе исходные удовлетворены
- [ ] `or` возвращает спецификацию, удовлетворённую если хотя бы одна исходная удовлетворена
- [ ] `not` инвертирует результат спецификации
- [ ] Спецификации можно произвольно комбинировать в цепочки
- [ ] `filterBySpec` фильтрует массив по спецификации

## Как проверить себя

1. `isInStock.and(isAffordable(50))` должна пропускать только товары в наличии до $50
2. `isInStock.not()` должна выбирать только товары не в наличии
3. Сложная композиция `(A and B).or(C and D)` должна работать корректно
