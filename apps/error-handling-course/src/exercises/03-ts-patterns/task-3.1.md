# Задание 3.1: Result тип

## Цель
Реализовать паттерн Result для типобезопасной обработки ошибок без throw.

## Требования
1. Определите тип `Result<T, E>` как union `{ ok: true; value: T } | { ok: false; error: E }`
2. Создайте хелперы `ok(value)` и `err(error)`
3. Реализуйте функции:
   - `safeDivide(a, b)` — деление с защитой от нуля
   - `safeParseInt(str)` — парсинг числа
   - `safeJsonParse<T>(json)` — парсинг JSON
4. Покажите цепочку Result-операций

## Чеклист
- [ ] Тип `Result` определён
- [ ] `ok` и `err` хелперы созданы
- [ ] 3 функции реализованы с Result
- [ ] Цепочка операций работает
- [ ] TypeScript корректно сужает типы через `if (result.ok)`
