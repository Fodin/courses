# Задание 4.3: Infer в шаблонных литералах

## Цель

Научиться парсить строковые типы с помощью `infer` и template literal types для извлечения структурированных данных из строк на уровне типов.

## Требования

1. Создайте тип `ExtractDomain<T>`, который извлекает домен из email-строки (`'user@example.com'` -> `'example.com'`)
2. Создайте тип `ParseRoute<T>`, который извлекает имена параметров маршрута (`'/users/:userId/posts/:postId'` -> `'userId' | 'postId'`)
3. Создайте тип `Split<S, D>`, разбивающий строку по разделителю в кортеж (`'a.b.c'` с `'.'` -> `['a', 'b', 'c']`)
4. Создайте тип `KebabToCamel<S>`, преобразующий kebab-case в camelCase (`'get-user-by-id'` -> `'getUserById'`)
5. Создайте тип `ParseQuery<T>`, извлекающий ключи query-параметров из URL-строки (`'page=1&limit=10'` -> `'page' | 'limit'`)

## Чеклист

- [ ] `ExtractDomain` корректно извлекает домен из email
- [ ] `ParseRoute` рекурсивно находит все `:param` в маршруте
- [ ] `Split` возвращает кортеж из частей строки
- [ ] `KebabToCamel` рекурсивно обрабатывает все дефисы
- [ ] `ParseQuery` извлекает ключи из строки query-параметров

## Как проверить себя

1. Нажмите кнопку «Запустить»
2. Убедитесь, что `ParseRoute` находит все параметры в сложном маршруте
3. Убедитесь, что `KebabToCamel` правильно преобразует строки с несколькими дефисами
4. Проверьте, что `Split` возвращает кортеж правильной длины
