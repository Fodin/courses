# Задание 9.3: Server-Sent Events (SSE)

## 🎯 Цель

Освоить SSE: создание endpoint с правильными заголовками, формат событий, автоматическое переподключение клиента с Last-Event-ID.

## Требования

1. Создайте SSE endpoint с заголовками: Content-Type, Cache-Control, Connection, X-Accel-Buffering
2. Покажите формат SSE-событий: id, event, data, retry
3. Отправляйте периодические heartbeat-события
4. Покажите клиентский код: EventSource, обработка именованных событий, переподключение с Last-Event-ID
5. Продемонстрируйте ограничения SSE: невозможность кастомных заголовков, решение через query string / cookies

## Чеклист

- [ ] SSE endpoint отправляет правильные заголовки
- [ ] События соответствуют SSE-формату (id, event, data, пустая строка)
- [ ] Heartbeat предотвращает закрытие соединения
- [ ] Клиент обрабатывает именованные события через addEventListener
- [ ] Переподключение использует Last-Event-ID для возобновления потока

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: SSE endpoint корректно форматирует события, клиент получает и обрабатывает их, переподключение работает с Last-Event-ID.
