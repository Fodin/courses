# Задание 1.4: Теги и версионирование

## Цель

Освоить стратегии тегирования Docker-образов.

## Требования

Создайте компонент, который отображает:

1. **Схему тегирования** — покажите стратегию версионирования для проекта:
   - Семантическое версионирование (`1.0.0`, `1.0`, `1`)
   - Хеш коммита (`abc123f`)
   - Дата сборки (`2024-01-15`)
   - Окружение (`prod`, `staging`, `dev`)
2. **Проблема тега `latest`** — объясните, почему `latest` не означает «последний» и чем это опасно
3. **Команды для работы с тегами**:
   - `docker tag` — создание дополнительных тегов
   - `docker push` — отправка в реестр
4. **Рекомендуемая стратегия** для CI/CD-пайплайна

## Подсказки

- `latest` — просто тег по умолчанию, не обновляется автоматически
- `docker tag myapp:latest myapp:1.0.0` — добавление тега к существующему образу
- В CI/CD обычно тегируют и по семверу, и по хешу коммита
