# Задание 5.1: Readable Streams

## Цель

Понять Readable Streams: два режима работы (flowing и paused), ключевые события и async iterator API.

## Требования

1. Покажите разницу между flowing mode (событие `data`) и paused mode (`readable` + `read()`)
2. Продемонстрируйте создание `ReadableStream` через `fs.createReadStream` с `highWaterMark`
3. Покажите переключение режимов: `pause()` и `resume()`
4. Продемонстрируйте async iterator (`for await...of`) как рекомендуемый подход
5. Симулируйте чтение файла чанками с отображением размера каждого чанка
6. Покажите все ключевые события: `data`, `end`, `error`, `close`, `readable`

## Чеклист

- [ ] Таблица сравнения flowing vs paused mode
- [ ] Примеры кода для обоих режимов
- [ ] pause()/resume() показаны
- [ ] Async iterator продемонстрирован
- [ ] Симуляция чтения чанками работает
- [ ] Все ключевые события перечислены

## Как проверить себя

1. Нажмите "Запустить" и изучите вывод
2. Убедитесь, что симуляция показывает чтение файла по частям
3. Проверьте, что каждый чанк имеет корректный размер
4. Убедитесь, что async iterator упомянут как рекомендуемый подход
