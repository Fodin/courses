# Задание 4.3: Pattern Extraction

## Цель

Реализовать утилиту `PatternMatcher` для типобезопасного извлечения данных из tagged union типов с поддержкой type guards, маппинга и исчерпывающей обработки.

## Требования

1. Реализуйте вспомогательные типы `ExtractTag<T, Tag>` и `ExtractData<T, Tag>` для извлечения конкретного варианта и его данных из union
2. Создайте интерфейс `PatternMatcher<T>` с методами: `extract`, `is`, `map`, `fold`
3. `extract(tag, value)` — возвращает данные варианта или `null`
4. `is(tag, value)` — type guard, сужающий тип value до конкретного варианта
5. `map(tag, value, fn)` — применяет функцию к данным, если тег совпал, иначе `null`
6. `fold(value, handlers)` — исчерпывающая обработка всех вариантов
7. Продемонстрируйте на типе `ApiResponse` с вариантами: `Ok`, `NotFound`, `Unauthorized`, `RateLimit`

## Чеклист

- [ ] `extract('Ok', response)` возвращает `{ items: string[]; total: number } | null`
- [ ] `is('Ok', response)` работает как type guard — после проверки доступны поля `data`
- [ ] `map('Ok', response, fn)` применяет `fn` только к данным варианта `Ok`
- [ ] `fold(response, handlers)` требует обработчик для каждого варианта
- [ ] Типы данных в обработчиках `fold` корректно выведены для каждого варианта
- [ ] Для вариантов без данных (NotFound) обработчик получает `undefined`

## Как проверить себя

1. После `if (matcher.is('Ok', resp))` проверьте, что `resp.data.items` доступен без ошибок
2. В `fold` попробуйте убрать один обработчик — должна быть ошибка компиляции
3. Проверьте, что `extract` возвращает `null` для несовпадающих тегов
