# Задание 5.3: Domain Events

## Цель

Реализовать типобезопасную систему доменных событий с EventBus, поддерживающую подписку, отписку и журналирование событий.

## Требования

1. Создайте generic интерфейс `DomainEvent<T, P>` с полями `type`, `payload`, `timestamp`, `eventId`
2. Определите конкретные типы событий заказа: `OrderCreated`, `OrderItemAdded`, `OrderConfirmed`, `OrderCancelled`
3. Реализуйте класс `EventBus<E>` с методами `on(type, handler)`, `emit(event)`, `getLog()`
4. Метод `on` должен возвращать функцию отписки
5. Обработчики должны получать типизированный payload на основе типа события (через `Extract`)
6. `getLog()` возвращает историю всех отправленных событий

## Чеклист

- [ ] `DomainEvent` generic по типу и payload
- [ ] `on('OrderCreated', handler)` -- handler получает правильно типизированный event
- [ ] `on` возвращает функцию отписки, после вызова которой handler не вызывается
- [ ] `emit` вызывает все зарегистрированные обработчики для данного типа
- [ ] `getLog()` содержит полную историю событий
- [ ] Компонент демонстрирует подписку, эмит, отписку и повторный эмит

## Как проверить себя

1. В обработчике `OrderCreated` проверьте, что `event.payload.orderId` доступен без ошибок TS
2. После отписки обработчик не должен вызываться при следующем `emit`
3. `getLog()` должен содержать все события, включая те, для которых нет обработчиков
