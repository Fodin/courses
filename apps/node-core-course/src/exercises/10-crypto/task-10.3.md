# Задание 10.3: Asymmetric Crypto

## Цель

Освоить асимметричную криптографию: генерацию ключевых пар (generateKeyPairSync), цифровые подписи (sign/verify) и шифрование публичным ключом (publicEncrypt/privateDecrypt).

## Требования

1. Продемонстрируйте генерацию RSA-ключевой пары с настройками кодирования
2. Покажите генерацию EC (Elliptic Curve) ключей
3. Реализуйте симуляцию создания и проверки цифровой подписи
4. Продемонстрируйте шифрование публичным ключом и расшифровку приватным
5. Покажите концепцию гибридного шифрования (RSA + AES)

## Чеклист

- [ ] RSA ключевая пара сгенерирована с PEM-кодированием
- [ ] EC ключи показаны как альтернатива
- [ ] Подпись создана приватным ключом и проверена публичным
- [ ] Шифрование/расшифровка с асимметричными ключами продемонстрированы
- [ ] Упомянуто ограничение RSA на размер шифруемых данных
- [ ] Показана идея гибридного шифрования

## Как проверить себя

1. Нажмите "Запустить" — должен отобразиться весь процесс работы с ключами
2. Убедитесь, что подпись проверяется корректно
3. Проверьте, что упомянуто ограничение ~214 байт для RSA-2048
4. Убедитесь, что гибридное шифрование объяснено
