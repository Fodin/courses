# Задание 10.4: .dockerignore и BuildKit

## Цель

Научиться правильно настраивать .dockerignore для уменьшения build context и использовать возможности BuildKit: параллельную сборку, секреты, SSH-агент и heredoc синтаксис.

## Требования

1. Создайте компонент с двумя секциями: ".dockerignore" и "BuildKit"
2. Секция ".dockerignore" показывает таблицу паттернов синтаксиса (комментарии, wildcards, исключения !, директории) с примерами
3. Добавьте кнопку "Показать шаблон для Node.js" и "Показать шаблон для Python" -- показывают готовые .dockerignore файлы
4. Визуализация: сравнение размера build context с .dockerignore и без (два прогресс-бара с числами)
5. Секция "BuildKit" показывает таблицу возможностей BuildKit: параллельная сборка, cache mounts, секреты (--mount=type=secret), SSH (--mount=type=ssh), heredoc
6. Для каждой возможности BuildKit: кнопка "Пример" показывающая блок кода
7. Внизу: блок с командой проверки версии BuildKit и рекомендацией использовать `# syntax=docker/dockerfile:1`

## Подсказки

- `useState<string>` для активной секции
- `useState<string | null>` для открытого шаблона .dockerignore
- `useState<string | null>` для открытого примера BuildKit
- Таблица паттернов: `{ pattern, description, example }`
- Таблица BuildKit: `{ feature, description, codeExample }`

## Чеклист

- [ ] Две секции переключаются
- [ ] Таблица синтаксиса .dockerignore (5+ паттернов)
- [ ] Кнопки шаблонов для Node.js и Python работают
- [ ] Визуализация размера build context (прогресс-бары)
- [ ] Таблица возможностей BuildKit (5 возможностей)
- [ ] Кнопки примеров BuildKit показывают код
- [ ] Блок с рекомендацией BuildKit внизу

## Как проверить себя

1. Переключайте секции -- содержимое меняется
2. Кнопки шаблонов .dockerignore показывают/скрывают содержимое
3. Прогресс-бары наглядно показывают разницу (например, 1.5 ГБ vs 45 КБ)
4. В "BuildKit" таблица с 5 возможностями
5. Кнопки примеров BuildKit показывают блоки кода
6. Блок с рекомендацией видно всегда
