# Задание 0.4: Conditional Inference

## Цель

Научиться комбинировать generics с условными типами и ключевым словом `infer` для создания утилитных типов, извлекающих информацию из сложных типов.

## Требования

1. Реализуйте тип `IsString<T>`, возвращающий `'yes'` если `T` -- `string`, иначе `'no'`. Создайте runtime-функцию `isString` с аналогичным поведением
2. Реализуйте тип `UnwrapPromise<T>`, извлекающий тип из `Promise<T>`. Если `T` не Promise, возвращайте `T` как есть
3. Реализуйте тип `ElementType<T>`, извлекающий тип элемента массива. Если `T` не массив, возвращайте `T`
4. Реализуйте тип `MyReturnType<T>`, извлекающий тип возвращаемого значения функции
5. Создайте type-safe систему обработки событий с `EventMap` и условными типами для классификации событий по их payload

## Чеклист

- [ ] `IsString<'hello'>` разрешается в `'yes'`, `IsString<42>` -- в `'no'`
- [ ] `UnwrapPromise<Promise<number>>` разрешается в `number`
- [ ] `UnwrapPromise<string>` разрешается в `string` (не Promise)
- [ ] `ElementType<string[]>` разрешается в `string`
- [ ] `MyReturnType<() => boolean>` разрешается в `boolean`
- [ ] Система событий корректно классифицирует mouse/keyboard/other события
- [ ] Показаны compile-time примеры типов наряду с runtime-демонстрациями

## Как проверить себя

1. Проверьте типы через hover в IDE или через `type _Test = IsString<...>`
2. Убедитесь, что `infer` корректно извлекает вложенные типы
3. Попробуйте передать несовместимый payload в обработчик событий
