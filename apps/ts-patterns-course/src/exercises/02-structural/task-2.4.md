# Задание 2.4: Proxy

## Цель

Реализовать два вида прокси для `ApiService` с помощью встроенного `Proxy` API: логирующий и кэширующий.

## Требования

1. Используйте предоставленный интерфейс и класс:
   - `ApiService` с методами `fetchUser(id)`, `fetchProduct(id)`, `updateUser(id, data)`
   - `RealApiService`, реализующий `ApiService`
2. Реализуйте функцию `createLoggingProxy<T extends object>(target: T, logs: string[]): T`:
   - Перехватывает все вызовы методов через `Proxy` и handler `get`
   - Логирует имя метода и аргументы: `[Proxy:LOG] methodName(args)`
   - Логирует результат: `[Proxy:LOG] -> result`
   - Записывает логи в массив `logs`
3. Реализуйте функцию `createCachingProxy<T extends object>(target: T, logs: string[]): T`:
   - Кэширует результаты вызовов по ключу `methodName:JSON.stringify(args)`
   - При попадании в кэш логирует `[Proxy:CACHE HIT]` и возвращает кэшированное значение
   - При промахе логирует `[Proxy:CACHE MISS]` и вызывает оригинальный метод
4. Оба прокси должны быть generic — работать с любым объектом, не только `ApiService`
5. Продемонстрируйте работу обоих прокси

## Чеклист

- [ ] `createLoggingProxy` перехватывает вызовы методов и логирует вход/выход
- [ ] `createCachingProxy` кэширует результаты по ключу метод+аргументы
- [ ] Оба прокси generic (`<T extends object>`) и возвращают тот же тип `T`
- [ ] Используется `Reflect.get` для доступа к свойствам оригинального объекта
- [ ] Проксированный объект имеет тот же интерфейс — клиентский код не замечает прокси
- [ ] Демонстрация показывает логирование и кэширование

## Как проверить себя

1. Нажмите кнопку запуска
2. Logging proxy — каждый вызов метода логирует имя, аргументы и результат
3. Caching proxy — первый вызов `fetchUser('42')` показывает `CACHE MISS`, второй — `CACHE HIT`
4. Прокси полностью прозрачен: `loggingApi.fetchUser('42')` работает как `api.fetchUser('42')`
