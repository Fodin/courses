# Задание 6.4: OAuth 2.0

## 🎯 Цель

Реализовать OAuth 2.0 Authorization Code Flow: редирект на провайдер, callback, обмен кода на токен и получение данных пользователя.

## Требования

1. Покажите шаг 1: редирект на OAuth-провайдер с client_id, redirect_uri, scope, state
2. Покажите шаг 2: callback с authorization code и проверка state (защита от CSRF)
3. Покажите шаг 3: обмен code на access_token через POST к провайдеру
4. Покажите шаг 4: получение данных пользователя с access_token провайдера
5. Покажите шаг 5: создание/поиск пользователя в БД и выдача собственной сессии/JWT

## Чеклист

- [ ] Редирект содержит: client_id, redirect_uri, scope, state (random)
- [ ] State сохраняется в сессии и проверяется при callback
- [ ] Code обменивается с client_secret на серверной стороне (не в браузере!)
- [ ] Access token провайдера используется для GET /user API
- [ ] Результат: создание локальной сессии для нашего приложения

## Как проверить себя

Нажмите "Запустить" и убедитесь, что все 5 шагов OAuth flow показаны последовательно с HTTP-запросами и ответами.
