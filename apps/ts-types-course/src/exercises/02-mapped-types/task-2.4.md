# Задание 2.4: Deep Mapped Types

## Цель

Научиться создавать рекурсивные mapped types для глубокой трансформации вложенных объектов: DeepPartial, DeepReadonly, DeepRequired, DeepNullable.

## Требования

1. Реализуйте `DeepPartial<T>` -- рекурсивно делает все вложенные свойства опциональными. Массивы не разворачивать
2. Реализуйте `DeepReadonly<T>` -- рекурсивно делает все вложенные свойства readonly
3. Реализуйте `DeepRequired<T>` -- рекурсивно удаляет опциональность на всех уровнях вложенности
4. Реализуйте `DeepNullable<T>` -- рекурсивно добавляет `| null` к каждому свойству
5. Реализуйте функцию `deepMerge<T>(target: T, source: DeepPartial<T>): T` для глубокого слияния с типобезопасностью

## Чеклист

- [ ] `DeepPartial` позволяет указать только вложенные свойства без верхнего уровня
- [ ] `DeepPartial` не разворачивает массивы (массив остаётся массивом, а не объектом с числовыми ключами)
- [ ] `DeepReadonly` блокирует мутацию на 2+ уровнях вложенности
- [ ] `DeepRequired` требует все свойства на всех уровнях
- [ ] `DeepNullable` допускает null на любом уровне вложенности
- [ ] `deepMerge` корректно сливает вложенные объекты, сохраняя типобезопасность

## Как проверить себя

1. Создайте `DeepPartial<AppConfig>` с только одним вложенным свойством 3-го уровня
2. Попробуйте мутировать глубокое свойство `DeepReadonly` -- ошибка
3. Вызовите `deepMerge` с частичным override -- проверьте, что базовые значения сохранились
