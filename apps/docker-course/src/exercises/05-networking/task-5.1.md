# Задание 5.1: Bridge network

## Цель

Разобраться в сетевой модели Docker: типы сетевых драйверов, различия между default bridge и пользовательскими bridge-сетями, визуализация архитектуры.

## Требования

1. Создайте интерактивный компонент с тремя секциями: "Сетевые драйверы", "Архитектура bridge", "Default vs Custom"
2. Секция "Сетевые драйверы" -- список всех 5 драйверов Docker (bridge, host, none, overlay, macvlan) с описанием и сценариями использования. Выделите bridge как драйвер по умолчанию
3. Секция "Архитектура bridge" -- текстовая диаграмма (ASCII в `<pre>`) показывающая: хост-машину, виртуальный мост, контейнеры с IP-адресами, NAT/iptables, связь с внешним миром. Добавьте пояснение под диаграммой
4. Секция "Default vs Custom" -- сравнительная таблица минимум из 5 строк: DNS по имени, изоляция, горячее подключение, настройка подсети, рекомендация. Добавьте вывод-рекомендацию
5. Используйте табы для переключения между секциями

## Подсказки

- Для списка драйверов: массив объектов `{ driver, desc, use }`
- Для таблицы: массив `{ feature, default, custom }`
- Используйте `useState<'drivers' | 'diagram' | 'comparison'>` для переключения табов
- ASCII-диаграмма в `<pre>` с моноширинным шрифтом

## Чеклист

- [ ] Три секции переключаются по табам/кнопкам
- [ ] Все 5 сетевых драйверов перечислены с описанием
- [ ] Bridge выделен как драйвер по умолчанию
- [ ] ASCII-диаграмма архитектуры bridge с пояснением
- [ ] Сравнительная таблица минимум из 5 строк
- [ ] Вывод-рекомендация по использованию пользовательских сетей

## Как проверить себя

1. Переключайте секции -- каждая должна отображать свой контент
2. Убедитесь, что в списке драйверов bridge визуально выделен
3. Диаграмма должна быть читаемой и показывать поток трафика
4. Таблица сравнения должна убедительно показывать преимущества custom bridge
