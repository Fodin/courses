# Задание 2.4: Suspense + use + ErrorBoundary

## Цель

Реализовать полный паттерн загрузки данных: Suspense для loading-состояния, ErrorBoundary для ошибок, use() для чтения данных.

## Требования

1. Создайте `ErrorBoundary` компонент (class component с `getDerivedStateFromError`)
2. Создайте функцию, возвращающую промис, который может как resolve, так и reject
3. Создайте компонент, читающий данные через `use(promise)`
4. Оберните всё в `ErrorBoundary > Suspense > Component`
5. Добавьте две кнопки: «Загрузить (успех)» и «Загрузить (ошибка)»
6. При ошибке ErrorBoundary показывает сообщение и кнопку «Попробовать снова»

## Чеклист

- [ ] ErrorBoundary корректно ловит ошибки rejected промиса
- [ ] Suspense показывает fallback во время загрузки
- [ ] use(promise) читает данные в дочернем компоненте
- [ ] Кнопка «успех» загружает данные
- [ ] Кнопка «ошибка» вызывает ErrorBoundary
- [ ] «Попробовать снова» сбрасывает ошибку

## Как проверить себя

1. Нажмите «Загрузить (успех)» — появится loading, затем данные
2. Нажмите «Загрузить (ошибка)» — появится loading, затем сообщение об ошибке
3. Нажмите «Попробовать снова» — ошибка исчезает
