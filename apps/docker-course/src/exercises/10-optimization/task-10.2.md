# Задание 10.2: Multi-stage builds

## Цель

Изучить паттерн multi-stage builds: как разделять сборку и runtime, использовать `COPY --from`, и применять паттерн builder-runner для разных языков программирования.

## Требования

1. Создайте компонент с четырьмя переключаемыми вкладками: "Node.js", "Go", "Python", "Java"
2. Каждая вкладка показывает два блока кода рядом (или друг под другом): "До оптимизации" (однострочный Dockerfile) и "Multi-stage Dockerfile"
3. Для каждого языка покажите размер образа до и после оптимизации с визуальным сравнением (два прогресс-бара)
4. Добавьте кнопку "Показать схему" -- показывает текстовую ASCII-диаграмму, иллюстрирующую что попадает в финальный образ, а что отбрасывается
5. Внизу: таблица с командами для работы с multi-stage (`--target`, `COPY --from=<stage>`, `COPY --from=<image>`)
6. Блок-предупреждение о том, что `COPY --from` может копировать из внешних образов, не только из stages

## Подсказки

- `useState<string>` для активной вкладки языка
- `useState<boolean>` для показа/скрытия ASCII-схемы
- Данные: `{ lang, beforeDockerfile, afterDockerfile, beforeSize, afterSize, sizeReduction }`
- Прогресс-бары: beforeSize = 100%, afterSize пропорционально

## Чеклист

- [ ] Четыре вкладки (Node.js, Go, Python, Java) переключаются
- [ ] Для каждого языка: Dockerfile "до" и "после"
- [ ] Визуальное сравнение размеров (прогресс-бары с числами)
- [ ] Кнопка "Показать схему" с ASCII-диаграммой
- [ ] Таблица команд multi-stage (--target, COPY --from)
- [ ] Предупреждение о COPY --from с внешними образами

## Как проверить себя

1. Переключайте вкладки языков -- Dockerfile-примеры меняются
2. Размеры до/после визуально различимы, числа указаны
3. Кнопка "Показать схему" показывает/скрывает диаграмму
4. Таблица команд содержит минимум 3 записи с примерами
5. Предупреждение видно всегда
