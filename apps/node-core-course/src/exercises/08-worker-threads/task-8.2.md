# Задание 8.2: SharedArrayBuffer & Atomics

## Цель

Понять механизм разделяемой памяти через SharedArrayBuffer, проблему race conditions и атомарные операции через Atomics.

## Требования

1. Создайте SharedArrayBuffer и продемонстрируйте работу с Int32Array view
2. Покажите проблему race condition при одновременном доступе без синхронизации
3. Продемонстрируйте все основные Atomics операции: store, load, add, sub, exchange, compareExchange
4. Объясните механизм wait/notify для синхронизации потоков
5. Покажите ограничение: Atomics.wait нельзя в главном потоке

## Чеклист

- [ ] SharedArrayBuffer создан и данные записаны
- [ ] Race condition проблема наглядно объяснена
- [ ] Все основные Atomics операции продемонстрированы с результатами
- [ ] wait/notify механизм описан
- [ ] Ограничения главного потока указаны

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что SharedArrayBuffer показан визуально с данными
3. Проверьте, что каждая Atomics операция показывает входные и выходные значения
4. Убедитесь, что race condition объяснён на конкретном примере
