# Задание 0.3: Версионирование API

## 🎯 Цель

Создать систему типобезопасного версионирования API, где каждая версия имеет свой набор эндпоинтов и типов ответов, а миграции между версиями типизированы.

## Требования

1. Определите минимум 3 версии API (v1, v2, v3) с эволюционирующими типами данных
2. Создайте интерфейс `ApiVersions`, описывающий все версии и их эндпоинты
3. Реализуйте `createVersionedClient()` — фабрику, создающую клиент для конкретной версии
4. Клиент должен разрешать только эндпоинты, существующие в данной версии
5. Создайте типизированные миграции `MigrationFn<TFrom, TTo>` между версиями
6. Продемонстрируйте цепочку миграций v1 -> v2 -> v3

## Чеклист

- [ ] `ApiVersions` описывает минимум 3 версии с различающимися эндпоинтами
- [ ] `v1Client.get('/users/by-role')` вызывает ошибку (эндпоинт добавлен в v2)
- [ ] `v3Client.get('/users/permissions')` работает (добавлен в v3)
- [ ] Миграция v1 -> v2 -> v3 преобразует данные с полной типизацией
- [ ] Каждая версия добавляет новые поля/эндпоинты

## Как проверить себя

Попробуйте обратиться к эндпоинту, который не существует в выбранной версии. Проверьте, что миграция корректно трансформирует структуру данных через все версии.
