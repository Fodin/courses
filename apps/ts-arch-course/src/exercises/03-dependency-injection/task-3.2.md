# Задание 3.2: Injection Tokens

## 🎯 Цель

Реализовать систему типобезопасных токенов для регистрации и получения сервисов, заменяющую строковые ключи.

## Требования

1. Создайте класс `InjectionToken<T>` с фантомным типом и описанием
2. Реализуйте `TokenContainer` с методами `provide(token, factory)` и `inject(token)`
3. Токен должен определять тип возвращаемого значения `inject()`
4. Поддержите токены для примитивных типов (string, number) и интерфейсов
5. Создайте объект `TOKENS` с минимум 6 токенами разных типов
6. Продемонстрируйте зависимости между сервисами через inject внутри factory

## Чеклист

- [ ] `container.inject(TOKENS.Logger)` возвращает `Logger` (не unknown)
- [ ] `container.inject(TOKENS.ApiBaseUrl)` возвращает `string`
- [ ] Разные токены с одинаковым описанием — разные объекты (нет коллизий)
- [ ] `hasProvider()` корректно проверяет наличие провайдера
- [ ] Фабрика может использовать `container.inject()` для зависимостей

## Как проверить себя

Проверьте, что `inject(TOKENS.Logger).apiUrl` вызывает ошибку (нет apiUrl у Logger). Убедитесь, что примитивные токены возвращают правильные типы.
