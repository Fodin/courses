# Задание 2.2: Error Events & maxListeners

## Цель

Понять особое поведение события "error" и механизм защиты от утечек памяти через maxListeners.

## Требования

1. Покажите, что emit("error") без слушателя крашит процесс
2. Продемонстрируйте корректную обработку ошибок
3. Объясните maxListeners и предупреждение об утечке
4. Покажите типичную утечку слушателей в request handler
5. Продемонстрируйте captureRejections для async ошибок

## Чеклист

- [ ] Показано поведение error без обработчика
- [ ] Показана корректная обработка error
- [ ] Объяснён maxListeners механизм
- [ ] Показан паттерн утечки слушателей
- [ ] Упомянут captureRejections

## Как проверить себя

1. Запустите демонстрацию
2. Проверьте, что симуляция error корректна
3. Убедитесь, что показано предупреждение maxListeners
