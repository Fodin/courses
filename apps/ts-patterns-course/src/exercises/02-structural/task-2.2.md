# Задание 2.2: Decorator

## Цель

Реализовать три функциональных декоратора (`withLogging`, `withCache`, `withRetry`), которые добавляют поведение к любой функции без изменения её сигнатуры.

## Требования

1. Определите тип `AnyFunction = (...args: never[]) => unknown`
2. Реализуйте декоратор `withLogging<T extends AnyFunction>(fn: T, name: string): T`:
   - Оборачивает вызов функции, добавляя к результату префикс `[LOG:<name>]`
   - Возвращает функцию с той же сигнатурой
3. Реализуйте декоратор `withCache<T extends AnyFunction>(fn: T): T`:
   - Кэширует результаты по ключу `JSON.stringify(args)`
   - При повторном вызове с теми же аргументами возвращает `[CACHE HIT] <cached result>`
   - При новых аргументах вызывает оригинальную функцию
4. Реализуйте декоратор `withRetry<T extends AnyFunction>(fn: T, maxRetries: number): T`:
   - При ошибке повторяет вызов до `maxRetries` раз
   - Если все попытки провалились, возвращает `[RETRY FAILED] after N attempts: <error message>`
5. Продемонстрируйте композицию декораторов: `withLogging(withCache(fn), name)`

## Чеклист

- [ ] Тип `AnyFunction` определён
- [ ] `withLogging` добавляет логирование, сохраняя сигнатуру `T`
- [ ] `withCache` кэширует результаты по аргументам
- [ ] `withRetry` повторяет вызов при ошибках
- [ ] Декораторы можно композировать (оборачивать друг в друга)
- [ ] Демонстрация показывает работу каждого декоратора и их композиции

## Как проверить себя

1. Нажмите кнопку запуска
2. `withLogging` — результат начинается с `[LOG:name]`
3. `withCache` — второй вызов с теми же аргументами возвращает `[CACHE HIT]`
4. `withRetry` — ненадёжная функция в итоге возвращает результат
5. Композиция — `withLogging(withCache(fn))` сначала проверяет кэш, потом логирует
