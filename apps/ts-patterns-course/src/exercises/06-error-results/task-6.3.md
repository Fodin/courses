# Задание 6.3: Option / Maybe

## Цель

Реализовать паттерн Option<T> для безопасной работы с отсутствующими значениями вместо null/undefined.

## Требования

1. Создайте классы `Some<T>` и `None` с полем `_tag` ('some' | 'none')
2. Реализуйте метод `map` — трансформация значения внутри Some (None остаётся None)
3. Реализуйте метод `flatMap` — цепочка, где каждый шаг может вернуть None
4. Реализуйте метод `getOrElse(defaultValue)` — извлечение значения с fallback
5. Создайте функцию `fromNullable(value)` — превращает null/undefined в None, остальное в Some
6. Продемонстрируйте безопасный доступ к вложенным свойствам объекта

## Чеклист

- [ ] Классы `Some` и `None` с корректными `_tag`
- [ ] `map` работает для Some, пропускает None
- [ ] `flatMap` разворачивает вложенные Option
- [ ] `getOrElse` возвращает значение или default
- [ ] `fromNullable` корректно обрабатывает null, undefined и значения
- [ ] Демонстрация безопасного доступа к вложенным полям

## Как проверить себя

- Нажмите кнопку — должна отобразиться цепочка доступа к вложенным полям
- Some-значения проходят через map/flatMap
- None пропускает трансформации и возвращает default через getOrElse
- fromNullable(null) === None, fromNullable(42) === Some(42)
