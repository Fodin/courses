# Задание 6.1: State Transitions

## Цель

Реализовать типобезопасный конечный автомат с картой разрешённых переходов, запрещающий невалидные переходы между состояниями.

## Требования

1. Определите тип `MachineState` как union строковых литералов для состояний жизненного цикла HTTP-запроса: `idle`, `fetching`, `success`, `error`
2. Создайте карту допустимых переходов `AllowedTransitions`, описывающую, из какого состояния в какие можно перейти
3. Реализуйте класс `TypedStateMachine` с методами: `state` (геттер текущего), `transition(to)`, `history` (список переходов)
4. `transition()` должен проверять допустимость перехода и выбрасывать ошибку при невалидном
5. Продемонстрируйте полный цикл: idle → fetching → success → fetching → error → idle
6. Покажите отклонение невалидных переходов: idle → success, fetching → idle

## Чеклист

- [ ] `MachineState` -- union type с 4 состояниями
- [ ] Карта переходов задаёт допустимые переходы для каждого состояния
- [ ] `transition('success')` из `idle` выбрасывает ошибку с описанием
- [ ] `transition('fetching')` из `idle` проходит успешно
- [ ] История переходов сохраняется и доступна через `history`
- [ ] Компонент отображает успешные и неуспешные переходы

## Как проверить себя

1. Пройдите полный цикл idle → fetching → success → idle и проверьте историю
2. Попробуйте невалидный переход -- должно быть информативное сообщение об ошибке
3. Убедитесь, что карта переходов покрывает все 4 состояния
