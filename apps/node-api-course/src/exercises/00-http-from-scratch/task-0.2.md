# Задание 0.2: Ручная маршрутизация

## 🎯 Цель

Реализовать маршрутизацию HTTP-запросов вручную с использованием парсинга URL, маршрутизации по методам и обработки query-параметров.

## Требования

1. Распарсите URL с помощью `new URL()` -- извлеките `pathname` и `searchParams`
2. Разбейте путь на сегменты и извлеките параметры маршрута (например, `/api/users/42` -> resource: `users`, id: `42`)
3. Создайте таблицу маршрутов с соответствием метод + паттерн -> обработчик
4. Реализуйте маршрутизацию: сопоставление входящего запроса с таблицей маршрутов
5. Обработайте query-параметры (`page`, `limit`, `sort`) из URL

## Чеклист

- [ ] URL парсится через `new URL(url, base)`
- [ ] Сегменты пути извлекаются через `pathname.split('/').filter(Boolean)`
- [ ] Таблица маршрутов содержит GET, POST, PUT, DELETE для `/api/users`
- [ ] Запрос корректно сопоставляется с маршрутом по методу и пути
- [ ] Возвращается 404 для неизвестных маршрутов и 405 для неподдерживаемых методов
- [ ] Query-параметры извлекаются через `searchParams.get()`

## Как проверить себя

Нажмите "Запустить" и убедитесь, что разные запросы (GET /api/users, GET /api/users/42, POST /api/users, GET /api/posts) обрабатываются с правильными статус-кодами и вызовами обработчиков.
