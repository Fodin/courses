# Задание 4.1: Zod Validation

## 🎯 Цель

Реализовать валидацию запросов API с помощью Zod: схемы для body, params, query, форматирование ошибок и TypeScript-вывод типов.

## Требования

1. Определите Zod-схему для создания пользователя: `name` (string, min 2), `email` (email), `age` (int, optional), `role` (enum), `tags` (array)
2. Создайте составную схему с валидацией body, params и query в одном объекте
3. Продемонстрируйте валидацию валидного и невалидного запроса через `safeParse()`
4. Реализуйте middleware `validate(schema)` для автоматической валидации с ответом 422
5. Покажите TypeScript-вывод типов: `z.infer<typeof schema>`

## Чеклист

- [ ] Zod-схема использует `.string()`, `.email()`, `.number().int()`, `.enum()`, `.array()`
- [ ] `safeParse()` используется вместо `parse()` для безопасной валидации
- [ ] Ошибки форматируются: path + message для каждого невалидного поля
- [ ] Middleware автоматически валидирует и сохраняет результат в `req.validated`
- [ ] TypeScript выводит тип из схемы без дублирования интерфейсов

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: валидный запрос проходит с выведенным типом, невалидный возвращает 422 со списком ошибок по полям.
