# Задание 4.3: tmpfs и readonly

## Цель

Изучить tmpfs-монтирования для хранения данных в оперативной памяти и read-only режимы для повышения безопасности контейнеров.

## Требования

1. Создайте компонент с тремя секциями: "tmpfs", "Read-only контейнер", "Комбинирование"
2. Секция "tmpfs" -- описание с use cases (секреты, временные файлы, высокопроизводительный I/O), таблица параметров tmpfs (tmpfs-size, tmpfs-mode), примеры команд
3. Секция "Read-only контейнер" -- пошаговая демонстрация: кнопка "Запустить --read-only" показывает ошибку, затем кнопка "Добавить tmpfs-исключения" показывает рабочую конфигурацию. Визуализируйте, какие директории доступны для записи, а какие нет
4. Секция "Комбинирование" -- интерактивный конструктор production-конфигурации: чекбоксы для выбора опций (--read-only, tmpfs /tmp, tmpfs /run, volume для данных, bind mount :ro для конфига), на выходе -- итоговая команда `docker run` и визуальная схема монтирований
5. Добавьте таблицу сравнения всех трёх типов хранения: volumes, bind mounts, tmpfs

## Подсказки

- Для пошаговой демонстрации: состояние `step` (0, 1, 2), каждый шаг показывает свой контент
- Для конструктора: объект состояния с булевыми полями для каждой опции
- Для визуальной схемы: div-блоки с цветовой кодировкой (зелёный=rw, жёлтый=ro, синий=tmpfs)

## Чеклист

- [ ] Секция tmpfs содержит минимум 3 use case и таблицу параметров
- [ ] Пошаговая демонстрация read-only: ошибка -> решение
- [ ] Конструктор генерирует команду с выбранными опциями
- [ ] Визуальная схема показывает типы монтирований с цветовой кодировкой
- [ ] Таблица сравнения volumes / bind mounts / tmpfs

## Как проверить себя

1. Переключайте секции -- каждая отображает свой контент
2. В пошаговой демонстрации нажимайте кнопки -- должны видеть ошибку, затем решение
3. В конструкторе включайте/выключайте опции -- команда и схема обновляются
4. Проверьте, что итоговые команды синтаксически корректны
