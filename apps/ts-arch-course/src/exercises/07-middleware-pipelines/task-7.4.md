# Задание 7.4: Plugin Architecture

## 🎯 Цель

Спроектировать типобезопасную плагинную систему с типизированными hook-точками, где каждый плагин декларативно подписывается на события с проверкой payload на этапе компиляции.

## Требования

1. Создайте интерфейс `PluginHooks` с минимум 5 именованными хуками, каждый со своим типом payload
2. Создайте тип `HookHandler<T>` и интерфейс `Plugin` с полями `name`, `version`, `hooks` (Partial по PluginHooks)
3. Реализуйте класс `PluginSystem`:
   - `register(plugin)` — регистрирует плагин и его хуки
   - `emit(hook, payload)` — типобезопасно вызывает все обработчики хука
   - `getPlugins()` — возвращает список зарегистрированных плагинов
4. Создайте минимум 3 плагина: logging, metrics, security

## Чеклист

- [ ] `PluginHooks` определяет минимум 5 хуков с разными типами payload
- [ ] `Plugin.hooks` использует `Partial` — плагин подписывается только на нужные хуки
- [ ] `emit('app:init', payload)` проверяет тип payload по имени хука
- [ ] `emit('unknown', {})` вызывает ошибку компиляции
- [ ] Плагины вызываются в порядке регистрации
- [ ] `getPlugins()` возвращает типизированный список

## Как проверить себя

Зарегистрируйте 3 плагина и вызовите `emit` для каждого хука. Убедитесь, что: (1) вызываются только подписанные плагины, (2) payload проверяется компилятором, (3) неизвестные хуки вызывают ошибку.
