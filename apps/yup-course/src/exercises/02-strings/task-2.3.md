# Задание 2.3: matches (regex)

## Цель

Использовать `matches()` для валидации строк по регулярным выражениям.

## Требования

1. Создайте `phoneSchema` — валидация российского номера телефона:
   - Паттерн: `/^\+7\d{10}$/`
   - Сообщение: "Phone must be in format +7XXXXXXXXXX"

2. Создайте `hexColorSchema` — валидация hex-цвета:
   - Паттерн: `/^#[0-9A-Fa-f]{6}$/`
   - Сообщение: "Must be a valid hex color (e.g. #FF0000)"

3. При успешной валидации цвета покажите превью цвета (квадрат с этим цветом)

4. Оба поля должны быть обязательными

## Чеклист

- [ ] `phoneSchema` создана с `matches()` и паттерном для +7
- [ ] `hexColorSchema` создана с `matches()` и паттерном для hex
- [ ] Телефон "+79001234567" проходит валидацию
- [ ] Цвет "#FF0000" проходит валидацию и показывает красный квадрат
- [ ] Некорректные значения показывают ошибки

## Как проверить себя

1. Телефон: "+79001234567" — успех
2. Телефон: "89001234567" — ошибка (нет +7)
3. Телефон: "+7123" — ошибка (мало цифр)
4. Цвет: "#FF0000" — успех + красный квадрат
5. Цвет: "#abc" — ошибка (только 3 символа)
6. Цвет: "red" — ошибка (не hex формат)
