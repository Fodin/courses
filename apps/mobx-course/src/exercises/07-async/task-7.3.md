# Задание 7.3: Отмена flow

## 🎯 Цель

Научиться отменять выполняющийся flow через `promise.cancel()` и настраивать cleanup при размонтировании компонента.

## 📋 Требования

Создайте стор `SlowDataStore` с медленной загрузкой (3 секунды) и возможностью отмены:

1. Объявите observable-поля: `data: string | null`, `isLoading: boolean`
2. Реализуйте `fetchData` через `flow`:
   - Добавьте искусственную задержку 3 секунды (`yield new Promise(...)`)
   - После задержки запишите строку с текущим временем в `this.data`
   - В `finally` сбросьте `isLoading = false` (важно для корректной отмены)
3. В observer-компоненте:
   - Запустите `fetchData` в `useEffect` при монтировании
   - Верните `promise.cancel()` из cleanup-функции `useEffect`
   - Добавьте кнопку «Cancel» для ручной отмены
   - Добавьте кнопку «Reload» для повторной загрузки

```typescript
// flow возвращает CancellablePromise
const promise = store.fetchData()
promise.cancel() // Отменяет выполнение генератора
```

## Чеклист

- [ ] `SlowDataStore` с `fetchData` через `flow`
- [ ] Задержка 3 секунды перед загрузкой данных
- [ ] `finally` блок сбрасывает `isLoading`
- [ ] `useEffect` запускает `fetchData` и отменяет в cleanup
- [ ] Кнопка «Cancel» для ручной отмены
- [ ] Кнопка «Reload» для повторной загрузки

## 🔍 Как проверить себя

1. При монтировании компонента начинается 3-секундная загрузка
2. Нажмите «Cancel» до окончания — загрузка прервётся, `isLoading` станет `false`
3. Нажмите «Reload» — загрузка начнётся заново
4. Дождитесь 3 секунд — появится строка с временем
5. Быстро переключитесь на другое задание и обратно — при unmount старый запрос отменяется
