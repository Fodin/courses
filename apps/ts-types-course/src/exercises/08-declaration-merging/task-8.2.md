# Задание 8.2: Module Augmentation

## Цель

Научиться расширять типы сторонних модулей через `declare module`, добавлять новые методы к существующим типам и создавать типобезопасные plugin-системы.

## Требования

1. Объясните концепцию `declare module 'library-name' { ... }` и покажите синтаксис
2. Покажите пример расширения Express Request с пользовательским полем `user`
3. Объясните расширение встроенных типов через `declare global { interface Array<T> { ... } }`
4. Создайте локальную демонстрацию augmentation через слияние интерфейса `OriginalAPI`
5. Реализуйте plugin-систему через `PluginRegistry` с плагинами `core`, `auth`, `analytics`
6. Покажите augmentation через namespace для расширения lodash-подобных типов
7. Перечислите ограничения module augmentation (нельзя добавлять top-level, нужен модуль)

## Чеклист

- [ ] Синтаксис `declare module` объяснён
- [ ] Пример расширения Express Request показан
- [ ] `declare global` для встроенных типов объяснён
- [ ] Локальная демонстрация augmentation работает
- [ ] Plugin registry с 3 плагинами реализован и вызывается
- [ ] Ограничения module augmentation перечислены
- [ ] Результаты отображаются на странице

## Как проверить себя

1. Нажмите кнопку «Запустить»
2. Убедитесь, что `OriginalAPI` содержит и оригинальные, и расширенные методы
3. Проверьте, что plugin registry вызывает методы каждого плагина
4. Убедитесь, что ограничения augmentation перечислены
