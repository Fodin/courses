# Task 0.2: Docker Architecture

## Objective

Study the main components of Docker's architecture and understand how they interact with each other.

## Requirements

1. Create a visual diagram of Docker's architecture with the following components: Docker Client, Docker Daemon, containerd, runc, Registry
2. Add a description of each component's role
3. Show the direction of interaction between components (arrows or connections)
4. Add a section describing Docker objects: Images, Containers, Volumes, Networks
5. Implement interactivity: clicking on a component shows its detailed description

## Checklist

- [ ] The diagram contains all 5 key components
- [ ] Each component has a brief role description
- [ ] Connections/directions of interaction between components are shown
- [ ] A section with Docker objects is present (at least 4 types)
- [ ] Clicking a component shows its details
- [ ] The component uses `useState` to manage the selected element

## How to verify

1. Open the task in the browser
2. Make sure all architecture components are displayed
3. Click on each component — a description should appear
4. Verify that the connections between components are logically correct
5. Compare with the solution — make sure no component is missing
