# Задание 5.2: Unit of Work (Транзакции с commit/rollback)

## Цель

Реализовать паттерн Unit of Work для группировки операций над несколькими сущностями в атомарную транзакцию с поддержкой commit и rollback.

## Требования

1. Создайте интерфейс `UnitOfWork` с методами:
   - `registerNew<T extends Entity>(entity: T, repoName: string): void` -- регистрация новой сущности
   - `registerDirty<T extends Entity>(entity: T, repoName: string): void` -- регистрация изменённой сущности
   - `registerDeleted(id: string, repoName: string): void` -- регистрация удаления
   - `commit(): string[]` -- применение всех изменений, возвращает лог операций
   - `rollback(): void` -- очистка всех накопленных изменений
2. Реализуйте класс `InMemoryUnitOfWork`, принимающий `Map<string, Repository<Entity>>` -- карту репозиториев
3. `commit()` должен применять операции в порядке: создание -> обновление -> удаление
4. После `commit()` внутренние списки операций очищаются
5. `rollback()` очищает все накопленные операции без их применения

## Чеклист

- [ ] Интерфейс `UnitOfWork` определён со всеми методами
- [ ] `registerNew`, `registerDirty`, `registerDeleted` накапливают операции
- [ ] `commit()` применяет операции к соответствующим репозиториям
- [ ] Порядок commit: create -> update -> delete
- [ ] `commit()` возвращает лог выполненных операций
- [ ] После commit внутренние списки пусты
- [ ] `rollback()` очищает накопленные операции без применения
- [ ] Демонстрация показывает сценарий commit и rollback

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что после commit данные появились в репозитории
3. Убедитесь, что после rollback репозиторий не изменился
4. Проверьте, что лог операций содержит все применённые действия
