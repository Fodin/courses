# Задание 5.1: Ошибки fetch

## Цель
Понять особенности обработки ошибок в fetch API.

## Требования
1. Покажите, что fetch НЕ бросает ошибку при HTTP 404 — проверьте `response.ok`
2. Покажите сетевую ошибку (запрос к несуществующему домену) — это `TypeError`
3. Покажите ошибку парсинга JSON — `SyntaxError`
4. Реализуйте правильный паттерн: проверка `response.ok` + throw
5. Отобразите все результаты

## Чеклист
- [ ] fetch при 404 не кидает ошибку — показано
- [ ] Сетевая ошибка — TypeError
- [ ] Ошибка парсинга — SyntaxError
- [ ] Правильный паттерн с `response.ok` проверкой
