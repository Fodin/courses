# Задание 9.4: Recursion Limits

## Цель

Понять ограничения рекурсии в TypeScript, научиться использовать tail-recursive optimization и workaround-ы для глубокой рекурсии.

## Требования

1. Объясните лимиты рекурсии TypeScript (~50 уровней для conditional types)
2. Покажите пример наивной рекурсии, которая достигает лимита
3. Продемонстрируйте tail-recursive optimization (TypeScript 4.5+): `BuildTuple<N>` с аккумулятором
4. Покажите паттерн аккумулятора на примере `Reverse<T>` — наивная vs хвостовая версия
5. Реализуйте арифметику через tuple length: `Add<A, B>` с `BuildTuple`
6. Покажите рекурсивную проверку массива `CheckAll<T>` с ранним выходом
7. Перечислите best practices для рекурсивных типов
8. Реализуйте паттерн depth limiter: `SafeDeepReadonly<T, D>` с `MaxDepth` tuple

## Чеклист

- [ ] Лимиты рекурсии объяснены (50 уровней, сообщение об ошибке)
- [ ] Наивная рекурсия показана как anti-pattern
- [ ] Tail recursion с аккумулятором продемонстрирована
- [ ] `Reverse<[1,2,3,4,5]>` = `[5,4,3,2,1]` через хвостовую рекурсию
- [ ] `Add<3, 4>` = `7` через tuple length
- [ ] `CheckAll` с ранним выходом работает
- [ ] Best practices перечислены
- [ ] Depth limiter предотвращает бесконечную рекурсию
- [ ] Результаты отображаются на странице

## Как проверить себя

1. Нажмите кнопку «Запустить»
2. Убедитесь, что объяснена разница между наивной и хвостовой рекурсией
3. Проверьте, что `Add<3, 4>` корректно вычисляет 7
4. Убедитесь, что depth limiter паттерн с `MaxDepth` tuple объяснён
