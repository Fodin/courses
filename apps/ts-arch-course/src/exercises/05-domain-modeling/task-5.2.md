# Задание 5.2: Entities & Aggregates

## Цель

Реализовать модель сущностей (Entities) с идентичностью по ID и агрегат (Aggregate) с корневой сущностью, контролирующей бизнес-инварианты.

## Требования

1. Создайте generic тип `EntityId<T>` на основе branded types для типизированных идентификаторов
2. Реализуйте базовый интерфейс `Entity<Id>` с полями `id`, `createdAt`, `updatedAt`
3. Создайте агрегат `Order` с вложенными `OrderItem` сущностями
4. `Order` должен быть Aggregate Root: все операции с `OrderItem` проходят через функции Order
5. Реализуйте операции: `createOrder` (минимум 1 товар), `addItemToOrder` (только для draft), `confirmOrder`
6. Все изменения должны возвращать новый объект (иммутабельность) и обновлять `updatedAt`

## Чеклист

- [ ] `OrderId` и `OrderItemId` -- разные типы, несовместимы друг с другом
- [ ] `createOrder` с пустым массивом items выбрасывает ошибку
- [ ] `addItemToOrder` работает только для заказов со статусом `draft`
- [ ] `confirmOrder` меняет статус и обновляет `updatedAt`
- [ ] Два `OrderItem` с одинаковыми данными имеют разные `id`
- [ ] `calculateOrderTotal` корректно суммирует цены с учётом количества

## Как проверить себя

1. Создайте заказ, добавьте товар, подтвердите -- проверьте статусы на каждом шаге
2. Попробуйте добавить товар к подтверждённому заказу -- должна быть ошибка
3. Сравните ID двух независимо созданных OrderItem -- они должны быть уникальны
