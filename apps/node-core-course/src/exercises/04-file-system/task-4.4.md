# Задание 4.4: File Watching

## Цель

Научиться отслеживать изменения файлов с помощью `fs.watch()` и `fs.watchFile()`, понять разницу между ними и типичные проблемы.

## Требования

1. Продемонстрируйте `fs.watch()` с обработкой событий `change` и `rename`
2. Покажите `fs.watchFile()` с polling и сравнение `curr`/`prev` stats
3. Покажите таблицу сравнения `fs.watch()` vs `fs.watchFile()`
4. Продемонстрируйте проблему дублирования событий и решение через debouncing
5. Покажите async iterator API для `fs.watch()` (Node.js 18.11+)
6. Продемонстрируйте использование chokidar как production-решение

## Чеклист

- [ ] fs.watch() с событиями change/rename показан
- [ ] fs.watchFile() с polling продемонстрирован
- [ ] Таблица сравнения watch vs watchFile присутствует
- [ ] Проблема дублирования и debouncing объяснены
- [ ] Async iterator для watch показан
- [ ] Симуляция событий работает корректно

## Как проверить себя

1. Нажмите "Запустить" и изучите сравнительную таблицу
2. Убедитесь, что объяснена разница между OS notifications и polling
3. Проверьте, что симуляция показывает дублированные события и их фильтрацию
4. Убедитесь, что chokidar упомянут как production-решение
