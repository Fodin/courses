# Задание 10.3: Оптимизация слоёв и кэширование

## Цель

Понять как работает кэширование слоёв Docker, научиться оптимизировать порядок инструкций для максимального использования кэша, объединять RUN-инструкции и использовать BuildKit cache mounts.

## Требования

1. Создайте компонент с тремя секциями: "Порядок инструкций", "Объединение RUN", "Cache mounts"
2. Секция "Порядок инструкций" показывает два Dockerfile: плохой (COPY . . перед npm install) и хороший (package.json перед npm install), с визуализацией кэширования слоёв (cached/miss индикаторы для каждой инструкции)
3. Добавьте интерактивный переключатель "Файл изменён" -- при включении показывает какие слои инвалидируются в каждом из двух вариантов
4. Секция "Объединение RUN" показывает примеры: раздельные RUN vs объединённые, с указанием размера (файл удалён в отдельном слое vs в том же слое)
5. Секция "Cache mounts" показывает примеры `--mount=type=cache` для npm, pip, go, apt
6. Внизу: правило "Редко меняющееся -- сверху, часто -- снизу" с иконкой

## Подсказки

- `useState<string>` для активной секции
- `useState<boolean>` для переключателя "Файл изменён"
- Визуализация кэша: список инструкций с цветовыми индикаторами (зелёный = cached, красный = miss, жёлтый = rebuild)
- Массив: `{ instruction, cached: boolean, reason: string }`

## Чеклист

- [ ] Три секции переключаются
- [ ] Два Dockerfile с визуализацией кэша слоёв
- [ ] Переключатель "Файл изменён" меняет индикаторы кэша
- [ ] Примеры объединения RUN с указанием размера
- [ ] Примеры cache mounts для 4 пакетных менеджеров
- [ ] Правило-вывод внизу

## Как проверить себя

1. В секции "Порядок инструкций" два варианта Dockerfile с цветовыми индикаторами
2. Переключатель "Файл изменён" изменяет индикаторы (в плохом варианте больше miss)
3. В "Объединение RUN" понятно видна разница в размере
4. Cache mounts: 4 примера для разных языков
5. Правило видно всегда
