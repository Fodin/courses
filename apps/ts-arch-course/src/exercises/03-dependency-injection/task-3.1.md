# Задание 3.1: DI-контейнер

## 🎯 Цель

Реализовать типобезопасный DI-контейнер, в котором `resolve()` автоматически выводит тип сервиса из накопленного generic-параметра.

## Требования

1. Создайте класс `Container<TRegistry>` с методами `register()` и `resolve()`
2. `register()` принимает имя сервиса, фабричную функцию и возвращает расширенный контейнер
3. Фабричная функция получает контейнер как аргумент для доступа к другим сервисам
4. `resolve()` возвращает типизированный экземпляр сервиса (singleton)
5. Зарегистрируйте минимум 4 сервиса с зависимостями друг на друга
6. Добавьте метод `has()` для проверки наличия сервиса

## Чеклист

- [ ] `container.resolve('config')` возвращает типизированный Config
- [ ] `container.resolve('unknown')` — ошибка компиляции
- [ ] Фабрика может использовать `c.resolve()` для доступа к зависимостям
- [ ] Повторный `resolve()` возвращает тот же экземпляр (singleton)
- [ ] Generic-параметр накапливается при каждом `register()`

## Как проверить себя

Проверьте автодополнение после `container.resolve('config').` — должны быть доступны только поля Config. Попробуйте resolve несуществующего сервиса.
