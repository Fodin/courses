# Задание 3.3: COPY vs ADD, .dockerignore

## Цель

Разобраться в различиях между COPY и ADD, научиться создавать .dockerignore для оптимизации контекста сборки и обеспечения безопасности.

## Требования

1. Создайте таблицу сравнения COPY vs ADD с колонками: сценарий, COPY, ADD, рекомендация
2. Включите минимум 4 сценария: копирование файлов, копирование с правами, распаковка tar, скачивание URL
3. Подсветите рекомендуемый вариант для каждого сценария
4. Реализуйте интерактивную визуализацию размера контекста сборки: без .dockerignore vs с .dockerignore
5. Покажите разбивку по категориям файлов (node_modules, .git, source code и т.д.) с цветовыми индикаторами
6. Добавьте кнопку показа/скрытия примера .dockerignore для Node.js-проекта

## Подсказки

- Для таблицы используйте массив объектов с полем recommended: 'copy' | 'add' | 'neither'
- Для визуализации контекста создайте два набора данных (с .dockerignore и без) и переключайтесь между ними
- Покажите полоски разного размера для визуального отображения объёма файлов
- Пример .dockerignore должен быть реалистичным и содержать комментарии

## Чеклист

- [ ] Таблица COPY vs ADD содержит минимум 4 сценария
- [ ] Рекомендуемый вариант визуально выделен
- [ ] Кнопки переключения "Без .dockerignore" / "С .dockerignore" работают
- [ ] Отображается общий размер контекста и разбивка по категориям
- [ ] Разница в размере наглядно видна
- [ ] Пример .dockerignore показывается/скрывается по кнопке
- [ ] Пример содержит реалистичные паттерны с комментариями

## Как проверить себя

1. Проверьте таблицу -- для обычного копирования должен быть рекомендован COPY
2. Переключите контекст -- размер без .dockerignore должен быть значительно больше
3. Откройте пример .dockerignore -- он должен включать node_modules, .git, .env и другие типичные исключения
