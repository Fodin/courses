# Задание 5.4: Custom networks

## Цель

Научиться создавать и управлять пользовательскими сетями Docker: полный набор команд `docker network`, multi-network архитектуры, практические паттерны развёртывания.

## Требования

1. Создайте компонент с тремя секциями: "Команды", "Multi-network", "Паттерны"
2. Секция "Команды" -- интерактивный терминал-симулятор: список из 8 команд (create, create с параметрами, ls, inspect, connect, disconnect, rm, prune). При нажатии на команду показывается её вывод. Каждая команда -- отдельная кнопка
3. Секция "Multi-network" -- пример архитектуры с двумя сетями: блок кода с командами создания, текстовая диаграмма (ASCII в `<pre>`) показывающая frontend/backend сети, api как мост. Визуализация работающих и заблокированных маршрутов
4. Секция "Паттерны" -- три карточки с реальными сценариями: веб-приложение с БД, reverse proxy, отладка сети. Каждая карточка: название, команды, описание
5. Используйте табы для навигации

## Подсказки

- Для терминала: массив `{ cmd, output, desc }`, `useState` для активной команды
- Для multi-network: `<pre>` для диаграммы, объяснение отдельным блоком
- Для паттернов: массив карточек с `borderLeft` для визуального акцента
- Цветовая схема: фиолетовый (#7b1fa2) как акцентный цвет

## Чеклист

- [ ] Три секции переключаются по табам
- [ ] Минимум 8 команд docker network с выводом
- [ ] Пример multi-network с ASCII-диаграммой
- [ ] Показаны работающие и заблокированные маршруты
- [ ] Три карточки паттернов: веб+БД, reverse proxy, отладка
- [ ] Каждая карточка содержит реальные, исполнимые команды

## Как проверить себя

1. Нажимайте на команды -- вывод должен меняться
2. ASCII-диаграмма multi-network читаема и показывает изоляцию
3. Все команды Docker синтаксически корректны
4. Паттерны показывают реальные production-сценарии
