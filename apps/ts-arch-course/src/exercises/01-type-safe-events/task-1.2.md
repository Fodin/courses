# Задание 1.2: Event Bus

## 🎯 Цель

Создать централизованную шину событий с поддержкой middleware, истории событий и типобезопасной публикации/подписки.

## Требования

1. Создайте класс `EventBus<TEvents>` с методами `subscribe()`, `publish()` и `use()`
2. Реализуйте middleware-цепочку: каждый middleware получает event, payload и `next()`
3. Middleware может модифицировать поведение, логировать или блокировать событие (не вызывать next)
4. Добавьте историю событий: `getHistory()`, `clearHistory()`
5. Создайте минимум 2 middleware (логирование и таймер)
6. Используйте реалистичный набор бизнес-событий

## Чеклист

- [ ] `publish` проверяет тип payload для каждого события
- [ ] Middleware вызываются в порядке добавления
- [ ] Middleware может прервать цепочку, не вызвав `next()`
- [ ] История содержит все отправленные события с timestamps
- [ ] `subscribe` возвращает функцию отписки

## Как проверить себя

Добавьте middleware, который не вызывает `next()` для определённого события. Убедитесь, что подписчики этого события не получают уведомлений. Проверьте историю на полноту.
