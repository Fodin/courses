# Задание 8.1: Granular Observer

## 🎯 Цель

Научиться оптимизировать рендеринг больших списков, вынося каждый элемент в отдельный observer-компонент.

## 📋 Требования

Создайте `BigListStore` и компонент списка с гранулярным observer:

1. `BigListStore` содержит массив из 100 элементов `{ id, name, value }`
2. Метод `incrementValue(id)` увеличивает `value` конкретного элемента на 1
3. Computed `total` — сумма всех `value`
4. Компонент `BigListItem` — отдельный `observer`, отображающий один элемент
5. Каждый `BigListItem` отслеживает количество своих ререндеров через `useRef`
6. При клике на кнопку `+1` должен ререндериться **только** кликнутый элемент

```typescript
interface Item {
  id: number
  name: string
  value: number
}

class BigListStore {
  items: Item[] // 100 элементов
  incrementValue(id: number): void
  get total(): number
}
```

## Чеклист

- [ ] `BigListStore` создан с `makeAutoObservable` и 100 элементами
- [ ] `incrementValue` корректно увеличивает значение элемента по id
- [ ] Computed `total` считает сумму всех value
- [ ] `BigListItem` обёрнут в `observer`
- [ ] Счётчик ререндеров отображается для каждого элемента
- [ ] При клике на `+1` ререндерится только один элемент (счётчик растёт только у него)

## 🔍 Как проверить себя

1. Откройте список и кликните `+1` у любого элемента
2. Убедитесь, что счётчик ререндеров увеличился **только** у кликнутого элемента
3. Проверьте, что `total` обновляется корректно
4. Сравните поведение с вариантом, где `BigListItem` не обёрнут в `observer` — весь список будет ререндериться
