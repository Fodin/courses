# Задание 0.5: Контрактное тестирование

## 🎯 Цель

Реализовать систему runtime-валидации API-ответов с помощью композируемых type guards, которые одновременно проверяют данные и сужают типы.

## Требования

1. Создайте базовые type guards: `isString`, `isNumber`, `isBoolean`
2. Реализуйте композитор `isObject()` — создаёт guard для объекта из схемы guards для каждого поля
3. Реализуйте композитор `isArray()` — создаёт guard для массива из guard элемента
4. Реализуйте `isOneOf()` — guard для union-типов из литералов
5. Создайте функцию `validateContract()` — валидатор с накоплением ошибок
6. Продемонстрируйте валидацию на валидных, невалидных и граничных данных

## Чеклист

- [ ] Базовые guards корректно проверяют примитивные типы
- [ ] `isObject` композирует guards для всех полей объекта
- [ ] `isArray(isUser)` валидирует массив пользователей
- [ ] `isOneOf('admin', 'user')` проверяет принадлежность к набору значений
- [ ] `validateContract` возвращает структурированный результат с ошибками
- [ ] Граничные случаи обработаны: null, undefined, пустой объект, неверные типы

## Как проверить себя

Проверьте, что после проверки `isUser(data)` TypeScript сужает тип до `User`. Убедитесь, что guard отклоняет данные с лишними полями неверного типа и отсутствующими обязательными полями.
