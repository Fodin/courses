# Задание 2.1: Маршруты и JSON Schema

## 🎯 Цель

Освоить систему маршрутизации Fastify с встроенной валидацией запросов и сериализацией ответов через JSON Schema.

## Требования

1. Определите маршрут с JSON Schema для валидации `body` (required fields, типы, минимальная длина, формат email)
2. Определите `response` schema для автоматической сериализации и фильтрации полей
3. Продемонстрируйте валидацию валидного и невалидного запроса
4. Покажите, как response schema автоматически удаляет лишние поля (password, internalId)
5. Объясните преимущество `fast-json-stringify` для сериализации ответов

## Чеклист

- [ ] Body schema определяет required, types, constraints (minLength, format)
- [ ] `additionalProperties: false` блокирует лишние поля
- [ ] Невалидный запрос возвращает 400 с описанием ошибки
- [ ] Response schema фильтрует чувствительные поля (password, internal данные)
- [ ] Показана разница: что возвращает БД vs что получает клиент

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: валидный запрос проходит валидацию, невалидный -- отклоняется с 400, response schema отфильтровывает password и internalId из ответа.
