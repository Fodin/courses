# Задание 4.2: Bind mounts

## Цель

Понять механизм bind mounts -- прямого монтирования директорий и файлов хоста в контейнер, сравнить синтаксис `-v` и `--mount`, разобрать типичные сценарии использования.

## Требования

1. Создайте компонент с двумя основными секциями: "Сценарии использования" и "Синтаксис -v vs --mount"
2. Секция "Сценарии" -- карточки с тремя use case: разработка (hot-reload), конфигурационные файлы, логирование. Каждая карточка: название, описание, пример команды в `<pre>`-блоке
3. Секция "Синтаксис" -- параллельное сравнение `-v` и `--mount` для трёх операций: volume mount, bind mount, bind mount readonly. Показывайте оба варианта рядом для каждой операции
4. Добавьте интерактивный "конструктор команд" bind mount: выпадающие списки или кнопки для выбора параметров (тип: файл/директория, доступ: rw/ro, синтаксис: -v/--mount), на выходе -- готовая команда `docker run`
5. Покажите блок "Частые ловушки" с двумя примерами: относительный путь и перезапись node_modules

## Подсказки

- Для карточек: массив объектов с полями `title`, `description`, `command`
- Для конструктора: три состояния (type, access, syntax), генерация команды через шаблонную строку
- Для сравнения синтаксиса: таблица с 3 колонками (операция, `-v`, `--mount`)

## Чеклист

- [ ] Минимум 3 карточки сценариев с примерами команд
- [ ] Параллельное сравнение `-v` vs `--mount` для 3 операций
- [ ] Конструктор команд генерирует корректную команду при выборе параметров
- [ ] Блок "Частые ловушки" с примерами плохого и правильного кода
- [ ] Все примеры используют абсолютные пути

## Как проверить себя

1. Переключайте секции -- контент должен меняться
2. В конструкторе меняйте параметры -- команда должна обновляться
3. Проверьте, что сгенерированные команды синтаксически корректны
4. Убедитесь, что в примерах bind mount используются абсолютные пути
