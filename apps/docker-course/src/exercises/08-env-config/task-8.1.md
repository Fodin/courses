# Задание 8.1: ENV и .env файлы

## Цель

Разобраться в механизмах передачи переменных окружения: инструкция `ENV` в Dockerfile, флаг `-e` при запуске, `.env` файлы в Docker Compose, синтаксис подстановки переменных и приоритет их разрешения.

## Требования

1. Создайте компонент с пятью переключаемыми секциями: "ENV в Dockerfile", "ARG vs ENV", "Флаг -e при запуске", ".env файл", "Подстановка в Compose"
2. Каждая секция содержит блок кода (Dockerfile / bash / YAML) и список ключевых моментов
3. Для секции "ARG vs ENV" покажите различия: доступность при build/run, способы переопределения, предупреждение о секретах в ARG
4. Для секции "Подстановка в Compose" покажите синтаксис: `${VAR:-default}`, `${VAR:?error}`, `${VAR:+alt}`
5. Добавьте кнопку-переключатель для отображения таблицы приоритетов переменных (5 уровней: от `environment:` в compose до `ENV` в Dockerfile)
6. Внизу компонента покажите сводку по синтаксису подстановки (4 варианта в виде сетки)

## Подсказки

- Массив объектов для секций: `{ key, label, code, notes }` -- code это многострочная строка, notes это массив строк
- `useState<string>` для активной секции, `useState<boolean>` для показа/скрытия таблицы приоритетов
- Таблица приоритетов: `{ source, priority, example }[]`
- Сводка подстановки: grid 2x2 с вариантами синтаксиса

## Чеклист

- [ ] Пять секций переключаются корректно
- [ ] Каждая секция: блок кода + список ключевых моментов
- [ ] ARG vs ENV: чёткие различия (build vs run, --build-arg vs -e)
- [ ] Подстановка: ${VAR:-default}, ${VAR:?error}, ${VAR:+alt}
- [ ] Таблица приоритетов (5 уровней) по кнопке
- [ ] Предупреждение о секретах в ARG
- [ ] Сводка синтаксиса подстановки внизу

## Как проверить себя

1. Переключайте секции -- содержимое меняется, код для каждой уникален
2. В "ARG vs ENV" убедитесь, что есть предупреждение о docker history
3. Нажмите кнопку приоритетов -- таблица появляется/скрывается, 5 строк отсортированы по приоритету
4. В "Подстановка в Compose" описаны все три варианта синтаксиса
5. Сводка внизу показывает 4 варианта подстановки в виде сетки
