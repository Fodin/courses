# Задание 0.2: Архитектура Docker

## Цель

Изучить основные компоненты архитектуры Docker и понять, как они взаимодействуют друг с другом.

## Требования

1. Создайте визуальную схему архитектуры Docker с компонентами: Docker Client, Docker Daemon, containerd, runc, Registry
2. Для каждого компонента добавьте описание его роли
3. Покажите направление взаимодействия между компонентами (стрелки или связи)
4. Добавьте секцию с описанием Docker-объектов: Images, Containers, Volumes, Networks
5. Реализуйте интерактивность: при клике на компонент показывается его подробное описание

## Чеклист

- [ ] Схема содержит все 5 ключевых компонентов
- [ ] Каждый компонент имеет краткое описание роли
- [ ] Показаны связи/направления взаимодействия между компонентами
- [ ] Есть секция с Docker-объектами (минимум 4 типа)
- [ ] Клик по компоненту показывает подробности
- [ ] Компонент использует `useState` для управления выбранным элементом

## Как проверить себя

1. Откройте задание в браузере
2. Убедитесь, что все компоненты архитектуры отображаются
3. Кликните по каждому компоненту -- должно появиться описание
4. Проверьте, что связи между компонентами логичны
5. Сравните с решением -- убедитесь, что ни один компонент не пропущен
