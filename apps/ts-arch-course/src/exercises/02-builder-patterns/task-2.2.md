# Задание 2.2: Accumulating Builder

## 🎯 Цель

Реализовать Builder, который отслеживает установленные обязательные свойства в generic-параметре и разрешает build() только когда все обязательные поля заданы.

## Требования

1. Создайте класс `FormBuilder<TSet>`, где TSet накапливает информацию о заполненных полях
2. Определите минимум 3 обязательных поля (title, action, method) и 3 опциональных
3. Каждый setter обязательного поля возвращает `FormBuilder<TSet & { field: true }>`
4. Метод `build()` использует this-параметр для требования всех обязательных полей
5. Порядок вызова методов не должен быть фиксирован
6. Продемонстрируйте: полную сборку, минимальную сборку, невалидную попытку

## Чеклист

- [ ] `new FormBuilder().title("x").build()` — ошибка (не хватает полей)
- [ ] `new FormBuilder().title("x").action("/y").method("POST").build()` — работает
- [ ] Порядок вызова методов не влияет на результат
- [ ] Опциональные поля имеют дефолтные значения в build()
- [ ] TypeScript показывает, какие именно обязательные поля отсутствуют

## Как проверить себя

Попробуйте вызвать build() после установки только 1 и 2 обязательных полей — проверьте сообщения об ошибках. Убедитесь, что порядок вызовов не важен.
