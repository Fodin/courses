# Задание 12.3: Contract Tests

## Цель

Реализовать паттерн контрактного тестирования для верификации, что разные реализации интерфейса ведут себя одинаково.

## Требования

1. Определите интерфейс `StorageContract<T>` с методами: `get`, `set`, `has`, `delete`, `clear`, `size`
2. Создайте типы `ContractTest<T>` и `ContractSuite<T>` для описания и запуска тестов
3. Реализуйте `createContractSuite(name, tests)` с методом `run(impl)`
4. Напишите набор контрактных тестов `storageContractTests<T>(sample, another)` для StorageContract
5. Создайте две реализации: `MapStorage<T>` и `ObjectStorage<T>`
6. Продемонстрируйте: обе реализации проходят одни и те же тесты

## Чеклист

- [ ] `ContractTest` содержит name и функцию test, возвращающую `{ passed, message }`
- [ ] `ContractSuite.run()` запускает все тесты и возвращает результаты
- [ ] Минимум 8 контрактных тестов покрывают все методы StorageContract
- [ ] Каждый тест начинает с `clear()` -- изоляция от других тестов
- [ ] `MapStorage` и `ObjectStorage` проходят все тесты
- [ ] Тесты параметризованы: работают для любого типа T

## Как проверить себя

1. Запустите suite для MapStorage -- все тесты должны пройти
2. Запустите suite для ObjectStorage -- все тесты тоже должны пройти
3. Намеренно сломайте одну реализацию -- контрактный тест должен это поймать
4. Создайте suite для другого типа (string вместо number) -- тесты работают
