# Задание 2.4: Retry паттерн

## Цель
Реализовать функцию автоматических повторных попыток при ошибке.

## Требования
1. Создайте «нестабильную» API-функцию, которая успешно отвечает только с 3-й попытки
2. Реализуйте функцию `retry<T>(fn, options)` с параметрами:
   - `maxAttempts` — максимальное число попыток
   - `delay` — задержка между попытками (мс)
   - `onRetry` — колбэк при повторе (для логирования)
3. Отобразите процесс retry в реальном времени на странице
4. Покажите финальный результат (успех или все попытки исчерпаны)

## Чеклист
- [ ] Нестабильная API-функция создана
- [ ] `retry` реализована с типизацией
- [ ] Прогресс отображается в реальном времени
- [ ] Успешный результат после нескольких попыток
- [ ] Обработан случай исчерпания попыток
