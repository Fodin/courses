# Задание 0.1: Первая схема

## Цель

Создать Yup-схему для валидации объекта пользователя и проверить данные с помощью `validate()`.

## Требования

1. Создайте схему `userSchema` с помощью `yup.object()` со следующими полями:
   - `name` — обязательная строка (`string().required()`)
   - `age` — обязательное положительное число (`number().required().positive()`)
   - `email` — обязательная строка в формате email (`string().email().required()`)

2. Выведите тип `UserData` из схемы с помощью `yup.InferType`

3. При нажатии кнопки "Validate" вызовите `userSchema.validate(data)` с введёнными данными

4. При успешной валидации отобразите валидированные данные в зелёном блоке

5. При ошибке валидации отобразите все ошибки в красном блоке

## Чеклист

- [ ] Схема `userSchema` создана с тремя полями
- [ ] Тип `UserData` выведен через `InferType`
- [ ] Кнопка "Validate" запускает валидацию
- [ ] Валидные данные отображаются в зелёном блоке
- [ ] Ошибки отображаются в красном блоке
- [ ] Используется `{ abortEarly: false }` для сбора всех ошибок

## Как проверить себя

1. Оставьте все поля пустыми и нажмите "Validate" — должны появиться 3 ошибки
2. Введите имя "John", возраст 25, email "john@example.com" — должен появиться зелёный блок с данными
3. Введите возраст -5 — должна появиться ошибка "Age must be positive"
4. Введите email "bad" — должна появиться ошибка про невалидный email
