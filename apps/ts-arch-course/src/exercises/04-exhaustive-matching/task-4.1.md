# Задание 4.1: Match Expression

## Цель

Создать типобезопасную утилиту `match()`, которая гарантирует обработку всех вариантов строкового union type на уровне компиляции.

## Требования

1. Реализуйте функцию `match<T extends string>(value: T)`, возвращающую объект с методом `with(handlers)`
2. Тип `handlers` должен быть mapped type `{ [K in T]: (value: K) => R }`, гарантирующий наличие обработчика для каждого варианта
3. Реализуйте `matchTagged<T extends { kind: string }>(value: T)` для работы с tagged unions
4. В `matchTagged` используйте `Extract<T, { kind: K }>` для типобезопасного сужения типа в каждом обработчике
5. Продемонстрируйте на примере HTTP-методов (`GET | POST | PUT | DELETE | PATCH`) и геометрических фигур

## Чеклист

- [ ] `match()` принимает строковый литеральный тип и возвращает builder с методом `with`
- [ ] `with()` требует обработчик для каждого варианта — пропуск вызывает ошибку компиляции
- [ ] `matchTagged()` работает с объектами, имеющими поле-дискриминатор `kind`
- [ ] В обработчиках `matchTagged` доступны специфичные поля каждого варианта (radius для circle и т.д.)
- [ ] Оба match-выражения возвращают значение (expression, а не statement)
- [ ] Компонент отображает результаты работы обеих версий match

## Как проверить себя

1. Попробуйте убрать один обработчик из `with()` — должна появиться ошибка компиляции
2. В обработчике `matchTagged` для `circle` проверьте, что TypeScript знает про `radius`
3. Убедитесь, что `match` возвращает типизированный результат, а не `any`
