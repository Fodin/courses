# Задание 11.2: Сканирование уязвимостей

## Цель

Разобраться в инструментах сканирования Docker-образов на уязвимости (CVE), понять уровни серьёзности, научиться интегрировать сканирование в CI/CD. Создать интерактивный компонент-справочник по сканированию.

## Требования

1. Создайте компонент с тремя переключаемыми секциями: "Уровни CVE", "Инструменты сканирования", "CI/CD интеграция"
2. Секция "Уровни CVE" показывает таблицу с четырьмя уровнями серьёзности (CRITICAL, HIGH, MEDIUM, LOW), описанием каждого, примером CVE, SLA на исправление и цветной индикацией
3. Секция "Инструменты сканирования" показывает карточки для 4 инструментов: Docker Scout, Trivy, Grype, Snyk -- с описанием, ключевыми командами, плюсами/минусами
4. Для каждого инструмента добавьте кнопку "Показать команды" -- по клику отображается блок кода с основными командами
5. Секция "CI/CD интеграция" показывает пример GitHub Actions workflow для сканирования при push и периодического ночного сканирования -- с пояснениями к каждому шагу
6. Добавьте интерактивный "симулятор результатов сканирования" -- кнопка "Запустить сканирование", по клику показывается пример вывода с найденными уязвимостями разных уровней
7. Внизу компонента -- блок "Рекомендация" с советом по выбору инструмента

## Подсказки

- `useState<string>` для активной секции
- `useState<Record<string, boolean>>` для видимости команд каждого инструмента
- `useState<boolean>` для симулятора сканирования (показан/скрыт результат)
- Цвета CVE: CRITICAL -- красный, HIGH -- оранжевый, MEDIUM -- жёлтый, LOW -- серый

## Чеклист

- [ ] Три секции переключаются
- [ ] Таблица CVE-уровней с 4 записями и цветами
- [ ] Карточки 4 инструментов сканирования
- [ ] Кнопки "Показать команды" работают независимо
- [ ] CI/CD примеры с пояснениями
- [ ] Симулятор сканирования с кнопкой и результатами
- [ ] Блок "Рекомендация" внизу

## Как проверить себя

1. Переключайте секции -- содержимое меняется
2. Таблица CVE содержит 4 уровня с цветами и примерами
3. У каждого инструмента есть кнопка "Показать команды" -- работает
4. В CI/CD видны примеры workflow с пояснениями
5. Кнопка "Запустить сканирование" показывает результат с CVE разных уровней
6. Блок рекомендации виден внизу
