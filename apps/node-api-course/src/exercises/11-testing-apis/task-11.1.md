# Задание 11.1: Unit Testing

## 🎯 Цель

Освоить unit-тестирование API: мокирование зависимостей с vi.fn()/vi.spyOn(), тестирование сервисов и обработчиков запросов в изоляции.

## Требования

1. Создайте мок-зависимости (Database, Mailer) через `vi.fn()` с `mockResolvedValue`
2. Тестируйте сервис: проверяйте вызовы моков (toHaveBeenCalledWith, toHaveBeenCalledTimes)
3. Создайте mock req/res для тестирования обработчиков без Express
4. Тестируйте happy path (200) и error path (404, 400)
5. Используйте beforeEach для сброса моков и afterEach для restoreAllMocks

## Чеклист

- [ ] Зависимости замоканы через vi.fn() с правильными возвращаемыми значениями
- [ ] Тесты проверяют и результат, и взаимодействие с моками
- [ ] Mock req/res позволяют тестировать обработчики изолированно
- [ ] Покрыты happy path и error cases
- [ ] Моки сбрасываются между тестами

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: моки создаются и проверяются, обработчики тестируются через mock req/res, все тесты проходят.
