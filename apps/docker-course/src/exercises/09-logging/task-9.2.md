# Задание 9.2: inspect, stats, top

## Цель

Освоить команды инспекции и мониторинга контейнеров: `docker inspect` с Go templates, `docker stats` для мониторинга ресурсов, `docker top` для просмотра процессов и `docker events` для отслеживания событий.

## Требования

1. Создайте компонент с четырьмя переключаемыми секциями: "docker inspect", "docker stats", "docker top", "docker events"
2. Секция "docker inspect" показывает список полезных Go templates (IP-адрес, статус, exit code, env, mounts, OOMKilled, PID, порты) -- каждый с командой и примером вывода
3. Секция "docker stats" показывает таблицу колонок (CPU%, MEM, NET I/O, BLOCK I/O, PIDS) и пример пользовательского формата
4. Секция "docker top" показывает пример вывода и использование с дополнительными полями ps
5. Секция "docker events" показывает фильтры (event, container, type) и типичные события жизненного цикла контейнера
6. Добавьте кнопку "Показать шпаргалку" -- по нажатию отображается компактная сводка всех четырёх команд с самыми полезными вариантами

## Подсказки

- `useState<string>` для активной секции, `useState<boolean>` для шпаргалки
- Для inspect templates: массив `{ label, template, output }[]`
- Для stats: массив колонок `{ column, description }[]`
- Для events: массив фильтров `{ filter, example }[]` и массив событий жизненного цикла

## Чеклист

- [ ] Четыре секции переключаются корректно
- [ ] docker inspect: минимум 8 Go templates с командами и выводом
- [ ] docker stats: таблица колонок + пример формата
- [ ] docker top: пример вывода + пользовательские поля
- [ ] docker events: фильтры + события жизненного цикла
- [ ] Шпаргалка по кнопке: сводка всех четырёх команд

## Как проверить себя

1. Переключайте секции -- каждая показывает свой набор данных
2. В "docker inspect" минимум 8 шаблонов с реалистичным выводом
3. В "docker stats" описаны все колонки + пример пользовательского формата
4. В "docker top" есть пример вывода процессов
5. В "docker events" перечислены фильтры и типичные события (create, start, die, stop, destroy)
6. Шпаргалка показывается/скрывается по кнопке, содержит все 4 команды
