# Задание 8.3: Worker Pool

## Цель

Реализовать паттерн Worker Pool для переиспользования потоков и эффективной параллельной обработки задач.

## Требования

1. Создайте симуляцию Worker Pool с фиксированным количеством workers
2. Реализуйте очередь задач: если все workers заняты, задача ставится в очередь
3. Покажите распределение задач между workers визуально
4. Продемонстрируйте код класса WorkerPool с методами runTask и destroy
5. Покажите сравнение: параллельное vs последовательное выполнение

## Чеклист

- [ ] Pool создаётся с настраиваемым количеством workers
- [ ] Задачи распределяются между свободными workers
- [ ] Очередь задач работает при нехватке свободных workers
- [ ] Визуальное отображение распределения задач
- [ ] Код WorkerPool показан и объяснён
- [ ] Сравнение производительности (параллельно vs последовательно)

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что задачи распределены между workers (не все на одном)
3. Проверьте, что время параллельного выполнения меньше последовательного
4. Убедитесь, что код WorkerPool содержит все необходимые методы
