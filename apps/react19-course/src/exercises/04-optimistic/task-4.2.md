# Задание 4.2: Оптимистичный список

## Цель

Оптимистично добавлять элементы в список до подтверждения сервером.

## Требования

1. Создайте список элементов в `useState`
2. Используйте `useOptimistic` с reducer-функцией для добавления элементов
3. Оптимистичные элементы должны визуально отличаться (например, полупрозрачные)
4. После подтверждения сервером элемент становится обычным

## Чеклист

- [ ] `useOptimistic` с кастомным reducer
- [ ] Новый элемент появляется мгновенно с пометкой «отправка...»
- [ ] После ответа сервера пометка исчезает
- [ ] Список обновляется корректно

## Как проверить себя

1. Введите текст и нажмите «Добавить»
2. Элемент появится мгновенно (полупрозрачный)
3. Через 1-2 секунды он станет обычным
