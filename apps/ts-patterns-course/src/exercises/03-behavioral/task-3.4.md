# Задание 3.4: Chain of Responsibility

## Цель

Реализовать цепочку обработчиков HTTP-запроса: авторизация, rate limiting, валидация, логирование.

## Требования

1. Определите интерфейс `HttpRequest` с полями: `userId`, `token`, `body`, `timestamp`, `logs: string[]`
2. Создайте интерфейс `Handler<T>` с методами:
   - `setNext(handler: Handler<T>): Handler<T>`
   - `handle(request: T): T | null` (null = запрос отклонён)
3. Создайте абстрактный `BaseHandler` с реализацией `setNext` и передачей в `next`
4. Реализуйте обработчики:
   - `AuthHandler` — проверяет token из набора валидных
   - `RateLimitHandler` — считает запросы по userId, отклоняет при превышении лимита
   - `ValidationHandler` — проверяет, что body не пустой
   - `LogHandler` — логирует прохождение запроса
5. Соберите цепочку: Auth → RateLimit → Validation → Log
6. Продемонстрируйте: валидный запрос, невалидный токен, пустой body, превышение лимита

## Чеклист

- [ ] `Handler<T>` типизирован дженериком
- [ ] `BaseHandler` реализует передачу в следующий обработчик
- [ ] `AuthHandler` отклоняет невалидные токены
- [ ] `RateLimitHandler` считает запросы и отклоняет при превышении
- [ ] `ValidationHandler` проверяет наличие body
- [ ] `LogHandler` логирует финальную обработку
- [ ] Отклонённый запрос не доходит до следующих обработчиков

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что валидный запрос проходит все 4 обработчика
3. Убедитесь, что невалидный токен отклоняется на первом шаге
4. Убедитесь, что превышение лимита отклоняет запрос
