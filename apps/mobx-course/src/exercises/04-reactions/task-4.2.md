# Задание 4.2: reaction — Search Query Store

## 🎯 Цель

Создать стор поиска и настроить `reaction`, который запускает поиск только при изменении запроса, но не при инициализации.

## 📋 Требования

1. Создайте класс `SearchStore` с observable полями:
   - `query: string` (по умолчанию `''`)
   - `results: string[]` (по умолчанию `[]`)
   - Приватный массив `allItems` со списком элементов для поиска
2. Добавьте метод `setQuery(value)` — устанавливает поисковый запрос
3. Добавьте метод `search()` — фильтрует `allItems` по `query` и записывает результат в `results`
4. В компоненте создайте `reaction` внутри `useEffect`:
   - Data function: `() => searchStore.query`
   - Effect function: принимает `query` и `previousQuery`, вызывает `searchStore.search()` и пишет запись в лог
5. Верните disposer в cleanup-функции `useEffect`
6. Подключите `<input>` для ввода запроса и отобразите список результатов

## Чеклист

- [ ] `SearchStore` создан с `makeAutoObservable`
- [ ] Используется `reaction`, а не `autorun`
- [ ] При монтировании компонента effect **не** запускается (лог пуст)
- [ ] При вводе текста в поле запускается `search()` и результаты отображаются
- [ ] Лог показывает предыдущее и текущее значение запроса
- [ ] Disposer вызывается в cleanup `useEffect`

## 🔍 Как проверить себя

1. При загрузке страницы лог должен быть пуст — `reaction` не срабатывает при инициализации
2. Введите текст в поле поиска — в логе появится запись вида `Query changed: "" → "a"`
3. Продолжите ввод — каждое изменение query добавляет запись с предыдущим и текущим значением
4. Результаты поиска обновляются при каждом изменении запроса
