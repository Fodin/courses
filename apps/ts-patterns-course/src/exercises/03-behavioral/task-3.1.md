# Задание 3.1: Strategy (Стратегия сортировки)

## Цель

Реализовать паттерн Strategy для переключения между алгоритмами сортировки без изменения клиентского кода.

## Требования

1. Создайте интерфейс `SortStrategy<T>` с полями:
   - `name: string` — название алгоритма
   - `sort(data: T[], compare: (a: T, b: T) => number): T[]` — сортировка массива
2. Реализуйте три стратегии: `BubbleSort<T>`, `QuickSort<T>`, `MergeSort<T>`
3. Каждая стратегия должна возвращать **новый массив**, не мутируя исходный
4. Создайте класс `Sorter<T>` с методами:
   - `constructor(strategy: SortStrategy<T>)`
   - `setStrategy(strategy: SortStrategy<T>): void`
   - `sort(data: T[], compare): T[]`
   - `getStrategyName(): string`
5. Продемонстрируйте сортировку чисел и строк разными стратегиями

## Чеклист

- [ ] Интерфейс `SortStrategy<T>` определён с дженериком
- [ ] Три стратегии реализуют интерфейс
- [ ] Исходный массив не мутируется (создаётся копия)
- [ ] `Sorter` позволяет менять стратегию через `setStrategy`
- [ ] Все стратегии дают одинаковый результат сортировки
- [ ] Демонстрация работает с числами и строками

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что все три стратегии выдают одинаковый отсортированный массив
3. Убедитесь, что сортировка строк также работает
