# Задание 13.1: Domain Layer

## Цель

Построить полноценный доменный слой с Value Objects, сущностями, доменными событиями и спецификациями для системы управления заказами.

## Требования

1. Создайте Value Objects: `Email` (нормализация, валидация) и `Money` (amount, currency, операция add, защита от negative)
2. Оба Value Object используют приватный конструктор и фабричный метод `create()`, возвращающий `Result<T>`
3. Определите `DomainEvent` как discriminated union с типами: OrderCreated, ItemAdded, OrderConfirmed, OrderCancelled
4. Реализуйте сущность `Order` (иммутабельный интерфейс) и чистые функции: `createOrder`, `addItem`, `confirmOrder`, `cancelOrder`
5. Каждая функция возвращает `Result<Order>` и добавляет соответствующее событие
6. Реализуйте Specification pattern с комбинаторами `and`, `or`, `not`
7. Создайте спецификации: `isDraft`, `isConfirmed`, `hasItems`, `totalOver(amount)`

## Чеклист

- [ ] Email.create нормализует в lowercase и валидирует формат
- [ ] Money.create отклоняет отрицательные суммы и невалидную валюту
- [ ] Money.add проверяет совпадение валют
- [ ] addItem в не-draft заказ возвращает ошибку
- [ ] confirmOrder пустого заказа возвращает ошибку
- [ ] Каждая операция добавляет DomainEvent
- [ ] Спецификации комбинируются: isDraft.and(hasItems)
- [ ] Все объекты иммутабельны (readonly)

## Как проверить себя

1. Создайте Email с разным регистром -- результат lowercase
2. Попробуйте создать Money с отрицательной суммой -- получите ошибку
3. Пройдите полный цикл заказа: создание -> добавление товаров -> подтверждение
4. Проверьте, что после подтверждения нельзя добавить товар
5. Убедитесь, что события записаны в правильном порядке
