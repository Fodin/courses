# Задание 6.2: services, image, build

## Цель

Разобраться в способах определения сервисов: использование готовых образов (image), сборка из Dockerfile (build), расширенная конфигурация сборки и container_name.

## Требования

1. Создайте компонент с переключаемыми примерами (минимум 5): image (готовый образ), build (простая сборка), build (расширенная конфигурация), image + build (комбинация), container_name
2. Каждый пример содержит YAML-код и текстовое описание
3. Добавьте сводную таблицу с колонками: опция, когда использовать, пример
4. Добавьте блок-совет по выбору между image и build

## Подсказки

- Массив объектов `{ key, label, yaml, desc }` для примеров
- Покажите разницу между `build: ./api` и расширенным `build:` с context, dockerfile, args, target
- Объясните зачем нужна комбинация `image + build`

## Чеклист

- [ ] Минимум 5 переключаемых примеров
- [ ] Примеры image: Docker Hub, приватный реестр, digest
- [ ] Примеры build: простой и расширенный (context, dockerfile, args, target)
- [ ] Пример image + build для тегирования
- [ ] Пример container_name с предупреждением
- [ ] Сводная таблица с рекомендациями
- [ ] Блок-совет по выбору

## Как проверить себя

1. Переключайте примеры -- каждый показывает разный YAML
2. Таблица должна помочь быстро выбрать нужную опцию
3. Предупреждение о container_name видно и понятно
