# Задание 6.1: Структура docker-compose.yml

## Цель

Разобраться в структуре файла docker-compose.yml: корневые секции, их назначение, устаревший ключ version, и собрать полный пример конфигурации.

## Требования

1. Создайте интерактивный компонент с тремя секциями: "Секции файла", "Полный пример", "version (deprecated)"
2. Секция "Секции файла" -- интерактивный список из 3 корневых ключей: `services` (обязательная), `volumes`, `networks`. Для каждого: описание и пример YAML-кода. Визуально выделите обязательную секцию
3. Секция "Полный пример" -- реалистичный docker-compose.yml для веб-приложения (web + api + db) с 3 сервисами, 2 сетями и 1 томом. Под примером покажите статистику (количество сервисов, сетей, томов)
4. Секция "version (deprecated)" -- сравнение старого формата (с version: '3.8') и нового (без version), объяснение почему version устарел
5. Используйте табы для переключения между секциями

## Подсказки

- Массив объектов `{ key, label, required, desc, example }` для секций
- Для полного примера используйте `<pre>` с подсветкой
- `useState<'sections' | 'full' | 'version'>` для переключения

## Чеклист

- [ ] Три секции переключаются по табам
- [ ] Все 3 корневые секции описаны с примерами YAML
- [ ] services визуально отмечена как обязательная
- [ ] Полный пример содержит services, networks, volumes
- [ ] Сравнение старого и нового формата (version)
- [ ] Объяснение почему version устарел

## Как проверить себя

1. Переключайте табы -- каждый показывает свой контент
2. В списке секций кликайте по каждой -- YAML-пример меняется
3. Полный пример должен быть синтаксически корректным YAML
4. Раздел version должен убедительно показать, что ключ больше не нужен
