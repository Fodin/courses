# Задание 11.2: XOR Type

## Цель

Реализовать тип `XOR<A, B>`, который обеспечивает взаимоисключающие union-типы: объект может иметь свойства либо A, либо B, но не обоих одновременно.

## Требования

1. Продемонстрируйте проблему: покажите, что обычный `A | B` позволяет оба набора свойств
2. Реализуйте `Without<T, U>` -- делает ключи из T (отсутствующие в U) опциональными с типом `never`
3. Реализуйте `XOR<T, U>` через комбинацию `Without` и intersection
4. Реализуйте `XOR3<A, B, C>` для трёх взаимоисключающих вариантов
5. Покажите практический пример: три способа оплаты (карта, банк, криптовалюта)
6. Реализуйте функцию `processPayment`, которая обрабатывает XOR-тип

## Чеклист

- [ ] Обычный union `CardPayment | BankTransfer` позволяет оба (продемонстрировано)
- [ ] `XOR<CardPayment, BankTransfer>` принимает только один набор
- [ ] Попытка передать оба набора свойств приводит к ошибке компиляции
- [ ] `XOR3` работает с тремя взаимоисключающими вариантами
- [ ] `processPayment` корректно обрабатывает все варианты через type narrowing

## Как проверить себя

1. Попробуйте присвоить объект с полями из обоих типов -- должна быть ошибка
2. Убедитесь, что TypeScript корректно сужает тип внутри if-проверок
3. Проверьте, что каждый вариант XOR принимается по отдельности
