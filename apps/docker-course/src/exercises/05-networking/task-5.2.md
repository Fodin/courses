# Задание 5.2: Связь между контейнерами

## Цель

Понять, как контейнеры общаются внутри Docker-сети: DNS-резолвинг по имени, сетевая изоляция между разными сетями, использование сетевых алиасов.

## Требования

1. Создайте компонент с тремя секциями: "DNS-резолвинг", "Сетевая изоляция", "Алиасы"
2. Секция "DNS-резолвинг" -- пошаговая демонстрация (5 шагов): создание сети, запуск двух контейнеров, ping по имени, подключение к БД по DNS. Кнопки "Шаг 1"..."Шаг 5", при нажатии показывается команда и её вывод
3. Секция "Сетевая изоляция" -- демонстрация архитектуры с двумя сетями (frontend, backend) и контейнером-мостом (api в обеих). Покажите список команд для создания такой конфигурации. Визуализация: блоки-карточки показывающие какие связи работают (web->api, api->db) и какие нет (web->db)
4. Секция "Алиасы" -- пример `--network-alias` с объяснением. Список преимуществ: абстракция, миграция, балансировка
5. Используйте табы для навигации между секциями

## Подсказки

- Для пошаговой демонстрации: массив `{ step, cmd, output, desc }`, `useState` для активного шага
- Для изоляции: CSS Grid для карточек связей (зелёные -- работают, красные -- нет)
- Для алиасов: `<pre>` блок с командами, `<ul>` для списка преимуществ

## Чеклист

- [ ] Три секции переключаются по табам
- [ ] Пошаговая демонстрация DNS с 5 шагами, каждый с командой и выводом
- [ ] Визуализация сетевой изоляции: работающие и заблокированные связи
- [ ] Пример создания multi-network архитектуры с командами
- [ ] Пример использования --network-alias с объяснением
- [ ] Список преимуществ алиасов (минимум 3)

## Как проверить себя

1. Переключайте шаги DNS-демонстрации -- команда и вывод должны меняться
2. В секции изоляции визуально понятно, какие связи работают, а какие нет
3. Пример алиасов синтаксически корректен
4. Все команды Docker реальны и исполнимы
