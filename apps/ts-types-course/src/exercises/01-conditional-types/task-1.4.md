# Задание 1.4: Conditional Types with Generics

## Цель

Научиться комбинировать условные типы с generics для создания продвинутых утилитных типов: извлечение ключей по типу, условные возвращаемые типы функций, частичное обязательство.

## Требования

1. Реализуйте тип `Flatten<T>`, рекурсивно разворачивающий Promise и Array до базового типа
2. Реализуйте типы `OptionalKeys<T>` и `RequiredKeys<T>`, извлекающие опциональные и обязательные ключи объектного типа
3. Реализуйте типы `FunctionKeys<T>` и `NonFunctionKeys<T>`, разделяющие ключи объекта по типу значения (функция / не функция)
4. Реализуйте generic-функцию `processValue` с conditional return type: для строк возвращает `string[]` (split), для чисел -- `number` (удвоение)
5. Реализуйте утилитный тип `MakeRequired<T, K>`, делающий указанные ключи обязательными, оставляя остальные без изменений

## Чеклист

- [ ] `Flatten<Promise<string[]>>` разрешается в `string`
- [ ] `OptionalKeys<{ a: string; b?: number }>` разрешается в `'b'`
- [ ] `RequiredKeys<{ a: string; b?: number }>` разрешается в `'a'`
- [ ] `FunctionKeys` корректно извлекает только ключи с функциональными значениями
- [ ] `processValue("hello")` возвращает `string[]`, `processValue(42)` возвращает `number`
- [ ] `MakeRequired<T, K>` делает указанные ключи обязательными

## Как проверить себя

1. Проверьте `OptionalKeys` и `RequiredKeys` на интерфейсе со смешанными ключами
2. Убедитесь, что `processValue` имеет разные типы возврата для string и number
3. Попробуйте создать объект типа `MakeRequired<Form, 'name'>` без поля `name` -- ошибка
