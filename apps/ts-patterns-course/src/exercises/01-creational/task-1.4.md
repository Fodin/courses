# Задание 1.4: Singleton

## Цель

Реализовать паттерн Singleton для менеджера конфигурации с типобезопасным доступом через generics.

## Требования

1. Создайте класс `ConfigManager` с:
   - `private constructor()` — запрет прямого создания
   - `static getInstance(): ConfigManager` — получение единственного экземпляра
   - `get<T>(key: string): T | undefined` — получение значения
   - `set<T>(key: string, value: T): void` — установка значения
   - `has(key: string): boolean` — проверка наличия ключа
   - `getAll(): Record<string, unknown>` — получение всех значений
   - `clear(): void` — очистка конфигурации
2. Внутреннее хранилище — `Map<string, unknown>`
3. `getInstance()` создаёт экземпляр при первом вызове, далее возвращает тот же
4. Добавьте `static resetInstance()` для сброса (полезно для тестов)

## Чеклист

- [ ] Конструктор приватный — `new ConfigManager()` невозможен
- [ ] `getInstance()` всегда возвращает один и тот же экземпляр
- [ ] `get/set` типизированы через generics
- [ ] `has`, `getAll`, `clear` работают корректно
- [ ] Изменения через один «экземпляр» видны через другой
- [ ] Демонстрация доказывает идентичность экземпляров

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что `config1 === config2` возвращает `true`
3. Убедитесь, что `config2.get("theme")` видит значение, установленное через `config1`
4. Убедитесь, что `new ConfigManager()` вызывает ошибку TypeScript
