# Задание 2.1: Step Builder

## 🎯 Цель

Реализовать Step Builder для HTTP-запросов, который на уровне типов принуждает к правильному порядку вызова методов: выбор метода -> тело (для POST/PUT) -> настройка -> сборка.

## Требования

1. Создайте 3 интерфейса-шага, каждый из которых предоставляет только допустимые методы
2. Шаг 1: выбор HTTP-метода (get, post, put, delete) — каждый ведёт к соответствующему шагу
3. Шаг 2 (только для POST/PUT): установка тела запрос��
4. Шаг 3: настройка заголовков, таймаута и вызов build()
5. GET и DELETE должны пропускать шаг body и сразу переходить к настройке
6. Реализуйте фабричную функцию `createRequestBuilder()` с правильными переходами

## Чеклист

- [ ] `createRequestBuilder().get("/url").body({})` — ошибка компиляции
- [ ] `createRequestBuilder().post("/url").build()` — ошибка (нужен body)
- [ ] `createRequestBuilder().build()` — ошибка (нужен метод)
- [ ] Цепочка header().header().timeout().build() работает
- [ ] build() возвращает полный объект HttpRequest

## Как проверить себя

Попробуйте все невалидные цепочки вызовов — TypeScript должен показать ошибки. Каждый GET/DELETE должен перескакивать через body-шаг.
