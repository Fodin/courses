# Задание 7.5: Оптимистичные обновления

## 🎯 Цель

Реализовать паттерн оптимистичного обновления — изменить UI немедленно и откатить при ошибке сервера.

## 📋 Требования

Создайте стор `OptimisticTodoStore` со списком задач и методом toggle:

1. Объявите observable-поля:
   - `todos: OptTodo[]` — список задач (3 начальных элемента)
   - `lastError: string | null` — последнее сообщение об ошибке
2. Реализуйте `async toggle(id: string)`:
   - Найдите todo по `id`
   - **Сохраните** предыдущее значение `completed` в переменную `prevState`
   - **Оптимистично** обновите: переключите `completed`, поставьте `pending = true`, сбросьте `lastError`
   - **Отправьте** запрос на сервер (`mockApi.toggleTodo`)
   - **При успехе** — снимите `pending = false`
   - **При ошибке** — откатите `completed = prevState`, снимите `pending`, запишите ошибку в `lastError`
   - Все изменения после `await` обёрнуты в `runInAction`
3. В компоненте:
   - Отображайте чекбоксы с задачами
   - Задачи с `pending = true` — полупрозрачные и заблокированные
   - Покажите сообщение об ошибке при неудаче

```typescript
interface OptTodo {
  id: string
  title: string
  completed: boolean
  pending?: boolean
}
```

## Чеклист

- [ ] `OptimisticTodoStore` с `makeAutoObservable`
- [ ] `toggle` сохраняет `prevState` перед изменением
- [ ] UI обновляется мгновенно (до ответа сервера)
- [ ] При ошибке `completed` откатывается к `prevState`
- [ ] `pending` корректно устанавливается и снимается
- [ ] Сообщение об ошибке отображается в UI
- [ ] Все post-await мутации обёрнуты в `runInAction`

## 🔍 Как проверить себя

1. Кликните чекбокс — он переключится мгновенно, станет полупрозрачным (pending)
2. Через ~0.5 секунды pending снимется
3. Так как mock API имеет 30% шанс ошибки, повторяйте клики:
   - При успехе — чекбокс остаётся в новом состоянии
   - При ошибке — чекбокс откатывается назад, появляется сообщение «rolled back»
4. Повторите несколько раз, чтобы увидеть оба сценария
