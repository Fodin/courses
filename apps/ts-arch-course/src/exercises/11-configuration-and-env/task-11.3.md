# Задание 11.3: Environment Types

## Цель

Реализовать типобезопасную систему загрузки переменных окружения с декларативной спецификацией, поддержкой enum-ограничений и автоматическим выводом типов.

## Требования

1. Создайте тип `EnvVarDef<T>` с полями: `key`, `transform`, `fallback`, `required`
2. Реализуйте фабричные функции: `envString`, `envNumber`, `envBoolean`, `envEnum`
3. `envEnum` должен ограничивать допустимые значения через литеральные типы (as const)
4. Создайте тип `InferEnv<S>`, который выводит итоговый тип из спецификации
5. Реализуйте `loadEnv(spec, env)`, возвращающий `{ ok, value }` или `{ ok, errors }`
6. Продемонстрируйте: полную загрузку, fallback-значения, ошибки валидации, enum-ограничения

## Чеклист

- [ ] `envNumber` преобразует строку в число и выдаёт ошибку для невалидных значений
- [ ] `envBoolean` принимает 'true'/'false'/'1'/'0'
- [ ] `envEnum` ограничивает допустимые значения: 'test' для NODE_ENV -- ошибка
- [ ] `InferEnv` выводит `'development' | 'staging' | 'production'` для NODE_ENV
- [ ] Fallback-значения применяются для отсутствующих переменных
- [ ] Отсутствие required-переменной без fallback -- ошибка
- [ ] Все ошибки собираются в один массив

## Как проверить себя

1. Загрузите полный набор переменных -- все типы должны быть корректными
2. Загрузите минимальный набор -- fallback-значения должны подставиться
3. Передайте невалидные значения enum -- получите ошибки
4. Проверьте, что `result.value.port` -- число, а `result.value.nodeEnv` -- union литералов
