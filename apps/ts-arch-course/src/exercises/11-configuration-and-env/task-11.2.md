# Задание 11.2: Feature Flags

## Цель

Создать типобезопасную систему Feature Flags с типизированным реестром флагов, поддержкой разных типов значений и механизмом переопределений.

## Требования

1. Определите `FlagDefinition<T>` с полями: `description`, `defaultValue`, `type`
2. Создайте тип `FlagValues<R>`, который выводит конкретный тип значения для каждого флага
3. Реализуйте `createFeatureFlags(registry, overrides?)` возвращающий `FeatureFlagService`
4. Сервис должен поддерживать методы: `isEnabled`, `getValue`, `getAll`, `override`, `reset`, `resetAll`
5. `getValue` должен возвращать конкретный тип: `getValue('MAX_UPLOAD_SIZE')` -> `number`
6. Продемонстрируйте: значения по умолчанию, проверку `isEnabled`, переопределения и сброс

## Чеклист

- [ ] `FlagDefinition` поддерживает boolean, number и string типы
- [ ] `FlagValues` выводит правильный тип для каждого флага
- [ ] `getValue('DARK_MODE')` возвращает `boolean`, а не `string | number | boolean`
- [ ] `isEnabled` работает для всех типов (boolean, truthy/falsy для number/string)
- [ ] `override` принимает только значение правильного типа для данного флага
- [ ] `resetAll` возвращает все флаги к значениям по умолчанию
- [ ] Начальные overrides при создании сервиса работают корректно

## Как проверить себя

1. Получите все значения по умолчанию -- типы должны совпадать с определениями
2. Переопределите несколько флагов и проверьте, что старые значения не потеряны
3. Сбросьте один флаг -- он вернётся к default, остальные останутся
4. Используйте `resetAll` -- все значения вернутся к defaults
