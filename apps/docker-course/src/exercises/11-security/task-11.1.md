# Задание 11.1: Пользователи и Capabilities

## Цель

Научиться запускать контейнеры от непривилегированного пользователя, понимать систему Linux Capabilities, правильно использовать `--cap-drop`/`--cap-add` и знать, почему `--privileged` опасен. Создать интерактивный компонент-справочник по этим темам.

## Требования

1. Создайте компонент с тремя переключаемыми секциями: "Пользователи", "Capabilities", "Seccomp и AppArmor"
2. Секция "Пользователи" показывает таблицу с примерами настройки USER в Dockerfile: плохой вариант (root), хороший вариант (custom user), и для популярных образов (node, postgres, nginx) -- с описанием и командами
3. Для каждого примера добавьте кнопку "Показать Dockerfile" -- по клику отображается блок кода
4. Секция "Capabilities" показывает таблицу Linux capabilities (минимум 8 штук) с именем, описанием, уровнем риска (высокий/средний/низкий) и рекомендацией (drop/keep). Добавьте визуальную индикацию уровня риска цветом
5. Внизу секции Capabilities -- блок "Минимальные наборы" для типичных сервисов (web-сервер, Node.js, БД) с перечислением нужных capabilities
6. Секция "Seccomp и AppArmor" показывает описание каждой технологии, что блокируется по умолчанию, и примеры команд запуска с профилями
7. Внизу компонента -- блок "Ключевой принцип" с описанием принципа наименьших привилегий

## Подсказки

- `useState<string>` для переключения активной секции
- `useState<Record<string, boolean>>` для показа/скрытия блоков кода
- Для уровня риска используйте цвета: красный (высокий), оранжевый (средний), зелёный (низкий)
- Массив объектов для capabilities: `{ name, description, risk, recommendation }`

## Чеклист

- [ ] Три секции переключаются корректно
- [ ] Таблица пользователей с примерами (минимум 5 вариантов)
- [ ] Кнопки "Показать Dockerfile" работают независимо
- [ ] Таблица Capabilities (минимум 8 штук) с цветной индикацией риска
- [ ] Минимальные наборы для 3 типов сервисов
- [ ] Описание Seccomp и AppArmor с примерами команд
- [ ] Блок "Ключевой принцип" внизу

## Как проверить себя

1. Переключайте секции -- содержимое корректно меняется
2. В "Пользователи" нажимайте кнопки -- Dockerfile-примеры появляются/скрываются независимо
3. В "Capabilities" видны минимум 8 записей с цветной индикацией риска
4. Цвета соответствуют уровням: красный = высокий, оранжевый = средний, зелёный = низкий
5. Блоки минимальных наборов присутствуют для 3 типов сервисов
6. Seccomp и AppArmor описаны с примерами команд
7. Блок "Ключевой принцип" виден внизу
