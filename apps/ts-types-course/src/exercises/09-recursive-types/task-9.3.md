# Задание 9.3: Recursive String Types

## Цель

Научиться парсить и трансформировать строковые литеральные типы с помощью рекурсии: Split, Join, ReplaceAll, CamelCase, KebabCase, Trim, StringLength, PathKeys.

## Требования

1. Реализуйте `Split<S, D>` — разбивает строку по разделителю в tuple (обработайте пустую строку)
2. Реализуйте `Join<T, D>` — объединяет string tuple в строку через разделитель
3. Реализуйте `ReplaceAll<S, From, To>` — рекурсивно заменяет все вхождения подстроки
4. Реализуйте `CamelCase<S>` — преобразует `snake_case` в `camelCase`
5. Реализуйте `KebabCase<S>` — преобразует `camelCase` в `kebab-case`
6. Реализуйте `Trim<S>` через `TrimLeft` и `TrimRight` — убирает пробелы по краям
7. Реализуйте `StringLength<S>` — вычисляет длину строки через рекурсию с аккумулятором
8. Реализуйте `PathKeys<T>` — генерирует все dot-notation пути для вложенного объекта

## Чеклист

- [ ] `Split<'a.b.c', '.'>` возвращает `['a', 'b', 'c']`
- [ ] `Join<['a', 'b'], '-'>` возвращает `'a-b'`
- [ ] `ReplaceAll` заменяет все вхождения, не только первое
- [ ] `CamelCase<'user_first_name'>` = `'userFirstName'`
- [ ] `KebabCase<'helloWorld'>` = `'hello-world'`
- [ ] `Trim<'  hello  '>` = `'hello'`
- [ ] `StringLength<'hello'>` = `5`
- [ ] `PathKeys` генерирует все пути включая вложенные
- [ ] Результаты отображаются на странице

## Как проверить себя

1. Нажмите кнопку «Запустить»
2. Убедитесь, что все строковые трансформации корректны
3. Проверьте, что `PathKeys<Config>` включает вложенные пути типа `'server.host'`
4. Убедитесь, что `Split<'', '.'>` возвращает пустой массив, а не `['']`
