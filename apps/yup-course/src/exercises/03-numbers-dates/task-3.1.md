# Задание 3.1: min, max, positive, negative

## Цель

Научиться использовать числовые ограничения Yup: `min()`, `max()`, `positive()`, `negative()`.

## Требования

1. Создайте `ageSchema` — число, обязательное, положительное, от 18 до 120

2. Создайте `temperatureSchema` — число, обязательное, от -50 до 50

3. Создайте `balanceSchema` — число, обязательное, отрицательное (для моделирования долга)

4. Реализуйте валидацию каждой схемы по клику на кнопку

5. При успехе покажите результат в зелёном блоке, при ошибке — в красном

## Чеклист

- [ ] `ageSchema` использует `positive()`, `min(18)`, `max(120)`
- [ ] `temperatureSchema` использует `min(-50)`, `max(50)` (отрицательные числа разрешены)
- [ ] `balanceSchema` использует `negative()`
- [ ] Все схемы имеют кастомные сообщения об ошибках
- [ ] Пустое поле обрабатывается как `undefined`

## Как проверить себя

1. Age: 25 — успех
2. Age: 0 — ошибка (не положительное)
3. Age: 15 — ошибка (меньше 18)
4. Age: 150 — ошибка (больше 120)
5. Temperature: -30 — успех
6. Temperature: -60 — ошибка
7. Balance: -100 — успех
8. Balance: 50 — ошибка (не отрицательное)
9. Balance: 0 — ошибка (ноль не отрицательный)
