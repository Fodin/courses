# Задание 8.1: Interface Merging

## Цель

Научиться использовать слияние интерфейсов (interface merging) для расширения типов, понять правила слияния свойств и методов, порядок перегрузок.

## Требования

1. Создайте интерфейс `User` с полями `id` и `name`, затем расширьте его вторым объявлением с полями `email` и `role`
2. Продемонстрируйте слияние методов: создайте `Logger` с перегрузками `log(message)` и `log(message, level)`
3. Покажите порядок перегрузок при слиянии: строковые литералы поднимаются наверх, поздние блоки приоритетнее
4. Продемонстрируйте конфликт типов при слиянии: одинаковое свойство должно иметь одинаковый тип
5. Реализуйте слияние generic интерфейсов `Container<T>` с методами `getValue()`, `setValue()`, `isEmpty()`
6. Покажите слияние class + interface (класс с дополнительными свойствами из interface)
7. Объясните практическое использование: расширение Window

## Чеклист

- [ ] Два объявления `User` сливаются в один интерфейс с 4 полями
- [ ] Перегрузки `log` создаются через слияние двух блоков `Logger`
- [ ] Правила порядка перегрузок объяснены (литералы, приоритет блоков)
- [ ] Конфликт типов при слиянии продемонстрирован
- [ ] Generic `Container<T>` работает с обоими наборами методов
- [ ] Class + interface merging показан
- [ ] Результаты отображаются на странице

## Как проверить себя

1. Нажмите кнопку «Запустить»
2. Убедитесь, что объект `User` содержит все 4 поля из обоих объявлений
3. Проверьте, что `Logger` вызывается с 1 и 2 аргументами
4. Убедитесь, что `Container<string>` имеет методы из обоих объявлений
