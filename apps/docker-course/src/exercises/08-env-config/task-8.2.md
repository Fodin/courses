# Задание 8.2: Secrets и configs

## Цель

Изучить механизмы безопасной передачи конфиденциальных данных (Docker Secrets) и конфигурационных файлов (Docker Configs) в контейнеры. Понять, почему переменные окружения небезопасны для секретов и когда использовать каждый подход.

## Требования

1. Создайте компонент с четырьмя табами: "Проблема", "Secrets", "Configs", "Сравнение"
2. Таб "Проблема" -- три карточки с проблемами переменных окружения: видимость через docker inspect, /proc и логи. Внизу блок с решением (Docker Secrets)
3. Таб "Secrets" -- три переключаемых примера: "Secrets в Compose" (YAML с секцией secrets), "Чтение в приложении" (Node.js код с fallback на env), "Создание файлов" (bash-команды). Плюс информация о суффиксе `_FILE` в официальных образах
4. Таб "Configs" -- два переключаемых примера (nginx, prometheus). Плюс примечание о configs vs bind mount
5. Таб "Сравнение" -- таблица сравнения environment / Secrets / Configs по 7 критериям

## Подсказки

- `useState<'problem' | 'secrets' | 'configs' | 'comparison'>` для табов
- `useState<string>` для активного примера внутри таба
- Карточки проблем: `{ cmd, desc }[]`
- Таблица сравнения: массив массивов строк (строки таблицы)

1. Переключайте табы -- контент меняется
2. В "Проблема" убедитесь, что есть конкретные команды (docker inspect, cat /proc, console.log)
3. В "Secrets" переключайте примеры -- YAML, код Node.js, bash-команды
4. В "Configs" переключайте nginx/prometheus -- разные YAML
5. В "Сравнение" таблица читаема, 7 строк заполнены
