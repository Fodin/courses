# Задание 3.3: Валидация дат

## Цель

Научиться валидировать даты с помощью `yup.date()` и ограничений `min()` / `max()`.

## Требования

1. Создайте `birthdaySchema` — дата, обязательная, не в будущем (`max(new Date())`), не раньше 1900 года

2. Создайте `eventSchema` — дата, обязательная, только в будущем (`min(new Date())`)

3. Используйте `<input type="date">` для ввода дат

4. Отобразите валидированную дату в читаемом формате при успехе

## Чеклист

- [ ] `birthdaySchema` ограничена диапазоном 1900-01-01 до сегодня
- [ ] `eventSchema` принимает только будущие даты
- [ ] Пустое поле обрабатывается как `undefined` (срабатывает `required`)
- [ ] Результат отображается в цветном блоке

## Как проверить себя

1. Birthday: 2000-05-15 — успех (показывает дату)
2. Birthday: 2099-01-01 — ошибка (в будущем)
3. Birthday: пустое — ошибка "required"
4. Event: дата через месяц — успех
5. Event: вчерашняя дата — ошибка (в прошлом)
