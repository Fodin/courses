# Задание 3.3: Scoped-зависимости

## 🎯 Цель

Реализовать DI-контейнер с тремя lifetime-стратегиями: singleton, transient и scoped, с поддержкой дочерних scope.

## Требования

1. Создайте класс `ScopedContainer` с методами регистрации для каждого lifetime
2. `registerSingleton()` — один экземпляр на всё приложение
3. `registerTransient()` — новый экземпляр при каждом resolve
4. `registerScoped()` — один экземпляр в пределах scope
5. `createScope()` — создание дочернего scope
6. Singleton-ы должны разделяться между scope-ами (храниться в root)
7. Продемонстрируйте все три стратегии с числовыми ID для отслеживания

## Чеклист

- [ ] Singleton: один и тот же экземпляр при повторных resolve
- [ ] Transient: разные экземпляры при каждом resolve
- [ ] Scoped: одинаковый экземпляр внутри scope, разные между scope-ами
- [ ] Singleton разделяется между root и дочерними scope-ами
- [ ] createScope() наследует регистрации родителя

## Как проверить себя

Создайте 2 scope и проверьте: scoped-сервис одинаков внутри одного scope, но различается между scope-ами. Singleton одинаков везде. Transient всегда новый.
