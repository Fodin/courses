# Задание 8.3: Шаблоны конфигурации

## Цель

Научиться организовывать конфигурацию Docker-проекта для нескольких окружений (development, staging, production) с использованием базового compose-файла, override-файлов, .env для каждого окружения и YAML-якорей для DRY.

## Требования

1. Создайте компонент с четырьмя секциями: "Структура проекта", "Базовый compose", "Окружения", "YAML-якоря"
2. Секция "Структура проекта" -- дерево файлов проекта (compose-файлы, .env-файлы, secrets/) с блоком о принципах 12-factor app
3. Секция "Базовый compose" -- YAML базовой конфигурации с `${VAR:-default}` для безопасных дефолтов
4. Секция "Окружения" -- три переключаемых окружения (Dev / Staging / Prod), для каждого в двухколоночном layout: compose override слева, .env файл справа. Под ними -- команда запуска
5. Для Production покажите использование Docker Secrets вместо переменных окружения
6. Секция "YAML-якоря" -- пример с `x-` расширениями, `&name` / `*name` / `<<: *name` и объяснение синтаксиса
7. Внизу компонента совет про `docker compose config`

## Подсказки

- `useState<'structure' | 'base' | 'env' | 'anchors'>` для секций
- `useState<'dev' | 'staging' | 'prod'>` для окружений
- Объект `Record<EnvType, { label, color, compose, envFile, runCmd }>` для данных окружений
- `gridTemplateColumns: '1fr 1fr'` для двухколоночного layout
- Цвета: dev=#4caf50, staging=#ff9800, prod=#f44336

## Чеклист

- [ ] Четыре секции переключаются
- [ ] Структура проекта: дерево файлов + принципы 12-factor
- [ ] Базовый compose: YAML с ${VAR:-default}
- [ ] Три окружения (dev/staging/prod) с переключением
- [ ] Для каждого окружения: compose override + .env файл + команда запуска
- [ ] Production использует Docker Secrets
- [ ] YAML-якоря: пример + объяснение синтаксиса &, *, <<
- [ ] Совет про docker compose config внизу

## Как проверить себя

1. Переключайте секции -- контент меняется
2. В "Структура проекта" дерево файлов читаемо, принципы 12-factor перечислены
3. В "Окружения" переключайте dev/staging/prod -- compose и .env различаются
4. Production показывает secrets вместо обычных переменных
5. Команды запуска различаются для каждого окружения
6. В "YAML-якоря" объяснены &, *, <<: *
