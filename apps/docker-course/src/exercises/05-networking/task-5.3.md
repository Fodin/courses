# Задание 5.3: Проброс портов

## Цель

Освоить механизм проброса портов (port mapping) в Docker: синтаксис `-p`, привязка к интерфейсам, протоколы, различие между `EXPOSE` и `-p`.

## Требования

1. Создайте компонент с таблицей синтаксиса и интерактивными примерами
2. Таблица синтаксиса -- все варианты `-p`: `hostPort:containerPort`, `ip:hostPort:containerPort`, `containerPort` (случайный), `-P`, `/udp`. Минимум 5 строк
3. Интерактивные примеры -- минимум 5 вариантов проброса: базовый, только localhost, случайный порт, UDP, несколько портов. Кнопки для переключения, при выборе показывается команда и описание
4. Кнопка "Показать: EXPOSE vs -p" -- при нажатии раскрывается блок с двумя карточками: EXPOSE в Dockerfile (только документация) и -p при docker run (реальная публикация)
5. Предупреждение о безопасности: `-p 8080:80` открывает порт на всех интерфейсах, используйте `127.0.0.1:...` для ограничения

## Подсказки

- Для таблицы синтаксиса: HTML `<table>` с заголовками "Синтаксис" и "Значение"
- Для примеров: массив `{ title, cmd, desc }`, `useState` для выбранного примера
- Для EXPOSE vs -p: `useState<boolean>` для toggle, CSS Grid для двух карточек
- Для предупреждения: блок с фоном `#fce4ec`

## Чеклист

- [ ] Таблица с минимум 5 вариантами синтаксиса -p
- [ ] Минимум 5 интерактивных примеров с переключением
- [ ] Каждый пример -- команда + описание
- [ ] Toggle-блок "EXPOSE vs -p" с двумя карточками
- [ ] Предупреждение о безопасности (привязка к 0.0.0.0)
- [ ] Рекомендация использовать 127.0.0.1

## Как проверить себя

1. Все варианты синтаксиса в таблице корректны
2. Переключение примеров работает, команда и описание обновляются
3. Кнопка EXPOSE vs -p показывает/скрывает блок
4. Блок безопасности хорошо заметен
