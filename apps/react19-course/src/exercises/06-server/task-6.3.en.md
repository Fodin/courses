# Task 6.3: Server Actions

## Objective

Understand the concept of Server Actions — server functions callable from client code.

## Requirements

1. Create a Server Action simulation (async function with a delay)
2. Show how a Server Action is used in a form `action`
3. Show how a Server Action is called directly
4. Explain the `'use server'` directive

## Checklist

- [ ] A form with `action` uses a server function
- [ ] Direct Server Action call via button
- [ ] Code with the `'use server'` directive is shown
- [ ] Data serialization is explained

## How to verify

1. Submit the form — data is processed "on the server"
2. Click the direct call button — the Server Action executes
