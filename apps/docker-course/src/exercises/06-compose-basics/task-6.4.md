# Задание 6.4: docker compose up/down/logs

## Цель

Изучить основные команды Docker Compose для управления жизненным циклом приложения: запуск, остановка, логи, статус, выполнение команд.

## Требования

1. Создайте компонент с тремя группами команд: "Жизненный цикл" (up, down), "Логи", "Статус и управление" (ps, exec, restart, build)
2. В каждой группе -- список команд, кликабельных для просмотра. При клике показывается: команда, типичный вывод, описание
3. Группа "Жизненный цикл": `up -d`, `up -d --build`, `down`, `down -v`. Вывод должен быть реалистичным
4. Группа "Логи": `logs`, `logs -f <service>`, `logs --tail 20 -t`
5. Группа "Статус и управление": `ps`, `exec`, `restart`, `build --no-cache`
6. Блок сравнения stop vs down с визуальным выделением
7. Предупреждение о `down -v` (потеря данных)
8. Блок "Типичный рабочий цикл" -- последовательность команд для разработки

## Подсказки

- Массив объектов `{ key, label, commands: [{ cmd, output, desc }] }` для групп
- При смене группы сбрасывайте activeCmd на 0
- Для вывода используйте реалистичный формат Docker Compose V2

## Чеклист

- [ ] Три группы команд переключаются по табам
- [ ] В каждой группе кликабельный список команд
- [ ] Реалистичный вывод для каждой команды
- [ ] Сравнение stop vs down
- [ ] Предупреждение о down -v
- [ ] Блок "Типичный рабочий цикл"

## Как проверить себя

1. Переключайте группы -- список команд обновляется
2. Кликайте по командам -- вывод меняется
3. Вывод выглядит как реальный Docker Compose
4. Предупреждение о down -v визуально заметно
