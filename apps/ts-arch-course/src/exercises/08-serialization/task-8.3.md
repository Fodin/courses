# Задание 8.3: Data Migrations

## 🎯 Цель

Создать типобезопасный пайплайн миграций данных с версионированными схемами, где каждая миграция имеет типизированные вход и выход.

## Требования

1. Создайте интерфейс `Migration<TFrom, TTo>` с полями `fromVersion`, `toVersion`, `migrate`
2. Создайте интерфейс `VersionedData<T>` с полями `version` и `data`
3. Определите минимум 4 версии схемы (V1 -> V2 -> V3 -> V4) с реалистичной эволюцией:
   - V1->V2: разделение поля на несколько
   - V2->V3: изменение структуры (примитив -> объект)
   - V3->V4: добавление новых полей с дефолтными значениями
4. Реализуйте `MigrationPipeline` с методами `register(migration)` и `migrate(data, targetVersion)`
5. Pipeline должен автоматически применять промежуточные миграции

## Чеклист

- [ ] `Migration<TFrom, TTo>` типизирует функцию `migrate: (data: TFrom) => TTo`
- [ ] `VersionedData<T>` хранит версию и типизированные данные
- [ ] Каждая миграция создаёт новый объект (иммутабельность)
- [ ] Pipeline применяет V1->V2->V3->V4 при запросе миграции V1->V4
- [ ] Pipeline корректно работает, если данные уже на промежуточной версии
- [ ] Все промежуточные трансформации проверяются компилятором

## Как проверить себя

Создайте данные V1 и мигрируйте до V4. Проверьте, что все поля корректно трансформированы. Попробуйте мигрировать данные V3 до V4 — должна примениться только одна миграция.
