# Задание 2.3: Facade

## Цель

Реализовать паттерн Facade — создать `OrderFacade`, который упрощает работу с тремя подсистемами: `InventoryService`, `PaymentService`, `ShippingService`.

## Требования

1. Используйте предоставленные подсистемы:
   - `InventoryService` — метод `check(productId): { available, quantity }` и `reserve(productId): boolean`
   - `PaymentService` — метод `charge(userId, amount): { success, transactionId?, error? }`
   - `ShippingService` — метод `createShipment(orderId, address): { trackingId }`
2. Определите интерфейсы:
   - `OrderRequest` с полями: `orderId`, `productId`, `userId`, `amount`, `address`
   - `OrderResult` с полями: `success`, `orderId?`, `error?`, `logs`
3. Создайте класс `OrderFacade`:
   - Принимает три сервиса через конструктор
   - Метод `placeOrder(order: OrderRequest): OrderResult` координирует:
     - Шаг 1: Проверка наличия товара
     - Шаг 2: Списание оплаты
     - Шаг 3: Резервирование товара
     - Шаг 4: Создание доставки
   - При ошибке на любом шаге — возвращает `{ success: false }` с описанием
   - Каждый шаг логируется в массив `logs`
4. Продемонстрируйте три сценария: успех, товар не в наличии, недостаточно средств

## Чеклист

- [ ] Интерфейсы `OrderRequest` и `OrderResult` определены
- [ ] `OrderFacade` принимает три сервиса через конструктор
- [ ] Метод `placeOrder` координирует все четыре шага
- [ ] При ошибке на любом шаге фасад останавливается и возвращает ошибку
- [ ] Каждый шаг записывает лог в массив `logs`
- [ ] Демонстрация показывает три сценария

## Как проверить себя

1. Нажмите кнопку запуска
2. Заказ с `PROD-1` и `user-1` (достаточно средств) — успешно проходит все 4 шага
3. Заказ с `PROD-2` (qty=0) — останавливается на шаге проверки наличия
4. Заказ с `user-2` (баланс 50) и суммой 200 — останавливается на шаге оплаты
5. Логи отражают каждый выполненный шаг
