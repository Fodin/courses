# Задание 12.2: Container Registry и тегирование

## Цель

Создать интерактивный компонент-справочник по Container Registries и стратегиям тегирования Docker-образов. Научиться выбирать подходящий registry, правильно тегировать образы и автоматизировать этот процесс.

## Требования

1. Создайте компонент с тремя переключаемыми секциями: "Container Registries", "Стратегии тегов", "Автоматизация"
2. Секция "Registries": таблица из 6 реестров (Docker Hub, GHCR, ECR, ACR, GCR, Harbor) с провайдером и бесплатным планом
3. Кнопка "Подробнее" для каждого registry раскрывает URL, особенности (список) и кнопку "Показать auth" с командой авторизации
4. Секция "Стратегии тегов": 5 стратегий (Semver, Git SHA, Branch, Timestamp, Build Number) — каждая с описанием, примерами тегов и цветной индикацией по назначению
5. Предупреждение о том, что latest — плохая практика для production
6. Секция "Автоматизация": поле ввода для версии + кнопка "Генерировать" — показывает список тегов, которые будут созданы
7. Пример конфигурации docker/metadata-action для GitHub Actions

## Подсказки

- `useState<'registries' | 'tags' | 'automation'>` для секций
- `useState<number | null>` для выбранного registry
- `useState<Record<number, boolean>>` для показа/скрытия auth команд
- Для симулятора: парсите ввод regex `/^v?\d+\.\d+\.\d+$/` и генерируйте теги
- Для цветной индикации стратегий используйте `borderLeft` с цветом

## Чеклист

- [ ] Три секции переключаются корректно
- [ ] Таблица из 6 registries с кнопкой "Подробнее"
- [ ] Детали registry: URL, особенности, auth-команда
- [ ] 5 стратегий тегирования с примерами и цветной индикацией
- [ ] Предупреждение о latest
- [ ] Симулятор генерации тегов работает
- [ ] Пример metadata-action

## Как проверить себя

1. Переключайте секции — содержимое корректно меняется
2. В "Registries" нажимайте "Подробнее" — появляются URL, features и кнопка auth
3. Нажмите "Показать auth" — команда авторизации отображается
4. В "Стратегии тегов" видны 5 стратегий с цветной левой границей и примерами
5. Предупреждение о latest видно внизу секции
6. В "Автоматизация" введите "v1.2.3" и нажмите "Генерировать" — появляются теги (semver + sha + branch + date)
7. Пример metadata-action отображается с YAML-конфигурацией
