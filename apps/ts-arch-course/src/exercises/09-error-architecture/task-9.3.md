# Задание 9.3: Error Propagation

## 🎯 Цель

Реализовать цепочку Result с методами `map`, `flatMap`, `mapError`, `unwrapOr`, `match` для типобезопасного распространения ошибок через слои приложения.

## Требования

1. Создайте тип `ResultChain<T, E>` с методами: `map`, `flatMap`, `mapError`, `unwrapOr`, `match`
2. Реализуйте конструкторы `okChain(value)` и `errChain(error)`
3. `map` трансформирует значение, пропускает ошибки
4. `flatMap` связывает операции, **накапливая** типы ошибок через union
5. `mapError` трансформирует тип ошибки
6. `unwrapOr` возвращает значение или fallback
7. `match` выполняет одну из двух функций: ok или err
8. Создайте цепочку из 3 операций с разными типами ошибок

## Чеклист

- [ ] `okChain(42).map(x => x * 2)` возвращает `ResultChain<number, never>`
- [ ] `errChain(e).map(fn)` пропускает fn, возвращая ошибку
- [ ] `flatMap` объединяет типы ошибок: `ResultChain<T, E1 | E2>`
- [ ] `mapError` меняет тип ошибки без влияния на значение
- [ ] `unwrapOr` возвращает fallback при ошибке
- [ ] `match` обрабатывает оба варианта с выводом типов

## Как проверить себя

Создайте цепочку `parseEmail -> validateEmail -> saveUser`, где каждая функция может вернуть свой тип ошибки. Проверьте, что TypeScript знает все возможные типы ошибок в финальном `match`.
