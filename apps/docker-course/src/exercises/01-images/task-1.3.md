# Задание 1.3: docker build

## Цель

Понять процесс сборки образа и ключевые флаги команды `docker build`.

## Требования

Создайте компонент, показывающий:

1. **Основную команду** `docker build` с объяснением каждого аргумента
2. **Ключевые флаги**:
   - `-t` / `--tag` — тегирование образа
   - `-f` / `--file` — указание пути к Dockerfile
   - `--no-cache` — сборка без кэша
   - `--build-arg` — передача аргументов сборки
   - `--target` — сборка конкретного этапа (multi-stage)
3. **Пример вывода** сборки — покажите типичный лог с объяснением каждого шага
4. **Build context** — объясните, что такое контекст сборки и зачем нужен `.dockerignore`

## Подсказки

- `docker build -t myapp:1.0 .` — точка в конце это контекст сборки
- Каждая инструкция в Dockerfile создаёт новый слой
- `--build-arg VERSION=1.0` передаёт переменную, доступную через `ARG`
