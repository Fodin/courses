# Задание 6.2: Validation

## Цель

Реализовать паттерн Validation, который собирает ВСЕ ошибки валидации вместо остановки на первой.

## Требования

1. Создайте классы `Valid<A>` и `Invalid<E>` с полем `_tag` ('valid' | 'invalid')
2. `Invalid` хранит массив ошибок `E[]`
3. Реализуйте функцию `combine` — объединяет несколько Validation в один, накапливая ошибки
4. Создайте валидаторы для формы регистрации:
   - `validateName(name: string)` — не пустое, минимум 2 символа
   - `validateEmail(email: string)` — содержит @
   - `validatePassword(password: string)` — минимум 8 символов, содержит цифру
5. Реализуйте метод `map` на Valid/Invalid
6. Продемонстрируйте: валидная форма и форма со множеством ошибок

## Чеклист

- [ ] Классы `Valid` и `Invalid` с корректными `_tag`
- [ ] `Invalid` хранит массив ошибок
- [ ] `combine` собирает все ошибки из нескольких Validation
- [ ] Три валидатора реализованы с осмысленными сообщениями
- [ ] `map` работает для Valid, пропускает Invalid
- [ ] Демонстрация: валидная форма возвращает Valid, невалидная — все ошибки сразу

## Как проверить себя

- Нажмите кнопку — должны отобразиться два сценария
- Валидная форма: Valid с данными пользователя
- Невалидная форма: Invalid со ВСЕМИ ошибками (не одной)
