# Задание 8.3: Финальный проект — Todo с обработкой ошибок

## Цель
Создать Todo-приложение, объединяющее все техники обработки ошибок из курса.

## Требования
1. Симулируйте API для CRUD-операций (list, add, toggle, delete)
2. Добавьте ошибки:
   - Валидация: пустой текст
   - Лимит: максимум 5 задач
   - Серверная ошибка: 20% шанс при добавлении
   - Not found: при toggle/delete несуществующей задачи
3. Используйте `AppError` с кодами ошибок
4. Отображайте ошибки через `role="alert"` с кнопкой закрытия
5. Добавьте индикатор загрузки
6. Приложение должно оставаться работоспособным при любой ошибке

## Чеклист
- [ ] CRUD API с симуляцией задержки
- [ ] 4 типа ошибок (VALIDATION, LIMIT, SERVER, NOT_FOUND)
- [ ] AppError с кодами
- [ ] Ошибки отображаются и закрываются
- [ ] Loading индикатор
- [ ] Приложение не падает при ошибках
