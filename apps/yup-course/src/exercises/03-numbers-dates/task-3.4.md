# Задание 3.4: Диапазоны дат

## Цель

Реализовать кросс-полевую валидацию дат: дата окончания должна быть позже даты начала с использованием `yup.ref()`.

## Требования

1. Создайте `dateRangeSchema` — объект с полями `startDate` и `endDate`

2. Оба поля обязательны (`required`)

3. `endDate` должна быть позже `startDate` (используйте `min(yup.ref('startDate'))`)

4. Используйте `abortEarly: false` для показа всех ошибок

5. При успехе покажите диапазон и количество дней

## Чеклист

- [ ] Схема — объект (`yup.object()`) с двумя полями дат
- [ ] `endDate` ссылается на `startDate` через `yup.ref()`
- [ ] `abortEarly: false` показывает все ошибки сразу
- [ ] При успехе отображается диапазон и количество дней
- [ ] Оба пустых поля показывают две ошибки required

## Как проверить себя

1. Start: 2024-01-01, End: 2024-12-31 — успех (365 дней)
2. Start: 2024-12-31, End: 2024-01-01 — ошибка "End must be after start"
3. Start: пусто, End: пусто — две ошибки required
4. Start: 2024-06-01, End: 2024-06-01 — успех (0 дней, min разрешает равенство)
