# Задание 10.1: Анализ и уменьшение размера образов

## Цель

Научиться анализировать размер Docker-образов с помощью `docker history`, `docker image inspect` и `dive`, понять разницу между базовыми образами (full, slim, alpine, distroless, scratch) и их влияние на размер финального образа.

## Требования

1. Создайте компонент с двумя переключаемыми секциями: "Инструменты анализа" и "Сравнение базовых образов"
2. Секция "Инструменты анализа" показывает таблицу инструментов (`docker history`, `docker image inspect`, `dive`) с описанием, командой и что показывает
3. Для каждого инструмента добавьте кнопку "Показать пример вывода" -- по клику отображается блок с примером консольного вывода
4. Секция "Сравнение базовых образов" показывает таблицу типов образов (full, slim, alpine, distroless, scratch) с размером, описанием содержимого и когда использовать
5. Добавьте визуализацию: горизонтальные прогресс-бары, показывающие относительный размер каждого типа образа (full = 100%)
6. Внизу компонента: блок "Ключевой вывод" с рекомендацией по выбору базового образа

## Подсказки

- `useState<string>` для активной секции
- `useState<Record<string, boolean>>` для управления видимостью примеров вывода каждого инструмента
- Прогресс-бары: `div` с `width` в процентах и `backgroundColor`
- Массив объектов для образов: `{ name, example, size, sizePercent, description, useCase }`

## Чеклист

- [ ] Две секции переключаются корректно
- [ ] Таблица инструментов анализа (3 инструмента)
- [ ] Кнопки показа примеров вывода работают независимо
- [ ] Таблица базовых образов (5 типов) с размерами
- [ ] Визуальные прогресс-бары размеров
- [ ] Блок "Ключевой вывод" с рекомендацией

## Как проверить себя

1. Переключайте секции -- содержимое меняется
2. В "Инструменты анализа" нажмите кнопки примеров -- примеры вывода показываются/скрываются независимо
3. В "Сравнение базовых образов" видны 5 типов с размерами и прогресс-барами
4. Прогресс-бары визуально отражают разницу в размерах (scratch -- минимальный, full -- максимальный)
5. Блок с рекомендацией видно всегда
