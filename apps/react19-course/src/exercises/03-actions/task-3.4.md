# Задание 3.4: Прогрессивное улучшение

## Цель

Создать форму, которая работает как с JavaScript, так и без него (progressive enhancement).

## Требования

1. Форма должна использовать `action` и стандартные HTML-атрибуты
2. Используйте `<input required>`, `<input type="email">` для валидации
3. Добавьте `useActionState` для клиентской обработки
4. Покажите разницу между HTML-валидацией и JS-валидацией

## Чеклист

- [ ] Форма имеет HTML-валидацию (`required`, `type="email"`)
- [ ] `useActionState` обрабатывает данные на клиенте
- [ ] Форма семантически корректна (label + input)
- [ ] Показана разница подходов

## Как проверить себя

1. Попробуйте отправить пустую форму — сработает HTML-валидация
2. Заполните поля — сработает JS-обработка через action
