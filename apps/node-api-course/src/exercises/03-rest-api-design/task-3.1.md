# Задание 3.1: CRUD Endpoints

## 🎯 Цель

Спроектировать RESTful CRUD endpoints с правильными HTTP-методами, статус-кодами и форматом ответа (Envelope Pattern).

## Требования

1. Определите соответствие HTTP-методов и CRUD-операций: POST=Create, GET=Read, PUT/PATCH=Update, DELETE=Delete
2. Покажите правильные статус-коды для каждой операции (201 Created, 200 OK, 204 No Content)
3. Реализуйте Envelope Pattern для ответов: `{ data, meta }` для коллекций, `{ data }` для единичных ресурсов
4. Реализуйте формат ошибок: `{ error: { code, message, details } }`
5. Перечислите основные клиентские ошибки (400, 401, 403, 404, 409, 422) с описанием

## Чеклист

- [ ] Все CRUD-операции сопоставлены с HTTP-методами и путями
- [ ] POST возвращает 201 + Location header, DELETE возвращает 204
- [ ] Коллекции оборачиваются в `{ data: [...], meta: { total, page, ... } }`
- [ ] Ошибки валидации содержат `details` с указанием полей
- [ ] Перечислены все основные статус-коды (2xx, 4xx) с use-cases

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: таблица CRUD -> HTTP показана корректно, примеры ответов содержат правильную структуру, ошибки имеют machine-readable коды.
