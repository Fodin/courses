# Задание 5.4: Pipeline

## Цель

Освоить `stream.pipeline()` как безопасную замену `.pipe()` с автоматической обработкой ошибок и поддержкой отмены через `AbortSignal`.

## Требования

1. Покажите проблему с `.pipe()` — утечка ресурсов при ошибке
2. Продемонстрируйте `pipeline()` в callback и promise стилях
3. Покажите автоматическое уничтожение стримов при ошибке
4. Продемонстрируйте `AbortSignal` для отмены pipeline
5. Покажите реальные примеры: сжатие файла, HTTP streaming
6. Симулируйте три сценария: успех, ошибка с pipe, ошибка с pipeline

## Чеклист

- [ ] Проблема .pipe() с утечкой объяснена
- [ ] pipeline() в двух стилях показан
- [ ] Автоматическое уничтожение стримов продемонстрировано
- [ ] AbortSignal для отмены показан
- [ ] Реальные примеры использования приведены
- [ ] Симуляция трёх сценариев работает

## Как проверить себя

1. Нажмите "Запустить" и сравните сценарии
2. Убедитесь, что pipe() оставляет стримы открытыми при ошибке
3. Проверьте, что pipeline() уничтожает все стримы
4. Убедитесь, что AbortController корректно отменяет pipeline
