# Задание 0.3: nextTick Starvation

## Цель

Понять, как `process.nextTick` может заблокировать Event Loop, и научиться избегать этой проблемы.

## Требования

1. Объясните, почему рекурсивный `process.nextTick` голодает (starves) I/O
2. Покажите опасный пример кода с рекурсивным nextTick
3. Продемонстрируйте безопасную альтернативу с `setImmediate`
4. Симулируйте ограниченное количество рекурсивных вызовов для сравнения
5. Сформулируйте best practices использования nextTick vs setImmediate

## Чеклист

- [ ] Объяснён механизм starvation
- [ ] Показан опасный пример с рекурсивным nextTick
- [ ] Показана безопасная альтернатива с setImmediate
- [ ] Есть симуляция для наглядного сравнения
- [ ] Сформулированы best practices

## Как проверить себя

1. Откройте задание и запустите симуляцию
2. Убедитесь, что видна разница между nextTick и setImmediate
3. Проверьте, что best practices корректно сформулированы
