# Задание 6.3: ports, volumes, environment

## Цель

Разобраться в конфигурации портов, томов и переменных окружения в docker-compose.yml: синтаксис, варианты записи, типичные паттерны.

## Требования

1. Создайте компонент с тремя секциями, переключаемыми по кнопкам: `ports`, `volumes`, `environment`
2. Секция **ports**: короткий и длинный синтаксис, expose vs ports, привязка к 127.0.0.1. Заметки о кавычках
3. Секция **volumes**: bind mount (./path), именованные тома, анонимные тома, read-only монтирование, объявление томов в корне. Заметки о различии ./path и name
4. Секция **environment**: inline (environment:), из файла (env_file:), подстановка переменных (${VAR:-default}). Заметки о .env и безопасности
5. Под каждой секцией -- список кратких заметок (минимум 3)
6. Общий пример -- полный docker-compose.yml объединяющий все три концепции

## Подсказки

- Массив объектов `{ key, label, yaml, notes }` для секций
- notes -- массив строк с ключевыми фактами
- Покажите реалистичный пример: API + БД с портами, томами и env

## Чеклист

- [ ] Три секции переключаются по кнопкам
- [ ] ports: короткий/длинный синтаксис, expose, 127.0.0.1
- [ ] volumes: bind mount, named volume, anonymous, :ro
- [ ] environment: inline, env_file, ${VAR:-default}
- [ ] Минимум 3 заметки под каждой секцией
- [ ] Полный пример, объединяющий все три концепции

## Как проверить себя

1. Переключайте секции -- YAML и заметки меняются
2. Заметки лаконичны и содержат самое важное
3. Полный пример синтаксически корректен и реалистичен
