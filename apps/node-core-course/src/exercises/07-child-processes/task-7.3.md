# Задание 7.3: fork и IPC

## Цель

Научиться создавать Node.js worker-процессы через `fork`, обмениваться сообщениями через IPC и реализовать паттерн Worker.

## Требования

1. Создайте симуляцию `fork` с созданием дочернего Node.js-процесса
2. Продемонстрируйте IPC-обмен: `child.send()` и `process.on('message')`
3. Реализуйте паттерн Worker: parent отправляет задачу, child возвращает результат
4. Покажите ограничения IPC (JSON-сериализация)
5. Продемонстрируйте корректное завершение дочернего процесса

## Чеклист

- [ ] Симуляция fork с IPC-каналом
- [ ] Двусторонний обмен сообщениями parent ↔ child
- [ ] Паттерн Worker (task → result)
- [ ] Показаны ограничения сериализации
- [ ] Корректное завершение через shutdown-сообщение

## Как проверить себя

1. Нажмите кнопку запуска
2. Убедитесь, что сообщения Parent→Child и Child→Parent различаются цветом
3. Проверьте, что показан полный цикл: задача → обработка → результат
4. Убедитесь, что описаны ограничения IPC
