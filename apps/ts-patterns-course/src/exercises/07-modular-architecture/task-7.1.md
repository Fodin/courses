# Задание 7.1: Dependency Injection

## Цель

Реализовать типобезопасный DI-контейнер с Token<T>, поддержкой Singleton и Transient lifecycle.

## Требования

1. Создайте класс `Token<T>` с полем `name: string` для идентификации зависимости
2. Создайте класс `Container` с методами:
   - `register<T>(token: Token<T>, factory: () => T)` — Transient (новый экземпляр каждый раз)
   - `registerSingleton<T>(token: Token<T>, factory: () => T)` — Singleton (один экземпляр)
   - `resolve<T>(token: Token<T>): T` — получение зависимости по токену
3. При resolve несуществующего токена — бросить Error
4. Создайте интерфейсы `Logger` и `Database` с методами
5. Зарегистрируйте реализации и продемонстрируйте:
   - Transient: каждый resolve возвращает новый экземпляр
   - Singleton: каждый resolve возвращает тот же экземпляр
   - Ошибку при resolve незарегистрированного токена

## Чеклист

- [ ] Класс `Token<T>` с типовым параметром
- [ ] `register` сохраняет фабрику
- [ ] `registerSingleton` кэширует первый вызов
- [ ] `resolve` возвращает типизированный результат
- [ ] Ошибка при отсутствии привязки
- [ ] Демонстрация Transient vs Singleton

## Как проверить себя

- Нажмите кнопку — должна отобразиться демонстрация
- Transient: два resolve дают разные id экземпляров
- Singleton: два resolve дают одинаковый id
- Незарегистрированный токен вызывает ошибку
