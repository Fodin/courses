# Задание 0.4: Query-параметры

## 🎯 Цель

Реализовать типобезопасное построение и парсинг URL query-параметров на основе декларативной схемы.

## Требования

1. Создайте интерфейс `QuerySchema` для описания допустимых параметров и их типов
2. Реализуйте условный тип `QueryValues<T>`, который выводит TypeScript-типы из схемы
3. Реализуйте функцию `buildQueryString()` для сериализации типизированных параметров в строку
4. Реализуйте функцию `parseQueryString()` для десериализации строки обратно в типизированный объект
5. Поддержите типы: `string`, `number`, `boolean`, `string[]`
6. Продемонстрируйте roundtrip: build -> parse -> сравнение

## Чеклист

- [ ] Схема описывает параметры с типами `string | number | boolean | string[]`
- [ ] `QueryValues` корректно выводит TypeScript-типы из схемы
- [ ] `buildQueryString` не принимает параметры с неверными типами
- [ ] `parseQueryString` возвращает объект с правильными типами
- [ ] Массивы сериализуются как повторяющиеся параметры (`roles=a&roles=b`)
- [ ] Roundtrip-тест проходит: build(parse(qs)) === qs

## Как проверить себя

Попробуйте передать числовое значение в строковый параметр — TypeScript должен показать ошибку. Проверьте, что парсинг корректно восстанавливает типы из строки.
