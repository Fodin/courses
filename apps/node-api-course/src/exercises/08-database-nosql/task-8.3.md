# Задание 8.3: Redis

## 🎯 Цель

Освоить Redis: базовые операции get/set/del с TTL, паттерн cache-aside для кэширования данных из БД, и pub/sub для обмена сообщениями между сервисами.

## Требования

1. Подключитесь к Redis через `createClient`, покажите операции со строками (set, get, setEx с TTL), хешами (hSet, hGetAll), списками (lPush, rPop), множествами (sAdd, sMembers)
2. Реализуйте паттерн cache-aside: проверка кэша -> запрос к БД -> сохранение в кэш с TTL -> инвалидация при обновлении
3. Покажите pub/sub: отдельное соединение для подписчика, subscribe/pSubscribe, publish, unsubscribe

## Чеклист

- [ ] Redis подключен, продемонстрированы 4+ структуры данных
- [ ] Все ключи имеют TTL (нет ключей без срока жизни)
- [ ] Cache-aside: cache hit возвращает данные мгновенно, cache miss делает запрос к БД
- [ ] Инвалидация кэша при обновлении данных
- [ ] Pub/sub использует отдельное соединение для подписчика

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: Redis-операции показаны для всех структур данных, cache-aside демонстрирует разницу cache hit/miss, pub/sub работает с pattern-подпиской.
