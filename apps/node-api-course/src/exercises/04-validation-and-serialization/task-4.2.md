# Задание 4.2: Joi Validation

## 🎯 Цель

Реализовать валидацию с Joi: пользовательские сообщения об ошибках, условная валидация через `when/alternatives` и сравнение с Zod.

## Требования

1. Определите Joi-схему с кастомными сообщениями: `.messages({ 'string.min': '...' })`
2. Реализуйте валидацию пароля с pattern и кастомным сообщением
3. Реализуйте условную валидацию через `Joi.when()`: разные поля в зависимости от типа (card vs bank)
4. Продемонстрируйте валидацию с ошибками и показом кастомных сообщений
5. Сравните Joi и Zod: плюсы и минусы каждого

## Чеклист

- [ ] `.messages()` переопределяет стандартные сообщения на русский
- [ ] `Joi.ref('password')` проверяет совпадение полей
- [ ] `Joi.when('type', { is, then, otherwise })` реализует условную логику
- [ ] `Joi.forbidden()` запрещает поля в определённых условиях
- [ ] Таблица сравнения: Joi (условная валидация, сообщения) vs Zod (TypeScript, размер)

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: кастомные сообщения показаны на русском, условная валидация корректно проверяет card/bank поля, Joi vs Zod сравнение информативно.
