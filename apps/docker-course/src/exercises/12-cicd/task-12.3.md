# Задание 12.3: Деплой и мониторинг

## Цель

Создать интерактивный компонент-справочник по стратегиям деплоя Docker-приложений и мониторингу в production. Научиться выбирать стратегию деплоя, настраивать health checks и понимать, какие метрики мониторить.

## Требования

1. Создайте компонент с тремя переключаемыми секциями: "Стратегии деплоя", "Health Checks", "Мониторинг"
2. Секция "Стратегии деплоя": три кнопки-переключателя (Rolling Update, Blue-Green, Canary) — по клику отображается описание, плюсы/минусы в два столбца, ASCII-диаграмма процесса и кнопка "Показать конфиг"
3. Секция "Health Checks": три карточки (Liveness, Readiness, Startup) — каждая с цветной верхней границей, описанием, действием и примером
4. Симулятор Health Check: кнопка "Запустить проверку" последовательно проверяет database, redis, app — показывает статус каждого (OK/FAIL) с цветной индикацией
5. Итоговое сообщение: зелёное "Все проверки пройдены" или красное "Health check не пройден! Требуется rollback"
6. Пример Docker Compose healthcheck конфигурации
7. Секция "Мониторинг": три группы метрик (Контейнер, Приложение, Инфраструктура) с метриками в виде бейджей
8. Пример стека Prometheus + Grafana + cAdvisor в docker-compose

## Подсказки

- `useState<'deploy' | 'health' | 'monitoring'>` для секций
- `useState<number | null>` для выбранной стратегии деплоя
- `setTimeout` для последовательной симуляции health checks
- `Math.random() > 0.3` для случайного результата проверки app (иногда FAIL)
- Массив объектов для стратегий: `{ name, description, pros, cons, diagram, config }`

## Чеклист

- [ ] Три секции переключаются корректно
- [ ] Три стратегии деплоя с плюсами/минусами и диаграммами
- [ ] Кнопка "Показать конфиг" для каждой стратегии
- [ ] Три карточки типов health check (Liveness, Readiness, Startup)
- [ ] Симулятор health check с последовательной проверкой
- [ ] Итоговое сообщение (зелёное/красное) после проверки
- [ ] Три группы метрик мониторинга с бейджами
- [ ] Пример docker-compose стека мониторинга

## Как проверить себя

1. Переключайте секции — содержимое корректно меняется
2. В "Стратегии деплоя" переключайте стратегии — описание, плюсы/минусы и диаграмма обновляются
3. Нажмите "Показать конфиг" — конфигурация деплоя отображается
4. В "Health Checks" видны три карточки с цветными верхними границами
5. Нажмите "Запустить проверку" — database, redis, app последовательно получают статус
6. После проверки появляется зелёное или красное итоговое сообщение
7. В "Мониторинг" видны три группы метрик и docker-compose конфигурация
