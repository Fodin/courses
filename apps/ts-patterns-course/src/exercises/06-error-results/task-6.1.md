# Задание 6.1: Result / Either

## Цель

Реализовать паттерн Result<T, E> для типобезопасной обработки ошибок без throw/catch.

## Требования

1. Создайте классы `Ok<T>` и `Err<E>` с полем `_tag` ('ok' | 'err')
2. Реализуйте метод `map` — трансформация значения внутри Ok (Err остаётся без изменений)
3. Реализуйте метод `flatMap` — цепочка операций, каждая из которых возвращает Result
4. Реализуйте метод `match` — обработка обоих вариантов через объект `{ ok, err }`
5. Создайте функцию `fromThrowable(fn)` — оборачивает функцию, которая может throw, в Result
6. Создайте хелперы `ok(value)` и `err(error)`
7. Продемонстрируйте цепочку: parseJSON → extractField → validateValue

## Чеклист

- [ ] Классы `Ok` и `Err` с корректными `_tag`
- [ ] `map` работает для Ok и пропускает Err
- [ ] `flatMap` разворачивает вложенные Result
- [ ] `match` вызывает правильный обработчик
- [ ] `fromThrowable` ловит throw и возвращает Err
- [ ] Хелперы `ok()` / `err()` создают экземпляры
- [ ] Демонстрация цепочки операций работает

## Как проверить себя

- Нажмите кнопку — должна отобразиться цепочка трансформаций
- Ok-значения проходят через map/flatMap
- Err-значения пропускают все трансформации до match
- fromThrowable корректно ловит исключения
