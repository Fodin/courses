# Задание 0.1: Branded Types

## Цель

Научиться создавать брендированные типы для предотвращения смешивания значений одного примитивного типа.

## Требования

1. Создайте утилитарный тип `Brand<T, B>` для брендирования
2. Создайте три branded-типа: `UserId`, `Email`, `OrderId`
3. Для каждого типа создайте функцию-конструктор с валидацией:
   - `createUserId(id: string)` — минимум 3 символа
   - `createEmail(value: string)` — валидный email (regex)
   - `createOrderId(id: string)` — формат `ORD-XXXX` (цифры)
4. Создайте функции, принимающие только конкретный branded-тип:
   - `findUser(userId: UserId): string`
   - `sendEmail(email: Email): string`
   - `getOrder(orderId: OrderId): string`
5. Продемонстрируйте успешное создание и ошибки валидации

## Чеклист

- [ ] Тип `Brand<T, B>` корректно определён
- [ ] Три branded-типа созданы
- [ ] Функции-конструкторы валидируют входные данные
- [ ] Невалидные данные вызывают throw Error
- [ ] Функции принимают только свой branded-тип
- [ ] Демонстрация работает — кнопка выводит результаты
