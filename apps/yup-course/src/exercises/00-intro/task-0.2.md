# Задание 0.2: Обработка ошибок

## Цель

Научиться извлекать ошибки из `ValidationError` и отображать их рядом с соответствующими полями.

## Требования

1. Создайте схему `registrationSchema` с полями:
   - `username` — обязательная строка, минимум 3 символа
   - `email` — обязательная строка в формате email
   - `password` — обязательная строка, минимум 6 символов

2. При валидации используйте `{ abortEarly: false }` для сбора всех ошибок

3. Из `ValidationError.inner` извлеките ошибки по полям (используя `error.path` и `error.message`)

4. Отображайте ошибку валидации под каждым полем, к которому она относится

5. Дополнительно покажите общий список всех ошибок в блоке внизу

6. При успешной валидации покажите валидированные данные

## Чеклист

- [ ] Схема `registrationSchema` создана
- [ ] Используется `abortEarly: false`
- [ ] Ошибки извлекаются из `err.inner`
- [ ] Каждое поле показывает свою ошибку
- [ ] Общий список ошибок отображается внизу
- [ ] Проверяется `instanceof yup.ValidationError`

## Как проверить себя

1. Оставьте все поля пустыми — под каждым полем должна быть ошибка "required"
2. Введите username "ab" — должна быть ошибка "at least 3 characters"
3. Введите password "123" — должна быть ошибка "at least 6 characters"
4. Заполните все поля корректно — ошибки должны исчезнуть, данные отобразиться
