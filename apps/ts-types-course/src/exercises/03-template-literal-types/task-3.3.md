# Задание 3.3: Template Literal Parsing

## Цель

Научиться парсить строковые типы с помощью template literals и `infer`: разделение строк, извлечение параметров маршрутов, создание type-safe route builder.

## Требования

1. Реализуйте тип `Split<S, D>` -- разделяет строку на tuple по разделителю. Создайте runtime-функцию
2. Реализуйте тип `Join<T, D>` -- объединяет tuple строк в одну строку через разделитель
3. Реализуйте тип `ParseRouteParams<S>` -- извлекает имена параметров из URL-шаблона (`:param`). Создайте runtime-функцию `extractRouteParams`
4. Реализуйте type-safe функцию `buildRoute(template, params)`, которая подставляет параметры в URL-шаблон
5. Реализуйте типы `TrimStart<S>`, `TrimEnd<S>`, `Trim<S>` для удаления пробелов
6. Реализуйте тип `ExtractDomain<S>` для извлечения домена из email-адреса

## Чеклист

- [ ] `Split<'a.b.c', '.'>` разрешается в `['a', 'b', 'c']`
- [ ] `Join<['x', 'y'], '-'>` разрешается в `'x-y'`
- [ ] `ParseRouteParams<'/users/:id/posts/:postId'>` разрешается в `'id' | 'postId'`
- [ ] `buildRoute` корректно подставляет параметры в шаблон
- [ ] `Trim<'  hello  '>` разрешается в `'hello'`
- [ ] `ExtractDomain<'user@example.com'>` разрешается в `'example.com'`

## Как проверить себя

1. Проверьте Split для строк с 1, 2 и 3+ разделителями
2. Вызовите buildRoute с маршрутом, содержащим 2-3 параметра
3. Проверьте Trim для строк с пробелами только слева, только справа, с обеих сторон
