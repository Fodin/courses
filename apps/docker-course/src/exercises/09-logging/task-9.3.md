# Задание 9.3: Отладка типичных ошибок

## Цель

Научиться диагностировать и решать типичные проблемы контейнеров: немедленное завершение, OOM, конфликт портов, permission denied, сетевые ошибки. Освоить пошаговый алгоритм отладки.

## Требования

1. Создайте компонент с переключаемыми сценариями ошибок: "Exit 0", "Exit 1", "Exit 137 (OOM)", "Port conflict", "Permission denied", "Network issues"
2. Каждый сценарий содержит: симптом (что видит пользователь), причину, команды диагностики и решение
3. Симптом отображается как блок кода с выводом `docker ps -a`
4. Решение отображается как блок кода с рабочими командами или конфигурацией
5. Добавьте кнопку "Показать алгоритм отладки" -- по нажатию отображается пошаговый алгоритм: ps → logs → inspect → exec → events → stats → system df
6. Внизу компонента: таблица exit codes (0, 1, 126, 127, 137, 143) с расшифровкой

## Подсказки

- Массив сценариев: `{ key, label, symptom, cause, diagnosis, solution }[]`
- symptom, diagnosis, solution -- многострочные строки для `<pre>`
- `useState<string>` для активного сценария, `useState<boolean>` для алгоритма
- Exit codes: `{ code, signal, meaning }[]`

## Чеклист

- [ ] Шесть сценариев ошибок переключаются корректно
- [ ] Каждый сценарий: симптом + причина + диагностика + решение
- [ ] Симптом и решение в блоках кода
- [ ] Алгоритм отладки (7 шагов) по кнопке
- [ ] Таблица exit codes (6 кодов) внизу
- [ ] Для Exit 137 проверка OOMKilled через inspect

## Как проверить себя

1. Переключайте сценарии -- каждый показывает уникальный набор данных
2. В каждом сценарии есть все 4 блока: симптом, причина, диагностика, решение
3. Сценарий "Exit 137" включает проверку OOMKilled и увеличение лимита памяти
4. Сценарий "Port conflict" включает lsof/netstat для поиска занятого порта
5. Алгоритм отладки: 7 пронумерованных шагов от ps до system df
6. Таблица exit codes: 0 (OK), 1 (ошибка), 126, 127, 137 (OOM/SIGKILL), 143 (SIGTERM)
