# Задание 7.1: depends_on и healthcheck

## Цель

Разобраться в механизмах управления порядком запуска сервисов: простая и расширенная формы `depends_on`, настройка `healthcheck` для популярных сервисов, три условия зависимости.

## Требования

1. Создайте компонент с тремя табами: "depends_on", "healthcheck", "Условия"
2. Таб "depends_on" -- два примера с переключением: простая форма (список) и расширенная (с `condition`). Для простой формы покажите предупреждение о том, что она не ждёт готовности сервиса
3. Таб "healthcheck" -- интерактивный список примеров healthcheck для разных сервисов: PostgreSQL, Redis, HTTP (curl), HTTP (wget), MySQL, MongoDB. Для каждого покажите YAML-конфигурацию с параметрами
4. Под примерами healthcheck покажите описание параметров: `interval`, `timeout`, `retries`, `start_period`
5. Таб "Условия" -- таблица с тремя условиями `depends_on`: `service_started`, `service_healthy`, `service_completed_successfully`. Для каждого: что означает и когда использовать
6. В табе "Условия" добавьте визуализацию порядка запуска стека (7 шагов)

## Подсказки

- Массив объектов для примеров depends_on: `{ key, label, yaml, warning }`
- Массив для healthcheck: `{ service, yaml }`
- `useState<'depends_on' | 'healthcheck' | 'conditions'>` для табов

## Чеклист

- [ ] Три таба переключаются корректно
- [ ] Простая форма depends_on с предупреждением
- [ ] Расширенная форма с condition: service_healthy
- [ ] Примеры healthcheck для 6 сервисов
- [ ] Описание параметров healthcheck (interval, timeout, retries, start_period)
- [ ] Таблица с тремя условиями depends_on
- [ ] Визуализация порядка запуска стека

## Как проверить себя

1. Переключайте табы -- контент меняется
2. В "depends_on" переключайте примеры -- YAML обновляется, предупреждение показывается только для простой формы
3. В "healthcheck" кликайте по сервисам -- YAML для каждого уникален
4. В "Условия" все три строки таблицы заполнены, порядок запуска визуально понятен
