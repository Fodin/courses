# Задание 3.2: Discriminated unions

## Цель
Использовать discriminated unions для моделирования состояний загрузки.

## Требования
1. Определите тип `AsyncState<T>` с вариантами: `idle`, `loading`, `success`, `error`
2. Каждый вариант имеет поле `status` (дискриминант)
3. `success` содержит `data: T`, `error` содержит `error: string` и `retryCount`
4. Создайте функцию `renderState(state)`, обрабатывающую все варианты
5. Симулируйте загрузку пользователя с случайным результатом

## Чеклист
- [ ] `AsyncState<T>` определён с 4 вариантами
- [ ] Дискриминант `status` работает для сужения типов
- [ ] Компонент корректно отображает все состояния
- [ ] Визуальная индикация: разные цвета для success/error
