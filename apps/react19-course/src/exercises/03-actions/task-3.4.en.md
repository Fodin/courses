# Task 3.4: Progressive Enhancement

## Objective

Create a form that works both with and without JavaScript (progressive enhancement).

## Requirements

1. The form should use `action` and standard HTML attributes
2. Use `<input required>`, `<input type="email">` for validation
3. Add `useActionState` for client-side processing
4. Show the difference between HTML validation and JS validation

## Checklist

- [ ] The form has HTML validation (`required`, `type="email"`)
- [ ] `useActionState` processes data on the client
- [ ] The form is semantically correct (label + input)
- [ ] The difference between approaches is demonstrated

## How to Verify

1. Try submitting the empty form — HTML validation triggers
2. Fill in the fields — JS processing via action triggers
