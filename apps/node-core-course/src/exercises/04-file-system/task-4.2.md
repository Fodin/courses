# Задание 4.2: Path Module

## Цель

Освоить модуль `path` для безопасной и кроссплатформенной работы с путями файловой системы.

## Требования

1. Продемонстрируйте `path.join()` для безопасной конкатенации путей
2. Покажите `path.resolve()` для получения абсолютного пути
3. Продемонстрируйте `path.dirname()`, `path.basename()`, `path.extname()` для разбора путей
4. Покажите `path.parse()` и полный объект разбора пути
5. Продемонстрируйте `path.relative()` для вычисления относительного пути
6. Покажите кроссплатформенные особенности: `path.sep`, `path.delimiter`, `path.posix`, `path.win32`

## Чеклист

- [ ] path.join() показан с примерами, включая `..`
- [ ] path.resolve() продемонстрирован с относительными и абсолютными путями
- [ ] dirname, basename, extname показаны на реальном примере пути
- [ ] path.parse() показан с описанием всех полей объекта
- [ ] path.relative() продемонстрирован
- [ ] Кроссплатформенные особенности упомянуты

## Как проверить себя

1. Нажмите "Запустить" и изучите результат каждой функции
2. Убедитесь, что path.join() корректно обрабатывает `..`
3. Проверьте, что path.parse() возвращает все 5 полей (root, dir, base, name, ext)
4. Убедитесь, что упомянуто правило: всегда использовать path.join() вместо конкатенации строк
