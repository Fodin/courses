# Задание 1.1: Типизированный Event Emitter

## 🎯 Цель

Реализовать generic EventEmitter, в котором TypeScript гарантирует соответствие между именем события и типом payload.

## Требования

1. Определите интерфейс `AppEvents` с минимум 4 событиями разной структуры
2. Создайте класс `TypedEventEmitter<TEvents>` с методами:
   - `on(event, handler)` — подписка, возвращает функцию отписки
   - `off(event, handler)` — отписка
   - `emit(event, payload)` — отправка события
   - `once(event, handler)` — одноразовая подписка
   - `listenerCount(event)` — количество подписчиков
   - `removeAllListeners(event?)` — удаление всех подписчиков
3. Все методы должны быть типобезопасными — TypeScript выводит тип payload из имени события
4. Продемонстрируйте работу всех методов

## Чеклист

- [ ] `emit('user:login', { userId: 42 })` вызывает ошибку компиляции (userId должен быть string)
- [ ] `emit('typo:event', {})` вызывает ошибку (события нет в карте)
- [ ] `on()` возвращает функцию отписки
- [ ] `once()` вызывает handler только один раз
- [ ] `removeAllListeners()` без аргумента очищает все подписки

## Как проверить себя

Попробуйте подписаться на несуществующее событие или передать неверный payload — TypeScript должен выдать ошибку. Проверьте, что after `on()` handler получает payload с правильным типом.
