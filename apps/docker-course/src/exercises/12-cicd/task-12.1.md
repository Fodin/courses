# Задание 12.1: CI Pipeline — сборка и тестирование

## Цель

Создать интерактивный компонент-справочник по CI/CD пайплайну для Docker: стадии, кэширование слоёв и matrix builds. Научиться понимать, из каких этапов состоит автоматическая сборка и тестирование Docker-образов.

## Требования

1. Создайте компонент с тремя переключаемыми секциями: "CI Pipeline", "Кэширование", "Matrix Builds"
2. Секция "CI Pipeline": список из 6 стадий (Lint, Build, Test, Scan, Push, Deploy) — каждая с названием, иконкой, описанием, инструментами и примерным временем выполнения
3. По клику на стадию раскрывается подробная информация с инструментами (в виде бейджей) и кнопкой "Показать конфиг"
4. Кнопка "Запустить Pipeline" анимирует последовательное прохождение стадий (зелёный = пройдено, жёлтый = в процессе)
5. Секция "Кэширование": таблица типов кэша (gha, registry, local, s3) с описанием, плюсами и минусами
6. Блок с примером использования cache-from/cache-to и пояснение mode=max
7. Секция "Matrix Builds": 3 примера matrix strategy (мультиплатформа, множество версий, include/exclude)

## Подсказки

- `useState<'pipeline' | 'cache' | 'matrix'>` для переключения секций
- `useState<number | null>` для раскрытия стадии
- `setInterval` для анимации пайплайна с очисткой в конце
- Массив объектов для стадий: `{ name, icon, description, tools, duration, config }`
- Inline styles для всей стилизации

## Чеклист

- [ ] Три секции переключаются корректно
- [ ] 6 стадий пайплайна с описанием и инструментами
- [ ] Кнопка "Показать конфиг" работает независимо для каждой стадии
- [ ] Анимация "Запустить Pipeline" последовательно проходит все стадии
- [ ] Таблица типов кэша (минимум 4 типа) с плюсами и минусами
- [ ] Пример cache-from/cache-to с пояснением mode=max
- [ ] 3 примера matrix strategy с конфигурацией

## Как проверить себя

1. Переключайте секции — содержимое корректно меняется
2. В "CI Pipeline" кликайте на стадии — раскрываются детали с инструментами
3. Нажмите "Показать конфиг" — YAML-конфигурация отображается
4. Нажмите "Запустить Pipeline" — стадии последовательно становятся жёлтыми, затем зелёными
5. В "Кэширование" видна таблица с 4+ типами и пример использования
6. В "Matrix Builds" видны 3 примера с YAML-конфигурацией
