# Задание 2.2: Плагины и декораторы

## 🎯 Цель

Понять систему плагинов Fastify: инкапсуляция, `fastify-plugin` для пробивания скоупа, декораторы для расширения instance, request и reply.

## Требования

1. Создайте плагин с функцией `async (fastify, opts) => { ... }`
2. Покажите инкапсуляцию: плагин A не видит плагин B по умолчанию
3. Продемонстрируйте `fastify-plugin` (fp) для пробивания инкапсуляции (глобальный scope)
4. Покажите три типа декораторов: `decorate` (instance), `decorateRequest`, `decorateReply`
5. Продемонстрируйте загрузку плагинов: CORS, DB, auth, routes

## Чеклист

- [ ] Плагин определён как async-функция с `(fastify, opts)`
- [ ] Инкапсуляция: плагины изолированы друг от друга
- [ ] `fastify-plugin` оборачивает плагин для глобального доступа (db, auth)
- [ ] Декораторы: `decorate`, `decorateRequest`, `decorateReply` показаны с примерами
- [ ] Порядок загрузки: сначала utility-плагины (fp), затем route-плагины

## Как проверить себя

Нажмите "Запустить" и убедитесь, что показаны: структура плагина, разница между инкапсулированным и глобальным плагином, три типа декораторов и порядок загрузки.
