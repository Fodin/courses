# Задание 4.2: Fallback UI

## Цель
Научиться создавать информативный и красивый fallback UI для Error Boundary.

## Требования
1. Создайте `ErrorFallback` компонент с:
   - Заголовком об ошибке
   - Описанием проблемы
   - Секцией `<details>` с полным стеком ошибки
2. Добавьте в Error Boundary проп `fallback` — функцию, принимающую `{ error }`
3. Если `fallback` передан — используйте его, иначе — стандартный `ErrorFallback`
4. Продемонстрируйте два варианта: стандартный и кастомный fallback

## Чеклист
- [ ] `ErrorFallback` с красивым UI
- [ ] Секция `<details>` со стеком
- [ ] Проп `fallback` принимает функцию
- [ ] Показаны два варианта fallback
