# Задание 5.1: Value Objects

## Цель

Реализовать иммутабельные Value Objects с использованием branded types, обеспечивающие типобезопасность доменных примитивов и защиту от смешивания типов.

## Требования

1. Реализуйте базовый тип `Brand<T, B>` с помощью `unique symbol` для предотвращения структурной совместимости
2. Создайте branded types: `Email`, `Money`, `PositiveInt`, `UserId`
3. Для каждого branded type реализуйте фабричную функцию с валидацией: `createEmail`, `createMoney`, `createPositiveInt`, `createUserId`
4. Создайте составной Value Object `Price` с полями `amount: Money` и `currency: Currency`
5. Реализуйте операции: `addPrices` (с проверкой совпадения валют), `formatPrice`
6. Продемонстрируйте, что смешивание разных branded types вызывает ошибку компиляции

## Чеклист

- [ ] `Brand<T, B>` использует `unique symbol` для изоляции типов
- [ ] `createEmail` валидирует формат и приводит к lowercase
- [ ] `createMoney` округляет до 2 знаков после запятой
- [ ] `createPositiveInt` отклоняет нецелые и неположительные числа
- [ ] `addPrices` выбрасывает ошибку при разных валютах
- [ ] Компонент демонстрирует успешные и неуспешные попытки создания

## Как проверить себя

1. Попробуйте присвоить обычный `string` переменной типа `Email` -- должна быть ошибка TS
2. Попробуйте передать `Email` туда, где ожидается `UserId` -- должна быть ошибка TS
3. `createEmail('invalid')` должен выбросить runtime ошибку
