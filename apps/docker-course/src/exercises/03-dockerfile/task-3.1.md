# Задание 3.1: WORKDIR, ENV, ARG

## Цель

Разобраться в инструкциях WORKDIR, ENV и ARG: понять их назначение, область видимости и различия между build-time и runtime переменными.

## Требования

1. Создайте интерактивный компонент с табами для переключения между инструкциями: WORKDIR, ENV, ARG, ARG + ENV
2. Для каждой инструкции покажите пример Dockerfile с подсветкой синтаксиса (используйте `<pre>` с тёмным фоном)
3. Под каждым примером добавьте пояснение -- что делает инструкция и когда её использовать
4. Создайте таблицу сравнения ENV vs ARG с колонками: характеристика, ENV, ARG
5. Таблица должна показываться/скрываться по кнопке
6. В таблице укажите минимум 5 характеристик: доступность при сборке, в контейнере, сохранение в образе, переопределение при сборке, переопределение при запуске

## Подсказки

- Используйте `useState` для управления активным табом и видимостью таблицы
- Для табов определите массив объектов с ключом и меткой
- Примеры Dockerfile храните в массиве объектов с полями: title, type, dockerfile, explanation
- Для таблицы сравнения используйте массив с булевыми значениями и подсветку зелёным/красным

## Чеклист

- [ ] Реализованы табы для переключения между WORKDIR, ENV, ARG, ARG + ENV
- [ ] Каждый таб показывает пример Dockerfile в `<pre>` блоке
- [ ] Под примером есть текстовое пояснение
- [ ] Кнопка показа/скрытия таблицы сравнения ENV vs ARG работает
- [ ] Таблица содержит минимум 5 строк с характеристиками
- [ ] Ячейки таблицы визуально подсвечены (зелёный/красный)

## Как проверить себя

1. Переключайте табы -- каждый должен показывать свой пример и пояснение
2. Нажмите кнопку сравнения -- таблица должна появиться/скрыться
3. Убедитесь, что примеры Dockerfile содержат корректный синтаксис
4. Проверьте, что в таблице правильно указано, какие возможности есть у ENV и ARG
