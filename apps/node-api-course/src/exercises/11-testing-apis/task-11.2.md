# Задание 11.2: Integration Tests

## 🎯 Цель

Освоить integration-тестирование через supertest: полный цикл HTTP-запроса, setup/teardown, тестирование с аутентификацией.

## Требования

1. Настройте supertest с Express-приложением (`request(app)`)
2. Тестируйте CRUD: POST (201, 400, 409), GET (200, 401), PUT, DELETE
3. Проверяйте статус, Content-Type, структуру тела ответа (toMatchObject)
4. Тестируйте аутентификацию: получение токена в beforeAll, отправка через set('Authorization')
5. Проверяйте, что пароли не утекают в ответе (not.toHaveProperty)

## Чеклист

- [ ] Supertest настроен и отправляет реальные HTTP-запросы
- [ ] CRUD endpoints покрыты (happy path + error cases)
- [ ] Аутентификация тестируется: 200 с токеном, 401 без токена
- [ ] Тело ответа проверяется через toMatchObject
- [ ] Чувствительные данные (password) не утекают

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: supertest выполняет запросы к API, все CRUD-операции проверяются, аутентификация работает.
