# Задание 12.2: Input Sanitization

## 🎯 Цель

Освоить защиту от инъекций: SQL injection через параметризованные запросы, NoSQL injection через express-mongo-sanitize, XSS через экранирование и DOMPurify.

## Требования

1. Покажите SQL injection атаку и защиту через параметризованные запросы ($1, $2)
2. Покажите NoSQL injection атаку ({ $gt: "" }) и защиту через express-mongo-sanitize
3. Реализуйте XSS-защиту: xss() для экранирования, DOMPurify для HTML-контента
4. Покажите валидацию через Zod как универсальную защиту от инъекций
5. Продемонстрируйте transform в Zod-схеме для санитизации на уровне валидации

## Чеклист

- [ ] SQL injection показан и предотвращён параметризацией
- [ ] NoSQL injection показан и предотвращён express-mongo-sanitize
- [ ] XSS экранирован через xss() и DOMPurify
- [ ] Zod валидирует и трансформирует входные данные
- [ ] Все подходы демонстрируются с атакой и защитой

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: инъекции показаны как атака, каждый тип имеет конкретную защиту, Zod валидирует input.
