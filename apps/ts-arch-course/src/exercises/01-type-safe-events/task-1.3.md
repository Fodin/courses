# Задание 1.3: Типизация DOM-событий

## 🎯 Цель

Создать типобезопасную обёртку над DOM CustomEvent API, обеспечивающую связь между именем события и типом detail.

## Требования

1. Определите `CustomEventMap` с минимум 5 пользовательскими DOM-событиями
2. Реализуйте `dispatchTypedEvent()` — отправка события с типизированным detail
3. Реализуйте `onTypedEvent()` — подписка с автоматическим выводом типа detail, возвращает cleanup
4. Создайте условный тип `TypedElementEvents<E>`, который определяет доступные события в зависимости от типа HTML-элемента
5. Продемонстрируйте dispatch и подписку для всех событий

## Чеклист

- [ ] `dispatchTypedEvent('app:theme-change', { theme: 'blue' })` вызывает ошибку
- [ ] `onTypedEvent` автоматически выводит тип detail из имени события
- [ ] Cleanup-функция корректно удаляет listener
- [ ] `TypedElementEvents` использует conditional types для element-specific событий
- [ ] События используют bubbles: true для корректного всплытия

## Как проверить себя

Отправьте событие и проверьте, что подписчик получает payload с правильным типом. Убедитесь, что cleanup-функция действительно предотвращает вызов handler после отписки.
