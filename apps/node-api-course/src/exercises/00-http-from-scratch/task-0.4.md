# Задание 0.4: Парсинг тела POST-запроса

## 🎯 Цель

Реализовать парсинг тела HTTP-запроса для различных Content-Type: JSON, URL-encoded form data и обработку чанковой передачи.

## Требования

1. Реализуйте чтение тела запроса через события `data` и `end` объекта `req` (Readable Stream)
2. Добавьте защиту от слишком больших тел запросов (лимит размера)
3. Распарсите JSON body через `JSON.parse()` и покажите результат
4. Распарсите URL-encoded form data через `new URLSearchParams()`
5. Реализуйте определение типа парсинга по заголовку Content-Type
6. Обработайте ошибки парсинга (невалидный JSON -> 400 Bad Request)

## Чеклист

- [ ] Тело запроса читается через `req.on('data')` и `req.on('end')`
- [ ] Реализована защита: `if (body.length > 1e6) req.destroy()`
- [ ] JSON body корректно парсится из `application/json`
- [ ] Form data корректно парсится из `application/x-www-form-urlencoded`
- [ ] Тип парсера определяется по Content-Type заголовку
- [ ] Невалидный JSON возвращает 400 Bad Request с сообщением об ошибке

## Как проверить себя

Нажмите "Запустить" и убедитесь, что: JSON body корректно парсится, form data преобразуется в key-value пары, показана таблица Content-Type -> парсер, невалидный JSON обрабатывается с ошибкой 400.
