# Задание 12.1: Typed Mocks

## Цель

Создать типобезопасную фабрику моков, которая автоматически сохраняет контракт интерфейса, трекает вызовы и позволяет задавать возвращаемые значения.

## Требования

1. Создайте тип `MockFunction<TArgs, TReturn>` с методами: `calls`, `mockReturnValue`, `mockImplementation`, `calledWith`, `calledTimes`, `reset`
2. Реализуйте `createMockFn(defaultReturn)`, который создаёт мок-функцию с заданным дефолтным возвращаемым значением
3. Создайте тип `MockOf<T>`, который преобразует каждый метод интерфейса в `MockFunction`
4. Реализуйте `createMock<T>(defaults)`, который создаёт мок целого интерфейса
5. Продемонстрируйте моки для `UserRepository`, `EmailService`, `Logger`
6. Покажите: трекинг вызовов, `mockReturnValue`, `mockImplementation`, `calledWith`, `reset`

## Чеклист

- [ ] `MockFunction` сохраняет типы аргументов и возвращаемого значения
- [ ] `calls` содержит массив всех вызовов с аргументами
- [ ] `mockReturnValue` принимает только значение правильного типа
- [ ] `mockImplementation` принимает функцию с правильной сигнатурой
- [ ] `calledWith` проверяет, был ли вызов с конкретными аргументами
- [ ] `reset` очищает историю вызовов
- [ ] `MockOf<T>` преобразует все методы интерфейса в моки

## Как проверить себя

1. Создайте мок UserRepository и вызовите `findById` -- проверьте `calls`
2. Задайте `mockReturnValue` и проверьте, что следующий вызов вернёт это значение
3. Используйте `mockImplementation` с кастомной логикой
4. Сбросьте мок через `reset` -- `calledTimes()` должен стать 0
