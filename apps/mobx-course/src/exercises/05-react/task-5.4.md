# Задание 5.4: Гранулярность observer и ререндеры

## 🎯 Цель

Понять, как разделение на мелкие observer-компоненты обеспечивает гранулярные ререндеры — при изменении одного элемента перерисовывается только он, а не весь список.

## 📋 Требования

1. Создайте класс `ItemStore` с `makeAutoObservable`:
   - Observable массив `items` — 20 элементов вида `{ id: number, name: string, count: number }`
   - Action `increment(id)` — увеличивает `count` элемента с данным `id` на 1
2. Создайте экземпляр стора на уровне модуля
3. Создайте **отдельный** observer-компонент `ListItem`:
   - Принимает `item` через пропсы
   - Отслеживает количество ререндеров через `useRef`
   - Отображает имя элемента, его `count`, кнопку `+1` и счётчик ререндеров
4. В основном observer-компоненте:
   - Отобразите все элементы через `ListItem`
   - Добавьте пояснение, что при нажатии +1 перерисовывается только один элемент

```typescript
class ItemStore {
  items = Array.from({ length: 20 }, (_, i) => ({
    id: i,
    name: `Item ${i + 1}`,
    count: 0,
  }))

  increment(id: number): void
}
```

## Чеклист

- [ ] `ItemStore` создан с `makeAutoObservable` и содержит 20 элементов
- [ ] `ListItem` — отдельный observer-компонент
- [ ] Счётчик ререндеров реализован через `useRef`
- [ ] При нажатии +1 увеличивается `count` только у нужного элемента
- [ ] Счётчик ререндеров растёт только у того элемента, на который нажали

## 🔍 Как проверить себя

1. При загрузке у всех элементов `renders: 1`
2. Нажмите +1 на Item 5 — его count станет 1, renders станет 2
3. Проверьте остальные элементы — их renders по-прежнему 1
4. Нажмите +1 на Item 10 — обновится только Item 10, остальные не затронуты
5. Это доказывает гранулярность observer: MobX перерисовывает только компоненты, которые читают изменившиеся данные
